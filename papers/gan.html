Generative Adversarial Networks

Ian Goodfellow, Jean Pouget-Abadie, Mehdi Mirza, Bing Xu, David Warde-Farley, Sherjil Ozair, Aaron Courville, Yoshua Bengio

University of Montreal · NeurIPS 2014

🎭 Adversarial Learning 🖼️ Generative AI Foundation

🎯 Problem Statement

Before GANs, deep generative models faced significant challenges:

  • Difficult optimization: Models like Boltzmann machines required complex approximate inference
  • Markov chain dependency: Many methods relied on expensive iterative sampling processes
  • Blurred outputs: Generative models struggled to produce sharp, realistic samples
  • Limited scalability: Training deep generative models was computationally prohibitive

Key Question: Can we design a generative model that avoids these pitfalls while producing high-quality samples in a single forward pass?

💡 Core Innovation

GANs introduced a revolutionary adversarial training framework where two neural networks compete in a zero-sum game:

🎨 Generator (G)

Takes random noise z ~ p(z) and transforms it into synthetic data G(z) that mimics the real data distribution.

Goal: Fool the discriminator

🔍 Discriminator (D)

Evaluates input data and outputs the probability that the sample came from the real data rather than G.

Goal: Detect fake samples

minG maxD V(D,G) = Ex~pdata(x)[log D(x)] + Ez~pz(z)[log(1 - D(G(z)))]

Analogy: The generator is like a counterfeiter trying to create fake money, while the discriminator is like the police trying to detect counterfeits. As both improve, the fakes become indistinguishable from real currency.

Businesses Using GANs

GANs have revolutionized multiple industries by enabling realistic synthetic data generation:

🏥 Healthcare

Synthetic medical imaging for training diagnostic AI without patient privacy concerns. CTGAN generates pharmacogenetic data for research.

💰 Finance

Fraud detection through synthetic transaction pattern simulation. Banks use GANs to model rare fraudulent activities for robust detection systems.

🎬 Entertainment

Deepfake technology, AI-generated characters in films, and video game asset creation. Enables realistic virtual actors and environments.

🛍️ Retail

Virtual try-on experiences and synthetic product imagery for advertising. Reduces photography costs while enabling personalization.

⚖️ Advantages & Limitations

✓ Advantages

  • No Markov chains needed for sampling
  • Only backpropagation required for training
  • No inference needed during learning
  • Can represent very sharp distributions
  • Wide variety of architectures supported
  • Generator never sees real data directly
  • Semi-supervised learning capabilities

✗ Limitations

  • No explicit representation of pg(x)
  • D and G must be well synchronized during training
  • Training instability and mode collapse
  • Gradient vanishing when D is too strong
  • Hard to achieve Nash equilibrium
  • Difficult to evaluate generation quality
  • Cannot model discrete data well

💻 PyTorch Implementation

import torch import torch.nn as nn # Generator Network class Generator(nn.Module): def __init__(self, latent_dim, img_dim): super().__init__() self.model = nn.Sequential( nn.Linear(latent_dim, 256), nn.LeakyReLU(0.2), nn.Linear(256, 512), nn.LeakyReLU(0.2), nn.Linear(512, 1024), nn.LeakyReLU(0.2), nn.Linear(1024, img_dim), nn.Tanh() # Output in [-1, 1] ) def forward(self, z): return self.model(z) # Discriminator Network class Discriminator(nn.Module): def __init__(self, img_dim): super().__init__() self.model = nn.Sequential( nn.Linear(img_dim, 1024), nn.LeakyReLU(0.2), nn.Dropout(0.3), nn.Linear(1024, 512), nn.LeakyReLU(0.2), nn.Dropout(0.3), nn.Linear(512, 256), nn.LeakyReLU(0.2), nn.Dropout(0.3), nn.Linear(256, 1), nn.Sigmoid() # Output probability ) def forward(self, img): return self.model(img) # Training Loop def train_gan(generator, discriminator, dataloader, epochs, lr, latent_dim): opt_g = torch.optim.Adam(generator.parameters(), lr=lr, betas=(0.5, 0.999)) opt_d = torch.optim.Adam(discriminator.parameters(), lr=lr, betas=(0.5, 0.999)) criterion = nn.BCELoss() for epoch in range(epochs): for real_imgs in dataloader: batch_size = real_imgs.size(0) real_imgs = real_imgs.view(batch_size, -1) # Train Discriminator z = torch.randn(batch_size, latent_dim) fake_imgs = generator(z).detach() real_loss = criterion(discriminator(real_imgs), torch.ones(batch_size, 1)) fake_loss = criterion(discriminator(fake_imgs), torch.zeros(batch_size, 1)) d_loss = (real_loss + fake_loss) / 2 opt_d.zero_grad() d_loss.backward() opt_d.step() # Train Generator z = torch.randn(batch_size, latent_dim) fake_imgs = generator(z) g_loss = criterion(discriminator(fake_imgs), torch.ones(batch_size, 1)) opt_g.zero_grad() g_loss.backward() opt_g.step()
