GraphSAGE

Inductive Representation Learning on Large Graphs
William L. Hamilton, Rex Ying, Jure Leskovec · Stanford University
NIPS 2017 · Stanford SNAP Lab
📄 arXiv 💻 Code 🔗 Website

🎯 Problem Statement

Graph representation learning is crucial for tasks from content recommendation to protein function identification. However, most existing approaches (like DeepWalk and GCN) are transductive: they require all nodes to be present during training and cannot naturally generalize to unseen nodes.

The Transductive Limitation

Existing methods train individual embeddings for each node. When new nodes arrive or graphs evolve, these methods require complete retraining, making them impractical for high-throughput production systems dealing with constantly evolving graphs.

Real-world applications like Reddit posts, YouTube videos, or protein interaction networks require systems that can quickly generate embeddings for unseen nodes without retraining. An inductive approach is essential.

💡 Key Innovation

Instead of training individual embeddings for each node, GraphSAGE learns a function that generates embeddings by sampling and aggregating features from a node's local neighborhood.

The Core Idea: Learnable Aggregation Functions

GraphSAGE learns a set of aggregator functions that combine information from a node's neighbors at each depth. This approach naturally generalizes to unseen nodes because the same aggregation parameters are applied regardless of node identity.

hvk = σ(Wk · AGGREGATEk({huk-1, ∀u ∈ N(v)}) + Bk · hvk-1)

🏗️ Architecture & Method

v Aggregation Functions: Mean Aggregator hvk ← mean({huk-1}) Element-wise mean of neighbor features LSTM Aggregator LSTM(permute({hu})) Apply LSTM to random permutation Pooling Aggregator max({σ(W·hu+b)}) Element-wise max of transformed features

GraphSAGE Algorithm Steps:

  • 1. Sample: For each node, sample a fixed-size set of neighbors (e.g., uniformly or based on importance) to maintain computational efficiency regardless of node degree.
  • 2. Aggregate: Apply an aggregation function to combine information from the sampled neighbors. Three options: Mean, LSTM, or Pooling aggregator.
  • 3. Update: Concatenate the aggregated neighbor representation with the node's current representation, then apply a nonlinear transformation.
  • 4. Iterate: Repeat for K layers, where each layer incorporates information from nodes K hops away.

Inductive vs Transductive:

Transductive Learning: All data must be present during training. Learning happens on a fixed graph. New nodes require complete retraining (DeepWalk, GCN).

Inductive Learning: Can generalize to unseen data without retraining. GraphSAGE learns a function that works on any node with feature attributes, enabling predictions on evolving graphs or entirely new graphs.

💻 Code Implementation

# GraphSAGE: Mean Aggregator in PyTorch
import torch
import torch.nn as nn
import torch.nn.functional as F

class SAGEAggregator(nn.Module):
    def __init__(self, input_dim, output_dim):
        super().__init__()
        self.W = nn.Linear(input_dim * 2, output_dim)  # Concatenate self + neighbor
    
    def forward(self, node_features, neighbor_features):
        # neighbor_features: [batch_size, num_samples, feature_dim]
        neighbor_mean = torch.mean(neighbor_features, dim=1)  # Mean aggregation
        
        # Concatenate node's own features with aggregated neighbor features
        combined = torch.cat([node_features, neighbor_mean], dim=1)
        
        # Apply nonlinear transformation
        output = F.relu(self.W(combined))
        return output

class GraphSAGE(nn.Module):
    def __init__(self, input_dim, hidden_dim, num_classes, num_layers=2):
        super().__init__()
        self.num_layers = num_layers
        self.aggregators = nn.ModuleList([
            SAGEAggregator(input_dim if i == 0 else hidden_dim, hidden_dim)
            for i in range(num_layers)
        ])
        self.classifier = nn.Linear(hidden_dim, num_classes)
    
    def forward(self, x, adj_list, num_samples=10):
        # x: node features [num_nodes, feature_dim]
        # adj_list: neighbor indices for each node
        
        for layer in range(self.num_layers):
            new_features = []
            for v in range(len(x)):
                # Sample fixed number of neighbors
                neighbors = adj_list[v]
                if len(neighbors) > num_samples:
                    sampled = torch.randperm(len(neighbors))[:num_samples]
                    neighbors = [neighbors[i] for i in sampled]
                
                # Aggregate neighbor features
                neighbor_feats = x[neighbors]  # [num_samples, feature_dim]
                if len(neighbors) == 0:
                    neighbor_feats = torch.zeros_like(x[v:v+1])
                
                # Apply aggregator
                new_feat = self.aggregators[layer](
                    x[v:v+1], neighbor_feats.unsqueeze(0)
                )
                new_features.append(new_feat.squeeze(0))
            
            x = torch.stack(new_features)
            x = F.normalize(x, p=2, dim=1)  # L2 normalization (important!)
        
        return self.classifier(x)

🎯 Impact & Legacy

  • Foundation for Scalable GNNs: GraphSAGE's sampling approach enables GNNs to handle billion-scale graphs, inspiring systems like PinSage (Pinterest) and Uber's ETA predictions.
  • Production Deployments: Widely adopted in industry for recommendation systems, fraud detection, and knowledge graph reasoning at scale.
  • Predecessor to GraphSAINT: GraphSAGE's sampling philosophy led to full-graph sampling methods that enable mini-batch training on massive graphs.
  • Inspiration for Temporal GNNs: The inductive framework naturally extends to dynamic graphs, influencing methods for evolving network analysis.
  • Bridge to GAT: While GraphSAGE treats all neighbors equally, it paved the way for attention-based aggregation in GAT.

🔑 Key Takeaways

What Made GraphSAGE Revolutionary

Inductive Capability: Learn functions, not embeddings — enabling predictions on unseen nodes without retraining.

Scalability: Fixed-size neighborhood sampling makes computation predictable regardless of graph size or node degree.

Feature Integration: Combines graph structure with rich node attributes (text, images, properties) for more powerful representations.

Flexibility: Multiple aggregation options (Mean, LSTM, Pooling) adapt to different graph types and task requirements.

GraphSAGE represents a paradigm shift in graph neural networks: from memorizing embeddings to learning generalizable functions. This inductive approach is essential for real-world applications where graphs constantly evolve with new users, products, and interactions.
