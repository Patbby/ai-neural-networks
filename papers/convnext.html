ConvNeXt: A ConvNet for the 2020s

The "Roaring 20s" of visual recognition began with Vision Transformers. But what if pure ConvNets could match their performance? ConvNeXt modernizes ResNet with Transformer-inspired designs to create a pure ConvNet that rivals Swin Transformer.

📅 CVPR 2022 👥 Liu, Mao, Wu, Feichtenhofer, Darrell, Xie 🏢 Facebook AI Research (FAIR) 📊 87.8% ImageNet Top-1
📄 Read Paper 💻 GitHub Code 🔍 Architecture

The Central Question

Can pure ConvNets compete with Vision Transformers in the 2020s?

🤔 The Challenge

By 2020, Vision Transformers (especially hierarchical ones like Swin Transformer) had overtaken ConvNets as the preferred backbone for computer vision. The prevailing belief was that Transformers were inherently superior due to their global self-attention mechanism.


But was this superiority due to the Transformer architecture itself, or could it be attributed to:

  • Better training procedures (300 epochs, advanced augmentations)
  • Macro design choices (stage ratios, patchify stems)
  • Architecture improvements (inverted bottlenecks, depthwise conv)

💡 The ConvNeXt Solution

Starting from a standard ResNet-50, the authors systematically "modernized" it step-by-step toward a Transformer-like design, discovering which components actually matter for performance.


The Result: A pure ConvNet built entirely from standard ConvNet modules that:

  • Matches Swin Transformer accuracy (87.8% top-1 on ImageNet)
  • Outperforms Swin on COCO detection and ADE20K segmentation
  • Maintains the simplicity and efficiency of standard ConvNets

Modernization Journey

Step-by-step transformation from ResNet to ConvNeXt

🏗️ From ResNet-50 to ConvNeXt: The Evolution Path

🔬 The 6 Key Modernization Steps

1. Training Techniques (76.1% → 78.8%)

Before changing architecture, the authors updated the training procedure to match modern practices: 300 epochs (vs 90), AdamW optimizer, Mixup, Cutmix, RandAugment, Random Erasing, Stochastic Depth, and Label Smoothing. This alone boosted ResNet-50 by 2.7%.

2. Macro Design (78.8% → 79.5%)

Changed the stage ratio from ResNet's (3,4,6,3) to Swin-T's (3,3,9,3) and replaced the 7×7 stride-2 stem with a 4×4 stride-4 "patchify" layer—similar to how ViT processes images as patches.

3. ResNeXt-ify (79.5% → 80.5%)

Adopted depthwise convolutions (grouped conv with groups = channels) and increased network width from 64 to 96 channels. This reduces FLOPs while maintaining representational capacity.

4. Inverted Bottleneck (80.5% → 80.6%)

Flipped the bottleneck structure: instead of (1×1 reduce → 3×3 conv → 1×1 expand), used (1×1 expand → 3×3 DW conv → 1×1 reduce), similar to MobileNetV2 and Transformer MLP blocks.

5. Large Kernel Sizes (maintained 80.6%)

Increased the depthwise conv kernel from 3×3 to 7×7—matching Swin Transformer's attention window size. The depthwise conv's position was moved up in the block to enable this without increasing FLOPs significantly.

6. Micro Design (80.6% → 82.0%)

Multiple small changes: ReLU → GELU activation, BatchNorm → LayerNorm, fewer activation/normalization layers (only one GELU and one LayerNorm per block), and separate downsampling layers between stages.

ConvNeXt Block Architecture

The fundamental building block: depthwise convolution + inverted MLP

🔧 ConvNeXt Block Structure

🧮 Mathematical Formulation

A ConvNeXt block can be expressed as:

# Input: x with shape [B, H, W, C] # Step 1: First LayerNorm y = LayerNorm(x) # [B, H, W, C] # Step 2: Depthwise Convolution (7×7) y = DWConv7x7(y) # [B, H, W, C] # Step 3: Second LayerNorm y = LayerNorm(y) # [B, H, W, C] # Step 4: Inverted MLP (expansion ratio = 4) y = Linear(y, out_features=4*C) # [B, H, W, 4C] y = GELU(y) # [B, H, W, 4C] y = Linear(y, out_features=C) # [B, H, W, C] # Step 5: Residual Connection output = x + y # [B, H, W, C]

⚡ Key Design Principles

  • 1. Depthwise Separation — The 7×7 depthwise conv mixes spatial information within each channel separately, then 1×1 convs mix across channels. This is more efficient than standard convolutions.
  • 2. Inverted Bottleneck — Following MobileNetV2 and Transformers, the hidden dimension is 4× wider than input/output, creating an "inverted" bottleneck.
  • 3. Minimalist Activation/Norm — Only one GELU and two LayerNorms per block—fewer than ResNet's ReLU-BN-ReLU-BN pattern.
  • 4. Large Receptive Field — The 7×7 kernel provides a large receptive field similar to Swin's 7×7 attention window, but more efficiently.

PyTorch Implementation

Clean, efficient implementation of ConvNeXt

🐍 ConvNeXt Block in PyTorch

import torch import torch.nn as nn class ConvNeXtBlock(nn.Module): """ConvNeXt Block: depthwise conv + inverted MLP with residual""" def __init__(self, dim, drop_path=0.): super().__init__() # Depthwise convolution: 7×7 kernel, groups=dim (depthwise) self.dwconv = nn.Conv2d(dim, dim, kernel_size=7, padding=3, groups=dim) # Layer norms (channels-last format) self.norm = nn.LayerNorm(dim, eps=1e-6) # Inverted MLP: expand 4× then project back self.pwconv1 = nn.Linear(dim, 4 * dim) # Pointwise 1×1 conv self.act = nn.GELU() self.pwconv2 = nn.Linear(4 * dim, dim) # Project back # Stochastic depth for regularization self.drop_path = DropPath(drop_path) if drop_path > 0. else nn.Identity() def forward(self, x): # Save input for residual input_tensor = x # Depthwise conv x = self.dwconv(x) # Channels-last for LayerNorm and MLP x = x.permute(0, 2, 3, 1) # [B, C, H, W] → [B, H, W, C] x = self.norm(x) # Inverted MLP x = self.pwconv1(x) x = self.act(x) x = self.pwconv2(x) # Back to channels-first x = x.permute(0, 3, 1, 2) # [B, H, W, C] → [B, C, H, W] # Residual with stochastic depth x = input_tensor + self.drop_path(x) return x

🔧 Complete ConvNeXt Model

class ConvNeXt(nn.Module): """ConvNeXt model architecture""" def __init__(self, in_chans=3, num_classes=1000, depths=[3, 3, 9, 3], dims=[96, 192, 384, 768], drop_path_rate=0.): super().__init__() # Stem: patchify layer (4×4 conv, stride 4) self.downsample_layers = nn.ModuleList() stem = nn.Sequential( nn.Conv2d(in_chans, dims[0], kernel_size=4, stride=4), nn.LayerNorm(dims[0], eps=1e-6) ) self.downsample_layers.append(stem) # Downsampling between stages for i in range(3): downsample = nn.Sequential( nn.LayerNorm(dims[i], eps=1e-6), nn.Conv2d(dims[i], dims[i+1], kernel_size=2, stride=2) ) self.downsample_layers.append(downsample) # Stochastic depth decay rule dp_rates = [x.item() for x in torch.linspace(0, drop_path_rate, sum(depths))] # Stages with ConvNeXt blocks self.stages = nn.ModuleList() cur = 0 for i in range(4): stage = nn.Sequential( *[ConvNeXtBlock(dim=dims[i], drop_path=dp_rates[cur + j]) for j in range(depths[i])] ) self.stages.append(stage) cur += depths[i] # Classification head self.norm = nn.LayerNorm(dims[-1], eps=1e-6) self.head = nn.Linear(dims[-1], num_classes) # Initialize weights self.apply(self._init_weights) def _init_weights(self, m): if isinstance(m, nn.Linear): nn.init.trunc_normal_(m.weight, std=.02) if m.bias is not None: nn.init.constant_(m.bias, 0) elif isinstance(m, nn.LayerNorm): nn.init.constant_(m.bias, 0) nn.init.constant_(m.weight, 1.0) def forward(self, x): # Stem + stages for i in range(4): x = self.downsample_layers[i](x) x = self.stages[i](x) # Global average pooling + classifier x = self.norm(x.mean([-2, -1])) # [B, C, H, W] → [B, C] x = self.head(x) return x # Model variantsdef convnext_tiny(num_classes=1000): return ConvNeXt(depths=[3, 3, 9, 3], dims=[96, 192, 384, 768], drop_path_rate=0.1) def convnext_small(num_classes=1000): return ConvNeXt(depths=[3, 3, 27, 3], dims=[96, 192, 384, 768], drop_path_rate=0.4) def convnext_base(num_classes=1000): return ConvNeXt(depths=[3, 3, 27, 3], dims=[128, 256, 512, 1024], drop_path_rate=0.5)

Impact & Significance

Why ConvNeXt matters for the future of computer vision

🎯 Key Contributions

  • ✓ Democratized ViT Performance — Showed that pure ConvNets can achieve Transformer-level accuracy without massive datasets like JFT-300M.
  • ✓ Systematic Modernization — Provided a recipe for upgrading any ConvNet with proven techniques from the Transformer era.
  • ✓ Simpler is Better — ConvNeXt achieves better results than Swin without complex modules like shifted windows or relative position biases.
  • ✓ Better Efficiency — ConvNeXt has higher throughput than equivalent Swin models due to the efficiency of optimized convolution implementations.

📈 Citations & Adoption

Since its publication at CVPR 2022, ConvNeXt has been:

  • Cited 5,500+ times as of 2025
  • Widely adopted in production systems
  • Integrated into major frameworks (PyTorch, Hugging Face)
  • Extended to video understanding (Video Swin, MViT)
  • Influenced the design of ConvNeXt V2 with masked autoencoders

The paper won the CVPR 2022 Outstanding Paper Award, recognizing its impact on challenging the Transformer dominance narrative.

🔮 Why ConvNeXt Won't Be Forgotten

ConvNeXt represents a pivotal moment in computer vision: the realization that architectural innovations can flow both ways. Just as Transformers borrowed from ConvNets (hierarchical structures, local attention), ConvNets can borrow from Transformers (training recipes, macro designs, inverted bottlenecks).

The paper's systematic approach—starting from ResNet and making one change at a time—provides a template for future architecture research. It showed that often, the difference between SOTA methods is not fundamental architectural superiority but a collection of well-tuned implementation details.

In an era where model sizes are exploding, ConvNeXt stands as proof that elegant, simple designs can still compete with the most complex systems.
