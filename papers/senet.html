Squeeze-and-Excitation Networks

Jie Hu, Li Shen, Samuel Albanie, Gang Sun, Enhua Wu

CVPR 2018 / TPAMI 2020 • Momenta & University of Oxford

📄 arXiv 🌐 TPAMI 💻 Code

🎯 Problem Statement

Before SENet, convolutional neural networks treated all feature channels equally, missing a critical insight:

  • Not all feature channels contribute equally to the final prediction
  • Some channels capture essential features (edges, textures), others capture background noise
  • Standard convolutions lack explicit channel-wise relationship modeling
  • How can we adaptively recalibrate channel-wise feature responses?

The key question: How can we explicitly model interdependencies between channels to enhance representation power?

💡 Key Innovation

Squeeze-and-Excitation (SE) Block

SENet introduces a novel architectural unit that adaptively recalibrates channel-wise feature responses by explicitly modeling interdependencies between channels. It performs "feature recalibration" through a two-step process: squeezing global spatial information into a channel descriptor, then exciting informative features through learned per-channel weights.

SE Block Transformation:

x̃c = sc · uc

where s = Fex(Fsq(u), W) — the learned channel attention weights

🏗️ Architecture

🔵 Squeeze

ℝ^(C×H×W) → ℝ^C

Global Average Pooling aggregates spatial information into a channel descriptor

→

🟠 Excitation

ℝ^C → ℝ^C

Two-layer MLP with bottleneck captures channel dependencies and produces per-channel weights

→

🟢 Scale

ℝ^C × ℝ^(C×H×W) → ℝ^(C×H×W)

Channel-wise multiplication recalibrates feature maps by learned importance

🔬 Method Details

1. Squeeze Operation

Global Average Pooling (GAP) squeezes each channel's spatial dimensions H×W into a single value, creating a channel descriptor z ∈ ℝ^C that captures global information.

2. Excitation Operation

A self-gating mechanism with two FC layers and a ReLU bottleneck learns nonlinear interactions between channels. The sigmoid produces weights in [0, 1].

3. Reduction Ratio (r)

The bottleneck dimension is C/r, where r is typically 16. This limits model capacity increase while maintaining performance gains.

4. Integration Strategy

SE blocks can be inserted after any transformation (conv, residual block). The paper explores different positions: after conv, after residual, and inside residual branches.

💻 PyTorch Implementation

# Squeeze-and-Excitation Block in PyTorch
import torch
import torch.nn as nn

class SEBlock(nn.Module):
    def __init__(self, channels, reduction=16):
        super(SEBlock, self).__init__()
        # Squeeze: Global Average Pooling
        self.squeeze = nn.AdaptiveAvgPool2d(1)
        
        # Excitation: Two FC layers with bottleneck
        self.excitation = nn.Sequential(
            nn.Linear(channels, channels // reduction, bias=False),
            nn.ReLU(inplace=True),
            nn.Linear(channels // reduction, channels, bias=False),
            nn.Sigmoid()
        )
    
    def forward(self, x):
        b, c, _, _ = x.size()
        
        # Squeeze: (B, C, H, W) → (B, C, 1, 1) → (B, C)
        y = self.squeeze(x).view(b, c)
        
        # Excitation: (B, C) → (B, C)
        y = self.excitation(y).view(b, c, 1, 1)
        
        # Scale: channel-wise multiplication
        return x * y.expand_as(x)

# SE-ResNet Block (SE block integrated with ResNet)
class SEBasicBlock(nn.Module):
    def __init__(self, inplanes, planes, reduction=16):
        super(SEBasicBlock, self).__init__()
        self.conv1 = nn.Conv2d(inplanes, planes, 3, padding=1, bias=False)
        self.bn1 = nn.BatchNorm2d(planes)
        self.relu = nn.ReLU(inplace=True)
        self.conv2 = nn.Conv2d(planes, planes, 3, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(planes)
        
        # SE block added after the residual transformation
        self.se = SEBlock(planes, reduction)
        
        # Shortcut connection
        self.shortcut = nn.Sequential()
        if inplanes != planes:
            self.shortcut = nn.Sequential(
                nn.Conv2d(inplanes, planes, 1, bias=False),
                nn.BatchNorm2d(planes)
            )
    
    def forward(self, x):
        residual = self.shortcut(x)
        
        out = self.conv1(x)
        out = self.bn1(out)
        out = self.relu(out)
        out = self.conv2(out)
        out = self.bn2(out)
        
        # Apply SE recalibration
        out = self.se(out)
        
        out += residual
        out = self.relu(out)
        return out

📚 Key Insights

Why SE Blocks Work

1. Explicit Channel Modeling: Unlike standard convolutions that implicitly learn channel relationships through weight sharing, SE blocks explicitly model interdependencies, allowing the network to learn which channels matter most.

2. Global Context: By squeezing spatial dimensions, each channel's global information is preserved in the descriptor, enabling more informed recalibration decisions.

3. Adaptive Recalibration: The excitation mechanism acts as a self-gating function, adaptively emphasizing informative features and suppressing less useful ones on a per-sample basis.

4. Minimal Overhead: The reduction ratio r (typically 16) keeps parameter count low — SE-ResNet-50 adds only ~10% parameters but gains 1.5% top-1 accuracy.
