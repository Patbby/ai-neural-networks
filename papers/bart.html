BART: Denoising Sequence-to-Sequence Pre-training

A denoising autoencoder that combines bidirectional and auto-regressive transformers, unifying BERT's bidirectional encoding with GPT's autoregressive generation for versatile natural language understanding and generation tasks.

Mike Lewis, Yinhan Liu, Naman Goyal, Marjan Ghazvininejad, Abdelrahman Mohamed, Omer Levy, Ves Stoyanov, Luke Zettlemoyer
Facebook AI
October 2019
arXiv:1910.13461
Paper (PDF) Code Hugging Face

Problem Statement

Before BART, pre-trained language models were typically designed for either understanding tasks (like BERT) or generation tasks (like GPT), but not both effectively. This created a fundamental divide in NLP: models optimized for discriminative tasks struggled with generation, while autoregressive models lacked bidirectional context for comprehension.

🔄 The BERT-GPT Gap

BERT uses bidirectional encoding but cannot generate text autoregressively. GPT generates text well but only uses left context, missing future information crucial for understanding.

🎯 Task-Specific Limitations

Models were typically optimized for specific task types (span prediction vs. generation), limiting their applicability across the full spectrum of NLP tasks.

⚡ Training Inefficiency

Different pre-training objectives required different architectures, leading to duplicated efforts and inability to leverage shared representations across tasks.

Key Innovation

💡 The BART Philosophy

BART combines a bidirectional encoder (like BERT) with an autoregressive decoder (like GPT) in a single sequence-to-sequence architecture. It uses a denoising autoencoder objective: corrupt text with arbitrary noising functions, then learn to reconstruct the original. This flexibility allows BART to excel at both comprehension and generation tasks.

The key insight is that by corrupting documents with various noising strategies and training the model to reconstruct them, BART learns robust representations that transfer well to diverse downstream tasks. The architecture can be seen as generalizing both BERT and GPT, while the pre-training approach unifies many previous objectives.

Architecture

BART uses the standard Transformer sequence-to-sequence architecture with some modifications from GPT: GeLU activations instead of ReLU, and parameter initialization from N(0, 0.02). The base model has 6 layers in both encoder and decoder (139M parameters), while the large model has 12 layers each (406M parameters).

BART Architecture: Bridging BERT and GPT

The encoder processes corrupted text bidirectionally, allowing each token to attend to all others. The decoder generates the original text autoregressively, with each layer performing cross-attention over the encoder's final hidden states. This design enables BART to leverage the best of both worlds: bidirectional context understanding and fluent text generation.

Noising Strategies

BART's flexibility comes from its ability to use arbitrary noising functions. The paper evaluates multiple strategies, finding that a combination of text infilling and sentence shuffling works best. This approach generalizes BERT's masking and next sentence prediction objectives.

Results & Impact

BART achieves competitive or state-of-the-art results across a wide range of tasks. It matches RoBERTa's performance on discriminative tasks (GLUE, SQuAD) while significantly outperforming previous work on generation tasks.

Key findings from ablation studies: (1) Token masking is crucial - document rotation and sentence permutation alone perform poorly. (2) Left-to-right pre-training consistently improves generation tasks. (3) Bidirectional encoders are essential for SQuAD performance. (4) BART achieves the most consistently strong performance across all tasks except ELI5, where pure language models excel due to loosely constrained outputs.

Code Implementation

Here's a PyTorch implementation using the Hugging Face Transformers library, demonstrating how to use BART for summarization:

🐍 PyTorch - BART Summarization 
from transformers import BartTokenizer, BartForConditionalGeneration

# Load pre-trained BART model and tokenizer
model_name = "facebook/bart-large-cnn"
tokenizer = BartTokenizer.from_pretrained(model_name)
model = BartForConditionalGeneration.from_pretrained(model_name)

# Input text for summarization
article = """
BART is a denoising autoencoder built with a sequence-to-sequence model 
that is applicable to a very wide range of end tasks. Pretraining has 
two stages: (1) text is corrupted with an arbitrary noising function, 
and (2) a sequence-to-sequence model is learned to reconstruct the 
original text. BART uses a standard Transformer-based neural machine 
translation architecture which generalizes BERT and GPT.
"""

# Tokenize input
inputs = tokenizer([article], max_length=1024, 
                   return_tensors="pt", truncation=True)

# Generate summary
summary_ids = model.generate(
    inputs["input_ids"],
    num_beams=4,
    max_length=128,
    min_length=30,
    length_penalty=2.0,
    early_stopping=True
)

# Decode and print summary
summary = tokenizer.decode(summary_ids[0], 
                          skip_special_tokens=True)
print(summary)
🐍 Fine-tuning BART for Custom Tasks 
from transformers import (
    BartTokenizer, 
    BartForSequenceClassification,
    TrainingArguments, 
    Trainer
)
from datasets import load_dataset

# For classification tasks, use BartForSequenceClassification
model = BartForSequenceClassification.from_pretrained(
    "facebook/bart-base",
    num_labels=2
)

# Training configuration
training_args = TrainingArguments(
    output_dir="./bart-finetuned",
    num_train_epochs=3,
    per_device_train_batch_size=8,
    per_device_eval_batch_size=8,
    warmup_steps=500,
    weight_decay=0.01,
    logging_dir="./logs",
    learning_rate=2e-5,
    fp16=True,  # Mixed precision training
)

# Initialize Trainer and train
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
    eval_dataset=eval_dataset,
)

trainer.train()
