Neural Turing Machines

Neural Networks with External Memory

📄 arXiv:1410.5401 📅 2014 🏢 Google DeepMind 👤 Graves, Wayne, Danihelka
arXiv PDF PyTorch Code

🎯 Problem Statement

The Memory Limitation: Traditional neural networks (RNNs, LSTMs) store information in their fixed-weight connections, limiting their ability to:
  • Store and retrieve large amounts of data efficiently
  • Perform algorithmic tasks requiring external storage
  • Generalize to longer sequences than seen during training
  • Learn complex algorithms like sorting, copying, or associative recall

Neural Turing Machines address this by coupling a neural network controller with an external memory matrix, similar to how a Turing machine uses a tape. The controller learns to read from and write to this memory through attention mechanisms.

💡 Key Innovation

Differentiable External Memory: NTM introduces a fully differentiable external memory that can be accessed via soft attention, enabling end-to-end gradient-based learning of algorithms that require storage and manipulation of data structures.
Core Architecture:
  • Controller: A neural network (LSTM or feedforward) that receives inputs and produces outputs
  • Memory Matrix: An N × M matrix storing information (N locations, M features each)
  • Read/Write Heads: Specialized modules that access memory via attention
  • Addressing Mechanism: Content-based and location-based memory access
Memory at time t: Mt ∈ ℝN×M
Read vector: rt = Σi wt(i) · Mt(i)
where wt is the attention weight vector

🔍 Addressing Mechanisms

NTM uses a combination of content-based and location-based addressing to access memory:

Content-Based Addressing Flow
t Similarity cosine similarity K[kₜ, M(i)] Memory M(i) Softmax wₜ(i) ∝ exp(β·sim) Attention weights wₜ
Addressing Pipeline:
  1. Content-based focusing: Compare key vector with each memory location using cosine similarity
  2. Location-based focusing: Shift attention to nearby locations (circular convolution)
  3. Sharpening: Adjust focus sharpness with a parameter γ to make attention more or less selective
Content similarity: K[u,v] = (u·v) / (||u||·||v||)
Content weights: wtc(i) = exp(βt · K[kt, Mt(i)]) / Σj exp(βt · K[kt, Mt(j)])

✏️ Read & Write Operations

Reading from Memory:
The read head produces a read vector rt by taking a weighted average of memory locations.
Read: rt = Σi=1N wtr(i) · Mt(i)
Writing to Memory:
The write head performs two operations: erase (selective clearing) and add (selective writing).
Erase: M̃t(i) = Mt-1(i) · [1 - wtw(i) · et]
Add: Mt(i) = M̃t(i) + wtw(i) · at

Where et is the erase vector (0 to 1 values) and at is the add vector. The write weighting wtw determines which locations are modified.

💻 PyTorch Implementation

import torch import torch.nn as nn import torch.nn.functional as F class NeuralTuringMachine(nn.Module): def __init__(self, input_size, output_size, controller_size, memory_size, memory_feature_size): super().__init__() self.controller_size = controller_size self.memory_size = memory_size self.memory_feature_size = memory_feature_size # Controller: LSTM self.controller = nn.LSTMCell(input_size + memory_feature_size, controller_size) # Memory: N × M matrix self.register_buffer('memory', torch.zeros(memory_size, memory_feature_size)) # Output layers for read/write heads self.read_head = nn.Linear(controller_size, memory_feature_size) self.write_head = nn.Linear(controller_size, memory_feature_size * 3) # key, add, erase self.output_layer = nn.Linear(controller_size + memory_feature_size, output_size) def content_addressing(self, key, beta, memory): # Cosine similarity key_norm = F.normalize(key, dim=-1) mem_norm = F.normalize(memory, dim=-1) similarity = torch.matmul(mem_norm, key_norm.unsqueeze(-1)).squeeze(-1) # Softmax with sharpness weights = F.softmax(beta * similarity, dim=-1) return weights def read(self, weights, memory): return torch.matmul(weights.unsqueeze(0), memory).squeeze(0) def write(self, weights, erase, add, memory): # Erase then add erase_term = memory * (1 - weights.unsqueeze(-1) * erase) add_term = weights.unsqueeze(-1) * add return erase_term + add_term

🎓 Key Insights

Why NTM Matters:
  1. Blurred line between data and program: NTM stores both in the same memory, like a von Neumann architecture
  2. Differentiable computing: Shows that algorithmic operations can be learned via gradient descent
  3. Attention as addressing: Demonstrates that soft attention mechanisms can emulate memory addressing
  4. Foundation for Transformers: The attention mechanisms in NTM influenced the Transformer architecture
  5. Generalization: Learns underlying algorithms rather than memorizing specific examples

NTM represents a pivotal step toward neural networks that can manipulate data structures, execute algorithms, and maintain long-term state—capabilities essential for artificial general intelligence.
