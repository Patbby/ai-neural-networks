Problem Statement

Before SVM (1995), classification faced fundamental challenges:

  • The Generalization Problem: Neural networks and decision trees often overfit training data, failing to generalize to unseen examples
  • Computational Complexity: Many algorithms struggled with high-dimensional data (the "curse of dimensionality")
  • No Theoretical Guarantees: Most methods lacked rigorous mathematical foundations for expected performance
  • Local Minima Issues: Neural networks could get stuck in poor local minima during training

Vapnik's question: "Can we build a classifier with strong theoretical guarantees on generalization performance, even in high-dimensional spaces?"

Key Innovation

SVM introduced a revolutionary approach: Maximize the Margin

Instead of just finding any separating boundary, SVM finds the optimal hyperplane that maximizes the distance to the nearest data points from each class. These closest points are called support vectors.

Structural Risk Minimization Kernel Trick Maximum Margin Quadratic Programming

Core Insight: By maximizing the margin, SVMs achieve better generalization because they leave more "buffer room" for unseen data. The decision boundary is determined by only a few critical points (support vectors), not all training examples.

SVM Architecture

Linear SVM: Maximum Margin Hyperplane
SVM finds the hyperplane that maximizes the margin between classes. The support vectors (highlighted in yellow) are the only points that define the decision boundary—all other training examples can be removed without changing the classifier.

The Kernel Trick: Non-linear Classification

For data that isn't linearly separable, SVM uses the kernel trick to implicitly map data to a higher-dimensional space where it becomes separable—without explicitly computing the transformation.

Kernel Trick Visualization
The kernel trick allows SVM to operate in high-dimensional feature spaces without explicitly computing the coordinates. Instead, it computes the dot product in that space directly using a kernel function.

Popular Kernel Functions

Linear Kernel: K(x, y) = x · y
Polynomial Kernel: K(x, y) = (x · y + c)ᵈ
RBF (Gaussian) Kernel: K(x, y) = exp(-γ||x - y||²)
Sigmoid Kernel: K(x, y) = tanh(α(x · y) + c)

Mathematical Foundation

Optimization Objective

minw,b ½||w||²

subject to: yᵢ(w · xᵢ + b) ≥ 1, ∀i

The optimization problem minimizes ||w||² (which maximizes the margin 2/||w||) while ensuring all training examples are correctly classified with a functional margin of at least 1.

Dual Formulation with Kernels

maxα Σᵢ αᵢ - ½ Σᵢ Σⱼ αᵢ αⱼ yᵢ yⱼ K(xᵢ, xⱼ)

subject to: Σᵢ αᵢ yᵢ = 0, αᵢ ≥ 0

The dual formulation reveals why only support vectors matter: αᵢ = 0 for all non-support vectors. The decision function becomes:

f(x) = sign(Σᵢ αᵢ yᵢ K(xᵢ, x) + b)

Code Implementation

svm_classifier.py 
import numpy as np
from sklearn import svm, datasets
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import classification_report

# Load dataset (e.g., Iris)
iris = datasets.load_iris()
X = iris.data[:, :2]  # Use only first 2 features for visualization
y = iris.target

# Make it binary classification (setosa vs rest)
y = (y == 0).astype(int) * 2 - 1  # Convert to +1/-1 labels

# Split and scale data
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3)
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test = scaler.transform(X_test)

# Create SVM with RBF kernel
clf = svm.SVC(
    kernel='rbf',      # Radial Basis Function (Gaussian) kernel
    C=1.0,             # Regularization parameter
    gamma='scale'    # Kernel coefficient
)

# Train the model
clf.fit(X_train, y_train)

# Predictions
y_pred = clf.predict(X_test)

# Support vectors
print(f"Support vectors: {len(clf.support_)} / {len(X_train)}")
print(f"Training accuracy: {clf.score(X_train, y_train):.3f}")
print(f"Test accuracy: {clf.score(X_test, y_test):.3f}")

Key Hyperparameters

  • C (Regularization): Trade-off between margin width and classification error. Smaller C = wider margin, more misclassifications allowed. Larger C = stricter classification, risk of overfitting.
  • γ (gamma): Kernel coefficient for RBF. Controls influence of individual training samples. High γ = close points have high influence (overfitting risk). Low γ = far points have influence.
  • kernel: 'linear', 'rbf', 'poly', 'sigmoid', or custom kernel function.
