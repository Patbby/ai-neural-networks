Improved Training of Wasserstein GANs

📅 2017 📄 NeurIPS 👥 Ishaan Gulrajani, Faruk Ahmed, Martin Arjovsky, Vincent Dumoulin, Aaron Courville 📊 4,500+ Citations
📄 arXiv 📰 NeurIPS 💻 Code

The Problem

🚨 WGAN's Weight Clipping Pathologies

While Wasserstein GAN made significant progress toward stable GAN training, the use of weight clipping to enforce the Lipschitz constraint can lead to undesired behaviors: (1) capacity underuse as the critic ignores higher-level features, (2) gradient vanishing or exploding with deep networks, and (3) generation of low-quality samples despite training stability.

The original WGAN enforces the 1-Lipschitz constraint by clipping weights to a fixed box [−c, c]. This crude approximation creates optimization difficulties—the critic becomes a k-Lipschitz function for some unknown k, rather than exactly 1-Lipschitz, and may learn simple, low-capacity functions to satisfy the constraint.

Key Innovation

💡 Gradient Penalty (GP)

Instead of weight clipping, penalize the norm of the critic's gradient with respect to its input. This provides a softer, more effective constraint that enables stable training across diverse architectures with minimal hyperparameter tuning.

🎯 Soft Constraint

Gradient penalty softly enforces 1-Lipschitzness without harsh weight restrictions, allowing the critic to learn more expressive functions.

⚡ Stable Training

Works with 101-layer ResNets, language models, and various GAN architectures without extensive hyperparameter tuning.

📈 Better Quality

Achieves high-quality generations on CIFAR-10 and LSUN bedrooms, outperforming WGAN with weight clipping.

🔧 Easy Implementation

Simple addition to loss function with automatic differentiation—no complex architectural changes needed.

Method

The gradient penalty is computed on interpolated samples between real and generated data. For each training iteration, sample uniformly along straight lines between pairs of points from the real and generated distributions, then penalize deviations from unit gradient norm.

Mathematical Formulation

WGAN-GP Critic Loss

LD = Ex̃~Pg[D(x̃)] - Ex~Pr[D(x)] + λ · Ex̂~Px̂[(‖∇x̂D(x̂)‖2 - 1)²]

Generator Loss (unchanged from WGAN)

LG = -Ex̃~Pg[D(x̃)]

Interpolation Sampling

x̂ = ε · x + (1 - ε) · x̃,    where ε ~ Uniform(0, 1)

Where λ is the gradient penalty coefficient (typically set to 10). The gradient penalty term enforces that the critic's gradients have unit norm along the straight lines between real and generated samples, which is a sufficient condition for the 1-Lipschitz constraint required by the Kantorovich-Rubinstein duality.

Algorithm

# WGAN-GP Training Algorithm
for iteration in range(num_iterations):
    # -------------------
    # Train Critic (n_critic times)
    # -------------------
    for t in range(n_critic):
        # Sample real and fake data
        x = sample_real_data(batch_size)
        z = sample_noise(batch_size)
        x_tilde = generator(z)
        
        # Sample interpolation
        eps = random.uniform(0, 1, batch_size)
        x_hat = eps * x + (1 - eps) * x_tilde
        
        # Compute gradient penalty
        gradients = gradients(critic, x_hat)
        gradient_norm = l2_norm(gradients)
        gradient_penalty = mean((gradient_norm - 1)**2)
        
        # Critic loss with gradient penalty
        critic_loss = mean(critic(x_tilde)) - mean(critic(x))
        critic_loss = critic_loss + lambda_gp * gradient_penalty
        
        # Update critic
        update_critic(-critic_loss)  # Gradient ascent
    
    # -------------------
    # Train Generator
    # -------------------
    z = sample_noise(batch_size)
    x_tilde = generator(z)
    generator_loss = -mean(critic(x_tilde))
    update_generator(generator_loss)

PyTorch Implementation

import torch
import torch.nn as nn
import torch.autograd as autograd

class WGANGP(nn.Module):
    def __init__(self, generator, critic, lambda_gp=10):
        super().__init__()
        self.generator = generator
        self.critic = critic
        self.lambda_gp = lambda_gp
    
    def compute_gradient_penalty(self, real_samples, fake_samples):
        # Random interpolation
        alpha = torch.rand(real_samples.size(0), 1, 1, 1).to(real_samples.device)
        interpolates = alpha * real_samples + (1 - alpha) * fake_samples
        interpolates = autograd.Variable(interpolates, requires_grad=True)
        
        # Critic output on interpolates
        d_interpolates = self.critic(interpolates)
        
        # Compute gradients
        gradients = autograd.grad(
            outputs=d_interpolates,
            inputs=interpolates,
            grad_outputs=torch.ones_like(d_interpolates),
            create_graph=True,
            retain_graph=True,
            only_inputs=True
        )[0]
        
        # Gradient penalty
        gradients = gradients.view(gradients.size(0), -1)
        gradient_norm = gradients.norm(2, dim=1)
        penalty = ((gradient_norm - 1) ** 2).mean()
        
        return penalty
    
    def critic_loss(self, real_samples, fake_samples):
        # Wasserstein distance
        real_validity = self.critic(real_samples)
        fake_validity = self.critic(fake_samples)
        wasserstein_distance = fake_validity.mean() - real_validity.mean()
        
        # Gradient penalty
        gp = self.compute_gradient_penalty(real_samples, fake_samples)
        
        # Total critic loss
        loss = wasserstein_distance + self.lambda_gp * gp
        return loss
    
    def generator_loss(self, fake_samples):
        return -self.critic(fake_samples).mean()

# Optimizers (Adam for both, unlike WGAN's RMSProp)
optimizer_G = torch.optim.Adam(generator.parameters(), lr=1e-4, betas=(0.5, 0.9))
optimizer_D = torch.optim.Adam(critic.parameters(), lr=1e-4, betas=(0.5, 0.9))

Results & Impact

🖼️ CIFAR-10

Generated high-quality images with Inception Score competitive with state-of-the-art GANs at the time. No mode collapse observed.

🏠 LSUN Bedrooms

Successfully trained on LSUN bedroom dataset, generating diverse, realistic room images with stable convergence.

🏗️ 101-Layer ResNets

First GAN variant to stably train 101-layer ResNet discriminators—previously impossible with standard GAN or WGAN.

📝 Language Models

Trained GANs on discrete data (language models) over one-hot representations—a notoriously difficult task for GANs.

The paper demonstrated that WGAN-GP enables stable training "with almost no hyperparameter tuning" across diverse architectures. This robustness made WGAN-GP the go-to choice for GAN training in research and production for years, until StyleGAN and diffusion models emerged.

Key Insights

  • Weight clipping is problematic: It leads to pathological behavior and limits critic capacity
  • Gradient penalty is superior: Provides a principled, soft constraint that works better in practice
  • Interpolation strategy matters: Sampling along straight lines between real and fake is effective
  • Adam works again: Unlike WGAN which required RMSProp, WGAN-GP works well with Adam
  • λ = 10 is standard: The gradient penalty coefficient doesn't require tuning for most cases
