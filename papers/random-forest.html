Random Forests

Leo Breiman • Machine Learning 45(1):5-32 • 2001

The foundational ensemble learning method combining bagging with random feature selection, creating uncorrelated decision trees that voted together for unprecedented accuracy and robustness.

📄 Original Paper 📚 Wikipedia 💻 Scikit-Learn

🎯 Problem Statement

Before Random Forest, decision trees were powerful but problematic: they were high-variance estimators that suffered from overfitting and instability. Small changes in training data could produce dramatically different trees. While bagging (Bootstrap Aggregating) helped by averaging multiple trees, the trees remained correlated, limiting the variance reduction.

Breiman's key insight: to achieve maximum variance reduction, we need many accurate yet uncorrelated trees. The question was how to decorrelate trees while keeping them strong.

💡 Key Innovation

🌲 Bagging

Bootstrap sampling with replacement creates diverse training sets. Each tree sees a different ~63.2% of the data.

For N samples, prob. not selected = (1-1/N)^N ≈ 1/e ≈ 36.8%

🎲 Random Subspaces

At each split, only m features (typically √p) are randomly selected. This dramatically reduces tree correlation.

m_try ≈ √p for classification
m_try ≈ p/3 for regression

📊 Out-of-Bag Error

Each tree's OOB samples (not in its bootstrap) provide built-in cross-validation—no need for separate test sets!

🔍 Variable Importance

Permuting each variable and measuring accuracy drop reveals which features matter most for prediction.

Theoretical Foundation

The generalization error of a Random Forest depends on two factors:

Generalization Error ≤ ρ × (1-s²) / s²

Where:
• ρ = correlation between any two trees (interdependence)
• s = strength (accuracy) of individual trees

Random Forests reduce ρ through random feature selection while maintaining s via deep trees. As more trees are added, the ensemble error converges almost surely (Strong Law of Large Numbers).

💻 Code Implementation

from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

# Load data
iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(
    iris.data, iris.target, test_size=0.3, random_state=42
)

# Random Forest with Breiman's recommendations
rf = RandomForestClassifier(
    n_estimators=100,      # Number of trees (B)
    max_features='sqrt',    # m_try ≈ √p for classification
    bootstrap=True,          # Use bagging
    oob_score=True,          # Out-of-bag error estimate
    max_depth=None,        # Grow deep trees (no pruning)
    random_state=42
)

# Train
rf.fit(X_train, y_train)

# OOB error (Breiman's innovation)
print(f"OOB Score: {rf.oob_score_:.3f}")

# Feature importance
for name, importance in zip(iris.feature_names, rf.feature_importances_):
    print(f"{name}: {importance:.3f}")

# Predict
predictions = rf.predict(X_test)

🎯 Key Takeaways

  • Ensemble Power: Many weak learners can combine into a strong learner through voting.
  • Decorrelation is Key: Random feature selection reduces tree correlation more than bagging alone.
  • No Overfitting: Adding more trees cannot overfit—the ensemble converges to a stable prediction.
  • Self-Validating: OOB error provides unbiased error estimates without cross-validation.
  • Universal Applicability: Works for classification, regression, clustering, and feature selection.
  • Minimal Tuning: Default parameters (m=√p, B=100) work well across most problems.
