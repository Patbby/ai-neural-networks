Capsule Networks:
Dynamic Routing Between Capsules

📅 NIPS 2017 👥 Sara Sabour, Nicholas Frosst, Geoffrey E. Hinton 🏛️ Google Brain

Hinton 推翻 CNN 池化机制的突破性工作。Capsule 用向量输出替代标量激活， 通过 Dynamic Routing by Agreement 机制保留空间层次信息， 解决了 CNN 面对"毕加索问题"时的根本缺陷。

📄 arXiv 📑 PDF 💻 Code 🔥 6,000+ Citations

问题陈述：CNN 的"毕加索问题"

卷积神经网络的根本缺陷：

CNN 通过卷积层提取局部特征，再通过池化（Max Pooling）实现空间不变性。但池化操作会丢弃精确的空间关系和姿态信息，导致 CNN 只能检测特征是否存在，而无法理解特征之间的空间关系。

这就是"毕加索问题"：如果一张人脸图像的眼睛、鼻子、嘴巴都在，但位置错乱（如毕加索的立体派画作），CNN 仍可能将其识别为正常人脸，因为它只检测了部件的存在，而没有验证它们之间的正确空间关系。

核心洞见：Hinton 指出，人类视觉系统使用「坐标框架」来理解物体。当一个部件被识别时，我们不仅知道它是什么，还知道它相对于整体的位置、方向、大小等姿态参数。Capsule 正是为了编码这种「实例化参数」而设计的。

核心创新：Capsule 架构

🎯 什么是 Capsule？

Capsule 是一组神经元，其活动向量的长度表示实体存在的概率，方向表示实例化参数（位置、方向、大小、变形、纹理等）。

  • 低层 Capsule 通过变换矩阵预测高层 Capsule 的姿态
  • 当多个预测达成一致（Agreement）时，高层 Capsule 被激活
  • 使用迭代路由机制（Routing by Agreement）替代池化

CapsNet 架构示意图：输入 → Primary Capsules → Digit Capsules → 输出

Dynamic Routing 算法

vj = ||sj||2 / (1 + ||sj||2) · sj / ||sj||

其中 squash 函数确保短向量被压缩到接近 0，长向量被压缩到接近 1（但不超 1）， 这样既保留了方向信息，又将长度压缩为概率值。

bij ← bij + ûj|i · vj

Routing by Agreement：低层 Capsule i 预测高层 Capsule j 的输出 ûj|i， 如果预测与高层实际输出 vj 一致（内积大），则增加耦合系数 bij。

方法细节

网络结构

  • ReLU Conv1：256 个 9×9 卷积核，步幅 2 → 输出 20×20×256
  • PrimaryCaps：32 个通道的 8D Capsule，每个 Capsule 包含 8 个 9×9 卷积核，步幅 2 → 6×6×32 Capsules
  • DigitCaps：10 个 16D Capsule（对应 MNIST 的 10 个数字）
  • Decoder：3 个全连接层（512→1024→784）用于重建

Margin Loss

为了让 DigitCaps 正确分类，使用基于间隔的损失函数。对每个数字类别 k：

Lk = Tk max(0, m+ - ||vk||)2 + λ(1-Tk) max(0, ||vk|| - m-)2

其中 Tk = 1 当且仅当类别 k 存在；m+ = 0.9, m- = 0.1, λ = 0.5

🎯 关键突破

重叠数字识别：在 MultiMNIST 数据集（两个高度重叠的数字）上， CapsNet 的错误率远低于 CNN（34% vs 更高的基线），证明了其对空间关系的理解能力。 CapsNet 能同时识别两个重叠数字，而 CNN 倾向于只能识别一个。

重构作为正则化：使用 DigitCaps 的输出重建输入图像， 迫使网络保留重建所需的所有信息（姿态、纹理、精确位置）。 重构不仅用于可视化，更是强大的正则化手段。

PyTorch 实现

Python / PyTorch Capsule Layer 核心实现 
import torch
import torch.nn as nn
import torch.nn.functional as F

class CapsuleLayer(nn.Module):
    def __init__(self, num_capsules, num_route_nodes, in_channels, 
                 out_channels, kernel_size=None, stride=None):
        super(CapsuleLayer, self).__init__()
        self.num_route_nodes = num_route_nodes
        self.num_capsules = num_capsules
        
        if num_route_nodes != -1:  # DigitCaps layer
            self.route_weights = nn.Parameter(
                torch.randn(num_capsules, num_route_nodes, 
                           in_channels, out_channels))
        else:  # PrimaryCaps layer
            self.capsules = nn.ModuleList([
                nn.Conv2d(in_channels, out_channels, 
                         kernel_size=kernel_size, stride=stride, padding=0)
                for _ in range(num_capsules)
            ])
    
    def squash(self, tensor, dim=-1):
        # Squash activation: ||s||^2 / (1 + ||s||^2) * s / ||s||
        squared_norm = (tensor ** 2).sum(dim=dim, keepdim=True)
        scale = squared_norm / (1 + squared_norm)
        return scale * tensor / torch.sqrt(squared_norm + 1e-8)
    
    def forward(self, x):
        if self.num_route_nodes != -1:
            # DigitCaps: Dynamic Routing
            priors = x[None, :, :, None, :] @ self.route_weights[:, None, :, :, :]
            
            logits = torch.zeros(*priors.size()).to(x.device)
            for i in range(3):  # 3 routing iterations
                probs = F.softmax(logits, dim=2)
                outputs = self.squash((probs * priors).sum(dim=2, keepdim=True))
                if i != 2:
                    logits = logits + (priors * outputs).sum(dim=-1, keepdim=True)
            return outputs.squeeze(2)
        else:
            # PrimaryCaps
            outputs = [capsule(x).view(x.size(0), -1, 1) 
                      for capsule in self.capsules]
            outputs = torch.cat(outputs, dim=-1)
            return self.squash(outputs)

# CapsNet 完整架构class CapsNet(nn.Module):
    def __init__(self):
        super(CapsNet, self).__init__()
        self.conv1 = nn.Conv2d(1, 256, kernel_size=9, stride=1)
        self.primary_capsules = CapsuleLayer(
            num_capsules=8, num_route_nodes=-1, 
            in_channels=256, out_channels=32, 
            kernel_size=9, stride=2)
        self.digit_capsules = CapsuleLayer(
            num_capsules=10, num_route_nodes=1152, 
            in_channels=8, out_channels=16)
        
    def forward(self, x):
        x = F.relu(self.conv1(x))
        x = self.primary_capsules(x)
        x = self.digit_capsules(x)
        return x

影响与局限

✨ 深远影响

  • 新范式：首次系统性提出用向量神经元替代标量激活，开辟了神经网络架构的新方向
  • 可解释性：Capsule 的姿态向量提供了对网络决策过程的直接解释
  • Equivariance：实现了真正的等变性（equivariance）而非不变性（invariance），这是 Hinton 长期追求的目标
  • 路由机制：Dynamic Routing 启发了后续注意力机制的研究（如 Transformer 中的注意力）

⚠️ 局限与挑战

  • 计算开销：迭代路由算法计算量大，难以扩展到大规模数据集（如 ImageNet）
  • 训练不稳定：Capsule 对初始化敏感，训练过程不如 CNN 稳定
  • 后续工作：EM Routing、Capsule-VAE、SAR-Capsule 等试图解决这些问题
