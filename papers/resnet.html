The Problem

As deep neural networks became deeper, researchers encountered a perplexing phenomenon: the degradation problem. Unlike overfitting where training error is low but test error is high, degradation occurs when deeper networks exhibit higher training error than their shallower counterparts.

This wasn't a matter of vanishing gradients—the networks could converge. The issue was that adding more layers actually made the networks perform worse, even on the training data. This created a fundamental barrier: how could we build deeper, more powerful networks if depth itself became a liability?

🎯 The Degradation Dilemma

56-layer networks performed worse than 20-layer networks on training data. Adding depth should theoretically increase capacity, but in practice, optimization became impossible.

📉 Not Vanishing Gradients

The problem wasn't vanishing gradients—networks using batch normalization still suffered. It was an intrinsic optimization difficulty with very deep networks.

🧩 Identity Mapping Paradox

If deeper layers could learn identity mappings, deeper networks should perform at least as well as shallow ones. But SGD couldn't find these solutions.

The Innovation

ResNet's breakthrough insight was elegantly simple: instead of learning the underlying mapping H(x) directly, learn the residual mapping F(x) = H(x) - x. The original mapping becomes F(x) + x.

💡 Core Idea: Residual Learning

If the optimal function is closer to an identity mapping than to zero, it's easier to learn the residual (the small perturbation) than to learn the full transformation.

y = F(x, {Wi}) + x

Where F(x) represents the stacked nonlinear layers. The operation F(x) + x is performed by a shortcut connection and element-wise addition.

This reformulation has profound implications: if identity mappings are optimal, the solver can simply drive the weights of the nonlinear layers toward zero to approach identity mappings. In contrast, learning identity mappings with a stack of nonlinear layers is inherently difficult.

PyTorch Implementation

Here's a clean PyTorch implementation of a basic Residual Block and the ResNet architecture:

resnet.py 
import torch
import torch.nn as nn
import torch.nn.functional as F


class BasicBlock(nn.Module):
    """Basic Residual Block for ResNet-18/34"""
    expansion = 1
    
    def __init__(self, in_channels, out_channels, stride=1):
        super(BasicBlock, self).__init__()
        
        # Main path: conv3x3 → bn → relu → conv3x3 → bn
        self.conv1 = nn.Conv2d(in_channels, out_channels, kernel_size=3, 
                               stride=stride, padding=1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_channels)
        self.conv2 = nn.Conv2d(out_channels, out_channels, kernel_size=3,
                               stride=1, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_channels)
        
        # Shortcut connection
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels * self.expansion:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels * self.expansion, kernel_size=1,
                         stride=stride, bias=False),
                nn.BatchNorm2d(out_channels * self.expansion)
            )
    
    def forward(self, x):
        out = F.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out += self.shortcut(x)  # Skip connection!
        out = F.relu(out)
        return out


class Bottleneck(nn.Module):
    """Bottleneck Block for ResNet-50/101/152"""
    expansion = 4
    
    def __init__(self, in_channels, out_channels, stride=1):
        super(Bottleneck, self).__init__()
        
        # 1x1 → 3x3 → 1x1 bottleneck structure
        self.conv1 = nn.Conv2d(in_channels, out_channels, kernel_size=1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_channels)
        self.conv2 = nn.Conv2d(out_channels, out_channels, kernel_size=3,
                               stride=stride, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_channels)
        self.conv3 = nn.Conv2d(out_channels, out_channels * 4, kernel_size=1, bias=False)
        self.bn3 = nn.BatchNorm2d(out_channels * 4)
        
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels * 4:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels * 4, kernel_size=1,
                         stride=stride, bias=False),
                nn.BatchNorm2d(out_channels * 4)
            )
    
    def forward(self, x):
        out = F.relu(self.bn1(self.conv1(x)))
        out = F.relu(self.bn2(self.conv2(out)))
        out = self.bn3(self.conv3(out))
        out += self.shortcut(x)  # The magic happens here!
        out = F.relu(out)
        return out
usage.py 
# Using pretrained ResNet from torchvision
import torchvision.models as models

# Load pretrained ResNet-50
resnet50 = models.resnet50(pretrained=True)

# For transfer learning, replace the final FC layer
num_classes = 10  # Your custom number of classes
resnet50.fc = nn.Linear(resnet50.fc.in_features, num_classes)

# Freeze early layers (optional)
for param in resnet50.parameters():
    param.requires_grad = False
for param in resnet50.fc.parameters():
    param.requires_grad = True

# Forward pass
with torch.no_grad():
    output = resnet50(input_image)
    probabilities = F.softmax(output, dim=1)
