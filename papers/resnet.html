The Problem: Degradation in Deep Networks

Before ResNet, deep neural networks faced a fundamental challenge: as networks became deeper, performance would paradoxically degrade after a certain depth. This wasn't due to overfitting—training error would actually increase with more layers.

Traditional wisdom suggested that adding more layers should always improve performance (or at least not hurt it), since deeper networks can represent more complex functions. However, in practice, a 56-layer network performed worse than a 20-layer network on training data.

Vanishing Gradients

As gradients propagate backward through many layers, they become exponentially smaller, preventing effective weight updates in early layers.

Degradation Problem

With increasing depth, accuracy gets saturated and then degrades rapidly—training error increases, not from overfitting, but from optimization difficulties.

Training Difficulty

Deeper networks are harder to optimize. Simply stacking more layers makes training unstable and convergence difficult.

The Innovation: Residual Learning

ResNet's revolutionary insight was simple yet profound: instead of learning the desired mapping H(x) directly, let layers learn the residual F(x) = H(x) - x. The original mapping becomes F(x) + x.

Residual Learning: y = F(x, {Wi}) + x

Where F(x) represents the residual mapping to be learned, and x is the identity shortcut connection

This reformulation has two key advantages:

  • If identity mapping is optimal, pushing residuals to zero is easier than learning an identity mapping directly
  • The shortcut connections enable gradients to flow directly through the network, eliminating vanishing gradients

The bottleneck design (1×1, 3×3, 1×1 convolutions) reduces computation while maintaining representational power. The 1×1 layers are responsible for reducing and then increasing dimensions, leaving the 3×3 layer a bottleneck with smaller input/output dimensions.

PyTorch Implementation

Basic Residual Block 
import torch
import torch.nn as nn

class BasicBlock(nn.Module):
    expansion = 1
    
    def __init__(self, in_channels, out_channels, stride=1):
        super(BasicBlock, self).__init__()
        self.conv1 = nn.Conv2d(in_channels, out_channels, kernel_size=3, 
                               stride=stride, padding=1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_channels)
        self.relu = nn.ReLU(inplace=True)
        self.conv2 = nn.Conv2d(out_channels, out_channels, kernel_size=3,
                               stride=1, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_channels)
        
        # Shortcut connection
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels, kernel_size=1, 
                         stride=stride, bias=False),
                nn.BatchNorm2d(out_channels)
            )
    
    def forward(self, x):
        out = self.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out += self.shortcut(x)  # Residual connection
        out = self.relu(out)
        return out
Bottleneck Block (for ResNet-50+) 
class Bottleneck(nn.Module):
    expansion = 4
    
    def __init__(self, in_channels, out_channels, stride=1):
        super(Bottleneck, self).__init__()
        
        # 1x1 reduce
        self.conv1 = nn.Conv2d(in_channels, out_channels, kernel_size=1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_channels)
        
        # 3x3 conv
        self.conv2 = nn.Conv2d(out_channels, out_channels, kernel_size=3,
                               stride=stride, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_channels)
        
        # 1x1 expand
        self.conv3 = nn.Conv2d(out_channels, out_channels * 4, kernel_size=1, bias=False)
        self.bn3 = nn.BatchNorm2d(out_channels * 4)
        
        self.relu = nn.ReLU(inplace=True)
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels * 4:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels * 4, kernel_size=1,
                         stride=stride, bias=False),
                nn.BatchNorm2d(out_channels * 4)
            )
    
    def forward(self, x):
        out = self.relu(self.bn1(self.conv1(x)))
        out = self.relu(self.bn2(self.conv2(out)))
        out = self.bn3(self.conv3(out))
        out += self.shortcut(x)
        out = self.relu(out)
        return out

Key Insights

Identity Mappings

When dimensions match, identity shortcuts require no extra parameters. The network only needs to learn residuals, making optimization easier.

Projection Shortcuts

When dimensions don't match, 1×1 convolution projections match dimensions while remaining efficient compared to fully connected shortcuts.

Batch Normalization

ResNet uses batch normalization after every convolution and before activation, stabilizing training and enabling higher learning rates.

No Extra FLOPs

Identity shortcuts add neither parameters nor computational complexity—the network is as efficient as a plain network of the same depth.
