🎯 The Problem: Degradation in Deep Networks

Before ResNet, adding more layers to neural networks actually hurt performance after a certain point—not due to overfitting, but due to optimization difficulties.

🚨 The Degradation Problem

As neural networks became deeper, they suffered from a surprising phenomenon: training error increased with added layers. This wasn't overfitting—the deeper networks performed worse on training data than their shallower counterparts.


The root causes were:

  • Vanishing gradients during backpropagation
  • Difficulty in optimizing very deep architectures
  • Information bottleneck in layer-by-layer transformations

💡 The ResNet Solution

Instead of learning the direct mapping H(x) from input to output, ResNet learns the residual function F(x) = H(x) - x, where the output becomes:


y = F(x) + x

The identity shortcut connection allows gradients to flow directly through the network, solving the vanishing gradient problem and enabling training of networks with 152+ layers.

🔑 Core Innovation: Skip Connections

By adding the input directly to the output of a stack of layers, the network only needs to learn the "residual"—the small changes needed to improve the representation. If the optimal mapping is close to identity, the network simply learns near-zero residuals.

Output = F(x, {Wᵢ}) + x

Where F(x) represents the stacked non-linear layers (convolutions, BN, ReLU)
and x is the identity shortcut connection

🏗️ Architecture & Building Blocks

ResNet introduces two fundamental building blocks: the Basic Block for shallower networks and the Bottleneck Block for deeper architectures.

ResNet Architecture Overview

Residual Building Blocks

🔷 Basic Block (ResNet-18/34)

Used in shallower ResNet variants. Contains two 3×3 convolution layers.

F(x) = W₂(ReLU(W₁(x)))
Output: ReLU(F(x) + x)

🔶 Bottleneck Block (ResNet-50/101/152)

Used in deeper ResNets. Uses 1×1 convolutions to reduce dimensions for efficient 3×3 conv.

1×1 → 3×3 → 1×1 structure
Reduces dimensions for efficient computation

💻 PyTorch Implementation

Clean implementation of ResNet BasicBlock and Bottleneck in PyTorch.

resnet.py 
import torch
import torch.nn as nn

# ResNet BasicBlock for ResNet-18/34
class BasicBlock(nn.Module):
    expansion = 1
    
    def __init__(self, in_channels, out_channels, stride=1):
        super(BasicBlock, self).__init__()
        
        # First 3x3 convolution
        self.conv1 = nn.Conv2d(in_channels, out_channels, kernel_size=3, 
                               stride=stride, padding=1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_channels)
        
        # Second 3x3 convolution
        self.conv2 = nn.Conv2d(out_channels, out_channels, kernel_size=3,
                               stride=1, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_channels)
        
        # Shortcut connection
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels * self.expansion:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels * self.expansion, 
                         kernel_size=1, stride=stride, bias=False),
                nn.BatchNorm2d(out_channels * self.expansion)
            )
    
    def forward(self, x):
        identity = x
        
        out = self.conv1(x)
        out = self.bn1(out)
        out = torch.relu(out)
        
        out = self.conv2(out)
        out = self.bn2(out)
        
        # Add residual connection
        out += self.shortcut(identity)
        out = torch.relu(out)
        
        return out


# ResNet Bottleneck for ResNet-50/101/152
class Bottleneck(nn.Module):
    expansion = 4  # Output is 4x the input channels
    
    def __init__(self, in_channels, out_channels, stride=1):
        super(Bottleneck, self).__init__()
        
        # 1x1 conv for dimension reduction
        self.conv1 = nn.Conv2d(in_channels, out_channels, kernel_size=1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_channels)
        
        # 3x3 conv (bottleneck)
        self.conv2 = nn.Conv2d(out_channels, out_channels, kernel_size=3,
                               stride=stride, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_channels)
        
        # 1x1 conv for dimension restoration
        self.conv3 = nn.Conv2d(out_channels, out_channels * self.expansion,
                               kernel_size=1, bias=False)
        self.bn3 = nn.BatchNorm2d(out_channels * self.expansion)
        
        # Projection shortcut when dimensions don't match
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels * self.expansion:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels * self.expansion,
                         kernel_size=1, stride=stride, bias=False),
                nn.BatchNorm2d(out_channels * self.expansion)
            )
    
    def forward(self, x):
        identity = x
        
        out = torch.relu(self.bn1(self.conv1(x)))
        out = torch.relu(self.bn2(self.conv2(out)))
        out = self.bn3(self.conv3(out))
        
        # The magic: F(x) + x
        out += self.shortcut(identity)
        out = torch.relu(out)
        
        return out

🔗 Related Work

Papers that inspired ResNet and influential follow-up work.

2014

VGG: Very Deep Convolutional Networks

Showed that depth matters, using small 3×3 filters. ResNet extended this idea by solving the degradation problem.

 2015

Highway Networks

Introduced gating mechanisms for skip connections. ResNet simplified this with parameter-free identity mappings.

 2016

Identity Mappings in Deep Residual Networks

Follow-up work analyzing why skip connections work. Showed that direct paths improve gradient flow.

 2017

Densely Connected Networks (DenseNet)

Extended residual connections to connect every layer to every other layer, improving feature reuse.

 2019

EfficientNet

Used compound scaling to uniformly scale all dimensions of ResNet-like architectures efficiently.

 2020

Vision Transformer (ViT)

Applied Transformer architecture to images. ResNet-style hybrid designs remain competitive.
