🎯 核心问题

一词多义（Polysemy）问题：

传统的词嵌入方法（如Word2Vec、GloVe）为每个词分配一个固定的向量，无论上下文如何变化。

例如，"bank"在"river bank"（河岸）和"bank account"（银行账户）中具有完全不同的含义，但传统嵌入会给出相同的表示。
核心洞察： 高质量的词表示应该同时建模：(1) 词使用的复杂特征（如句法和语义）；(2) 这些特征如何随语言上下文变化（即解决一词多义）。

💡 解决方案：深度上下文嵌入

ELMo的创新： 从双向语言模型（biLM）的内部状态学习深度上下文化的词表示。不同于仅使用顶层输出，ELMo利用所有层的表示，并通过任务特定的权重进行组合。

ELMo vs 传统嵌入方法

特性 Word2Vec/GloVe ELMo

💻 PyTorch实现示例

# ELMo表示提取示例 import torch from allennlp.modules.elmo import Elmo, batch_to_ids # 加载预训练ELMo模型 options_file = "elmo_options.json" weight_file = "elmo_weights.hdf5" elmo = Elmo(options_file, weight_file, num_output_representations=1) # 输入句子 sentences = [['I', 'sat', 'by', 'the', 'river', 'bank', '.'], ['I', 'went', 'to', 'the', 'bank', '.']] # 转换为字符ID character_ids = batch_to_ids(sentences) # 获取ELMo表示 (batch_size, max_len, 1024) emeddings = elmo(character_ids)['elmo_representations'][0] # emeddings[0, 5] 和 emeddings[1, 4] 都是"bank"，但表示不同！
# 手动计算ELMo加权组合 class ELMoWeightedSum(torch.nn.Module): def __init__(self, num_layers=3): super().__init__() # 学习各层权重 (包括token嵌入层) self.scalar_mix = torch.nn.Parameter( torch.randn(num_layers) ) self.gamma = torch.nn.Parameter(torch.ones(1)) def forward(self, layer_representations): # layer_representations: [batch, seq_len, num_layers, dim] weights = torch.nn.functional.softmax(self.scalar_mix, dim=0) # 加权求和 elmo_rep = (layer_representations * weights).sum(dim=2) return self.gamma * elmo_rep

🎯 影响与遗产

ELMo的革命性贡献：

1. 首次证明深度上下文表示的有效性 - 开启了NLP的"预训练+微调"时代

2. 基于特征的迁移学习 - 冻结预训练权重，仅训练输入表示，适应各种架构

3. 层次化语义捕获 - 展示不同神经网络层捕获不同语言现象

4. 多义词解决方案 - 为词义消歧提供了有效途径

ELMo为后续BERT、GPT等模型铺平了道路，是NLP从静态嵌入向上下文感知表示转变的关键里程碑。
