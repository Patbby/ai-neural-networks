ELMo

Deep Contextualized Word Representations

📄 NAACL 2018 (Best Paper) 👥 Peters et al. (AllenAI) 🏆 Pioneer of Contextualized Embeddings
📄 PDF 📚 arXiv 💻 Code 🏆 NAACL 2018 Best Paper

🎯 Problem Statement

Traditional word embeddings (Word2Vec, GloVe) assign a single fixed vector to each word, regardless of context. This leads to fundamental limitations:

  • Polysemy Problem: The word "bank" has the same embedding in "river bank" and "bank account"
  • Context Blindness: Static embeddings cannot capture syntax-dependent word usage
  • One Vector per Word: All nuances of meaning are compressed into a single representation

Key Question: How can we create word representations that dynamically adapt to their linguistic context, capturing both complex word characteristics (syntax, semantics) and how meanings vary across contexts?

💡 Key Innovation

ELMo introduces deep contextualized word representations — embeddings that are functions of the entire input sentence, computed by a deep bidirectional language model.

  • Context-Dependent: Each token's representation depends on the full input sequence
  • Deep BiLSTM: Uses internal states of a 2-layer bidirectional language model
  • Layer-wise Representations: Different layers capture different linguistic phenomena (syntax vs. semantics)
  • Task-Specific Weighting: Learnable weights combine layer representations for each downstream task
ELMoktask = γtask × Σj=0L sjtask · hk,jLM

Where sjtask are softmax-normalized weights and γtask is a task-specific scaling factor

🏗️ Architecture

ELMo consists of three main components working together to produce contextualized embeddings:

k (Token) LSTM Layer 1 Forward → LSTM Layer 1 Backward ← LSTM Layer 2 Forward → LSTM Layer 2 Backward ← ResNet hk,1 (Layer 1) hk,2 (Layer 2) Weighted Sum s0, s1, s2 ELMo Downstream Task (NER, SQuAD, SNLI...) hk,0 4096 units 512 projection Layer 0: Context-insensitive Layer 1: Syntax (POS) Layer 2: Semantics (WSD)

Architecture Components

  • Character CNN + Highway Layers: 2048 n-gram filters → 2 highway layers → 512-dim projection. Handles out-of-vocabulary words via character composition
  • 2-Layer BiLSTM: 4096 hidden units, 512-dimensional projections per direction, with residual connections between layers
  • Forward & Backward LMs: Separately trained to predict next/previous tokens, then combined
  • Layer-wise Representations: 3 layers (token, LSTM-1, LSTM-2) each capturing different linguistic abstractions

ELMo achieved state-of-the-art results on six major NLP benchmarks, with relative error reductions of 6-20% across tasks:

Key Finding: Different tasks favor different layer combinations — Syntactic tasks (POS tagging) prefer lower layers, while semantic tasks (word sense disambiguation) prefer higher layers. This validates the hypothesis that different LM layers capture different linguistic phenomena.

Training Details

  • Corpus: 1 Billion Word Language Model Benchmark
  • Training: 10 epochs, batch size 128
  • Perplexity: 39.7 (forward + backward average)
  • Vocabulary: Character-based (handles OOV words)

💻 Code Implementation

Using ELMo with AllenNLP (official implementation):

from allennlp.modules.elmo import Elmo, batch_to_ids

# Download pre-trained model files
options_file = "https://s3-us-west-2.amazonaws.com/allennlp/models/elmo/2x4096_512_2048cnn_2xhighway/elmo_2x4096_512_2048cnn_2xhighway_options.json"
weight_file = "https://s3-us-west-2.amazonaws.com/allennlp/models/elmo/2x4096_512_2048cnn_2xhighway/elmo_2x4096_512_2048cnn_2xhighway_weights.hdf5"

# Initialize ELMo (2 layers of representations)
elmo = Elmo(options_file, weight_file, num_output_representations=2, dropout=0)

# Prepare input sentences
sentences = [['First', 'sentence', '.''], ['Another', '.']]
character_ids = batch_to_ids(sentences)

# Get ELMo embeddings
embeddings = elmo(character_ids)

# embeddings['elmo_representations'] is a list of 2 tensors
# Shape: (batch_size, seq_len, 1024)

Simple PyTorch integration for downstream tasks:

class ELMoClassifier(nn.Module):
    def __init__(self, num_classes, elmo):
        super().__init__()
        self.elmo = elmo
        self.classifier = nn.Linear(1024, num_classes)
        self.dropout = nn.Dropout(0.3)
    
    def forward(self, sentences):
        # Get ELMo representations
        character_ids = batch_to_ids(sentences)
        embeddings = self.elmo(character_ids)
        
        # Use top layer representation
        elmo_output = embeddings['elmo_representations'][-1]
        
        # Mean pooling over sequence
        pooled = elmo_output.mean(dim=1)
        pooled = self.dropout(pooled)
        
        return self.classifier(pooled)

🔗 Related Work & Impact

Papers That Inspired ELMo

  • CoVe (2017): Context vectors from machine translation — ELMo extended this idea using bidirectional language models
  • Semi-supervised Sequence Learning (2015): Showed language model pre-training improves sequence learning
  • Neural Language Models (Bengio et al., 2003):strong> Foundation of distributed word representations

Follow-up Work

  • BERT (2018): Extended ELMo's bidirectional idea with Transformer architecture and masked language modeling
  • GPT Series (2018-2023): Explored left-to-right language modeling for generation tasks
  • Transformer-XL (2019): Extended context length for language modeling
  • XLNet (2019): Combined best of autoregressive and autoencoding approaches

Legacy: ELMo marked a paradigm shift in NLP — from context-independent embeddings (Word2Vec, GloVe) to contextualized representations. This "pre-train then fine-tune" paradigm became the standard approach for NLP tasks, inspiring BERT, GPT, T5, and virtually all modern language models. The paper has been cited over 15,000 times and won the NAACL 2018 Best Paper Award.
