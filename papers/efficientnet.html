EfficientNet: Rethinking Model Scaling for Convolutional Neural Networks

Mingxing Tan, Quoc V. Le

Google Research, Brain Team

📅 ICML 2019 📊 15,000+ Citations 🏷️ Computer Vision
📄 Read Paper 💻 Official Code

🎯 Problem Statement

ConvNets are widely used, but scaling them up to achieve better accuracy often comes with massive computational costs. The key insight: there's a principled way to scale all dimensions of a network—depth, width, and resolution—rather than arbitrarily adjusting one dimension.

Depth Scaling

Adding more layers captures richer features but makes gradients harder to propagate.

Width Scaling

Wider networks capture fine-grained features but struggle with high-level patterns.

Resolution Scaling

Higher resolution inputs capture more details but increase computation quadratically.

💡 Key Innovation

EfficientNet introduces Compound Scaling—a principled method to uniformly scale all three dimensions with a fixed ratio. By balancing depth, width, and resolution, it achieves much better accuracy and efficiency.

🔗 Compound Scaling Formula

Given available resources, scale all dimensions with constant ratios:

d = αφ, w = βφ, r = γφ

s.t. α · β² · γ² ≈ 2 and φ is a user-specified coefficient

Where:

  • d (depth): number of layers
  • w (width): channels per layer
  • r (resolution): input image size
  • φ (phi): compound coefficient

Unlike prior work that only scales one dimension, EfficientNet uses a compound coefficient φ to uniformly scale network width, depth, and resolution. The optimal values (α=1.2, β=1.1, γ=1.15) are found via a small grid search on the baseline EfficientNet-B0.

📈 Results & Impact

EfficientNet achieves state-of-the-art accuracy with dramatically fewer parameters and FLOPs:

  • EfficientNet-B7: 84.4% ImageNet top-1 accuracy (best at the time)
  • 66M parameters vs 557M for GPipe (8.4× smaller)
  • 37B FLOPs vs 42B for ResNet-152 (6.1× faster)
  • 5 transfer learning datasets achieved new SOTA

✨ Key Advantages

  • Parameter Efficiency: Achieves better accuracy with 8.4× fewer parameters
  • Speed: Up to 6.1× faster inference than comparable models
  • Transfer Learning: Significantly better on downstream tasks (CIFAR, Cars, Flowers, etc.)
  • Scalable: Simple compound coefficient controls model size

💻 PyTorch Implementation

import torch
import torch.nn as nn

class Swish(nn.Module):
    def forward(self, x):
        return x * torch.sigmoid(x)

class SEBlock(nn.Module):
    # Squeeze-and-Excitation block
    def __init__(self, in_ch, se_ratio=0.25):
        super().__init__()
        se_ch = max(1, int(in_ch * se_ratio))
        self.avg_pool = nn.AdaptiveAvgPool2d(1)
        self.fc1 = nn.Conv2d(in_ch, se_ch, 1)
        self.fc2 = nn.Conv2d(se_ch, in_ch, 1)
        self.act = Swish()

    def forward(self, x):
        y = self.avg_pool(x)
        y = self.act(self.fc1(y))
        y = torch.sigmoid(self.fc2(y))
        return x * y  # Channel attention
