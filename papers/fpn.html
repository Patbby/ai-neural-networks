Feature Pyramid Networks

Multi-Scale Object Detection Through Hierarchical Feature Fusion

📅 CVPR 2017 👤 Tsung-Yi Lin, Piotr Dollár, Ross Girshick, Kaiming He, Bharath Hariharan, Serge Belongie 🏢 Facebook AI Research (FAIR) 📊 20,000+ Citations
📄 arXiv 📑 PDF 💻 Code

🎯 Problem Statement

Feature pyramids have been a fundamental component of recognition systems for detecting objects at multiple scales. However, deep learning-based object detectors have historically avoided pyramid representations because they are computationally and memory intensive.

Scale Variation Challenge

Objects in images appear at vastly different scales, making it difficult for single-scale detectors to handle both small and large objects effectively.

Computational Cost

Traditional pyramidal approaches require processing multiple image scales independently, leading to significant computational overhead.

Semantic Gap

Lower-level features have high resolution but weak semantics, while high-level features are semantically strong but have low resolution.

Memory Intensity

Explicit pyramid representations consume substantial memory, limiting their deployment in resource-constrained settings.

Before FPN: Existing Approaches

  • Featurized Image Pyramids: Computing features on multiple scaled versions of the input image - accurate but prohibitively expensive
  • Single Scale Features: Fast but poor at detecting small objects that require fine-grained details
  • Pyramidal Feature Hierarchy: Using the natural multi-scale structure of CNNs, but high-level features lack fine spatial resolution
  • SSD-Style Pyramids: Adding extra layers to create a feature pyramid, but miss high-resolution maps important for small objects

💡 Key Innovation

Top-Down Architecture with Lateral Connections

FPN exploits the inherent multi-scale pyramidal hierarchy of deep convolutional networks to construct feature pyramids with marginal extra cost. The key insight is to build high-level semantic feature maps at all scales through a top-down pathway with lateral connections.

Core Design Principles

  • Bottom-Up Pathway: The standard feedforward computation of the backbone ConvNet, producing feature maps at multiple scales
  • Top-Down Pathway: Upsampling higher-resolution features from coarser but semantically stronger maps
  • Lateral Connections: Fusing upsampled features with corresponding bottom-up maps via 1×1 convolutions
  • Shared Classifier/Regressor: Using the same head network on all pyramid levels, enabling efficient multi-scale prediction
Lateral Connection:   Pl = Conv1×1(Cl) + Upsample(Pl+1)

where Pl is the feature pyramid at level l, and Cl is the corresponding backbone feature

💻 Implementation

PyTorch-style FPN Layer

import torch
import torch.nn as nn
import torch.nn.functional as F

class FPN(nn.Module):
    """Feature Pyramid Network"""
    def __init__(self, in_channels_list, out_channels):
        super(FPN, self).__init__()
        
        # 1×1 convolutions for lateral connections
        self.lateral_convs = nn.ModuleList([
            nn.Conv2d(in_ch, out_channels, 1)
            for in_ch in in_channels_list
        ])
        
        # 3×3 convolutions for output
        self.fpn_convs = nn.ModuleList([
            nn.Conv2d(out_channels, out_channels, 3, padding=1)
            for _ in in_channels_list
        ])
        
        self.out_channels = out_channels
    
    def forward(self, features):
        """
        Args:
            features: List of C2, C3, C4, C5 from backbone
        Returns:
            List of P2, P3, P4, P5
        """
        # Build lateral features
        laterals = [conv(f) for conv, f in zip(self.lateral_convs, features)]
        
        # Top-down pathway
        for i in range(len(laterals) - 1, 0, -1):
            upsampled = F.interpolate(
                laterals[i], 
                size=laterals[i-1].shape[-2:],
                mode='nearest'
            )
            laterals[i-1] = laterals[i-1] + upsampled
        
        # 3×3 convolutions to reduce aliasing
        outputs = [conv(lat) for conv, lat in zip(self.fpn_convs, laterals)]
        
        return outputs

RPN with FPN

class RPNHead(nn.Module):
    """Region Proposal Network Head - shared across all levels"""
    def __init__(self, in_channels, num_anchors):
        super().__init__()
        self.conv = nn.Conv2d(in_channels, in_channels, 3, padding=1)
        
        # Classification: object vs background
        self.cls_logits = nn.Conv2d(in_channels, num_anchors, 1)
        
        # Regression: box coordinates
        self.bbox_pred = nn.Conv2d(in_channels, 4 * num_anchors, 1)
    
    def forward(self, x):
        x = F.relu(self.conv(x))
        cls = self.cls_logits(x)
        reg = self.bbox_pred(x)
        return cls, reg

# Anchor assignment by level
# P2: 32² anchors  |  P3: 64² anchors  |  P4: 128² anchors
# P5: 256² anchors |  P6: 512² anchors (2× stride from P5)

Usage with Detectron2

# FPN configuration in Detectron2
from detectron2.config import get_cfg
from detectron2 import model_zoo

cfg = get_cfg()
# FPN-based Faster R-CNN with ResNet-50
cfg.merge_from_file(model_zoo.get_config_file(
    "COCO-Detection/faster_rcnn_R_50_FPN_1x.yaml"))

cfg.MODEL.BACKBONE.FPN_ON = True
cfg.MODEL.FPN.OUT_CHANNELS = 256
cfg.MODEL.FPN.NUM_LEVELS = 5  # P2-P6

# Anchor generator settings
cfg.MODEL.ANCHOR_GENERATOR.SIZES = [[32], [64], [128], [256], [512]]
cfg.MODEL.ANCHOR_GENERATOR.ASPECT_RATIOS = [[0.5, 1.0, 2.0]]

Key Extensions and Variants

  • PAFPN (Path Aggregation FPN): Adds a bottom-up path augmentation to shorten the information path between low and high levels
  • NAS-FPN: Uses neural architecture search to learn the best feature pyramid structure
  • BFP (Balanced FPN): Introduces balanced semantic features to better integrate multi-level features
  • HRFPN: Combines FPN with high-resolution networks for better feature representation
  • BiFPN: Efficient bidirectional FPN with weighted feature fusion (used in EfficientDet)

🎯 Conclusion

Feature Pyramid Networks represent a paradigm shift in multi-scale object detection. By cleverly exploiting the inherent pyramidal hierarchy of deep convolutional networks and adding a top-down pathway with lateral connections, FPN enables the construction of feature pyramids with high-level semantics at all scales at marginal extra cost.

The key insight—building high-level semantic feature maps at all scales—has become a foundational component of modern object detection systems. FPN's design principles have influenced countless subsequent works and remain relevant in today's transformer-based detection architectures.

Why FPN Matters

FPN demonstrated that you don't need expensive multi-scale image processing to handle objects at different scales. By working with the natural multi-scale structure of CNNs and adding smart connections, it achieved both accuracy and efficiency—a combination that has defined the field of object detection ever since.
