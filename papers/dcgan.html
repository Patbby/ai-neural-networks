DCGAN

Unsupervised Representation Learning with Deep Convolutional Generative Adversarial Networks

📅 2015 👥 Alec Radford, Luke Metz, Soumith Chintala 🏛️ indico Research, Facebook AI Research
📄 PDF 📚 arXiv 💻 PyTorch Tutorial

🎯 Problem Statement

Before DCGAN, Generative Adversarial Networks (GANs) were notoriously unstable to train. They often resulted in generators that produced nonsensical outputs, and there was very limited understanding of what GANs actually learned or how their intermediate representations worked.

While CNNs had achieved huge success in supervised learning for computer vision, unsupervised learning with CNNs had received far less attention and success. The challenge was to bridge this gap and make GANs train stably while learning meaningful hierarchical representations.

Key Challenges

  • GAN training instability and mode collapse
  • No clear architectural guidelines for convolutional GANs
  • Limited understanding of what GANs learn internally
  • Difficulty scaling GANs to higher resolutions

💡 Key Innovation

DCGAN introduced a class of CNNs called Deep Convolutional Generative Adversarial Networks with specific architectural constraints that made them stable to train. The core insight was that by carefully designing the architecture following proven CNN principles, GANs could learn stable, hierarchical representations from object parts to scenes.

Architecture Guidelines for Stable DCGANs

  • Replace pooling layers with strided convolutions (discriminator) and fractional-strided convolutions (generator)
  • Use Batch Normalization in both generator and discriminator
  • Remove fully connected hidden layers for deeper architectures
  • Use ReLU activation in generator for all layers except output (Tanh)
  • Use LeakyReLU activation in discriminator for all layers

🏗️ Architecture

The DCGAN generator takes a 100-dimensional uniform noise vector and projects it to a small spatial extent convolutional representation with many feature maps. A series of four fractionally-strided convolutions (often called "deconvolutions") then convert this high-level representation into a 64×64 pixel image.

Key Discoveries

Vector Arithmetic in Latent Space: DCGAN demonstrated that simple arithmetic operations on Z vectors produce semantically meaningful results. For example: smiling_woman - neutral_woman + neutral_man = smiling_man

Filter Visualization: Specific filters learned to draw specific objects (e.g., window filters). Removing "window" filters caused the generator to mostly forget to draw windows.

Feature Quality: DCGAN features achieved competitive performance on supervised tasks without being trained on those datasets, demonstrating the quality of learned representations.

💻 PyTorch Implementation

import torch
import torch.nn as nn

# Generator Architecture
class Generator(nn.Module):
    def __init__(self, nz=100, ngf=64, nc=3):
        super(Generator, self).__init__()
        self.main = nn.Sequential(
            # Input: nz x 1 x 1
            nn.ConvTranspose2d(nz, ngf * 8, 4, 1, 0, bias=False),
            nn.BatchNorm2d(ngf * 8),
            nn.ReLU(True),
            # 4 x 4
            nn.ConvTranspose2d(ngf * 8, ngf * 4, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ngf * 4),
            nn.ReLU(True),
            # 8 x 8
            nn.ConvTranspose2d(ngf * 4, ngf * 2, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ngf * 2),
            nn.ReLU(True),
            # 16 x 16
            nn.ConvTranspose2d(ngf * 2, ngf, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ngf),
            nn.ReLU(True),
            # 32 x 32
            nn.ConvTranspose2d(ngf, nc, 4, 2, 1, bias=False),
            nn.Tanh()
            # 64 x 64
        )
    
    def forward(self, input):
        return self.main(input)

# Discriminator Architecture
class Discriminator(nn.Module):
    def __init__(self, nc=3, ndf=64):
        super(Discriminator, self).__init__()
        self.main = nn.Sequential(
            # Input: nc x 64 x 64
            nn.Conv2d(nc, ndf, 4, 2, 1, bias=False),
            nn.LeakyReLU(0.2, inplace=True),
            # 32 x 32
            nn.Conv2d(ndf, ndf * 2, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ndf * 2),
            nn.LeakyReLU(0.2, inplace=True),
            # 16 x 16
            nn.Conv2d(ndf * 2, ndf * 4, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ndf * 4),
            nn.LeakyReLU(0.2, inplace=True),
            # 8 x 8
            nn.Conv2d(ndf * 4, ndf * 8, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ndf * 8),
            nn.LeakyReLU(0.2, inplace=True),
            # 4 x 4
            nn.Conv2d(ndf * 8, 1, 4, 1, 0, bias=False),
            nn.Sigmoid()
            # 1 x 1
        )
    
    def forward(self, input):
        return self.main(input).view(-1)

Training Hyperparameters

  • Batch size: 128
  • Learning rate: 0.0002 (Adam)
  • Adam β1: 0.5 (not default 0.9)
  • Weight initialization: Normal(0, 0.02)
  • LeakyReLU slope: 0.2
