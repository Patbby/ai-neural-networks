Proximal Policy Optimization

A simple, efficient, and stable policy gradient method that clips probability ratios to prevent destructive policy updates. The go-to algorithm for modern RL applications.

🎯 Problem Statement

Policy gradient methods are foundational to deep reinforcement learning, but they suffer from a critical flaw: destructive policy updates. When the policy network is updated too aggressively, it can collapse to a catastrophic state from which recovery is difficult or impossible.

Prior work attempted to solve this with Trust Region Policy Optimization (TRPO), which uses second-order methods and constraints to ensure stable updates. However, TRPO is:

  • Complex - Requires conjugate gradient and Fisher-vector products
  • Computationally expensive - Cannot take advantage of GPU batch processing efficiently
  • Incompatible with certain architectures (like those with parameter sharing between policy and value functions)

💡 The Question: Can we achieve the stability benefits of TRPO with the simplicity and computational efficiency of first-order optimization methods?

💡 Key Innovation

PPO introduces a clipped surrogate objective that removes the need for complex trust region calculations while achieving the same stability guarantees. The core insight is elegantly simple:

Instead of constraining the policy update, penalize updates that move the probability ratio too far from 1.

The Clipped Objective

LCLIP(θ) = Et[ min( rt(θ)Ât, clip(rt(θ), 1-ε, 1+ε)Ât ) ]

The PPO-Clip objective function, where rt(θ) is the probability ratio and ε is the clipping hyperparameter (typically 0.1 or 0.2)

This clipped objective implements a pessimistic bound on the policy update:

  • When the advantage is positive, the objective flattens when rt > 1+ε
  • When the advantage is negative, the objective flattens when rt < 1-ε
  • The minimum operation takes the lower of the two, preventing the policy from over-optimizing any single batch

The Complete Algorithm

  1. Collect Trajectories: Run the current policy πθold in the environment for N timesteps, storing (st, at, rt, st+1)
  2. Compute Returns: Use Generalized Advantage Estimation (GAE) to compute advantage estimates Ât and discounted returns
  3. Multiple Epochs: For K epochs, sample random minibatches from the collected trajectories
  4. Clipped Update: For each minibatch, compute the clipped surrogate loss and update both policy and value networks
  5. Repeat: Discard old trajectories and collect new ones with the updated policy

# PPO Clipped Surrogate Loss
def compute_ppo_loss(logits_new, logits_old, actions, advantages, epsilon=0.2):
    # 1. Compute probability ratios
    probs_new = torch.softmax(logits_new, dim=-1)
    probs_old = torch.softmax(logits_old, dim=-1)
    
    # Gather probabilities for taken actions
    probs_new_actions = probs_new.gather(-1, actions.unsqueeze(-1)).squeeze()
    probs_old_actions = probs_old.gather(-1, actions.unsqueeze(-1)).squeeze()
    
    # 2. Compute ratio r_t(θ)
    ratio = probs_new_actions / (probs_old_actions + 1e-8)
    
    # 3. Unclipped objective
    unclipped = ratio * advantages
    
    # 4. Clipped objective
    clipped_ratio = torch.clamp(ratio, 1 - epsilon, 1 + epsilon)
    clipped = clipped_ratio * advantages
    
    # 5. Take minimum (pessimistic bound)
    policy_loss = -torch.min(unclipped, clipped).mean()
