Proximal Policy Optimization

A family of policy gradient methods that enables multiple epochs of minibatch updates through a novel surrogate objective with clipped probability ratios

❓ Problem Statement

Before PPO, policy gradient methods faced a fundamental dilemma: how to take large steps to improve sample efficiency without taking excessively large steps that cause performance collapse. Trust Region Policy Optimization (TRPO) addressed this with complex second-order methods, but its implementation was difficult and computationally expensive.

🔥 Policy Collapse

Standard policy gradients perform one update per sample. Multiple gradient steps on the same data can cause the policy to change too much, leading to catastrophic performance drops.

⚖️ Sample vs Stability

On-policy methods require collecting new data after each update, making them sample-inefficient. But using the same data multiple times risks destabilizing training.

🧮 TRPO Complexity

TRPO's trust region constraint requires computing Fisher information matrices and conjugate gradient methods, making it complex to implement and slow to run.

💡 Key Innovation

PPO introduces a clipped surrogate objective that penalizes large policy updates while enabling multiple epochs of minibatch updates on the same data. The key insight: instead of enforcing hard constraints (like TRPO), use a soft clipping mechanism in the objective function itself.

🎯 The Core Ideas

  • Probability Ratio: rt(θ) = πθ(at|st) / πθold(at|st) — measures how much the policy has changed
  • Clipped Objective: LCLIP(θ) = Et[min(rt(θ)Ât, clip(rt(θ), 1-ε, 1+ε)Ât)] — ignores changes outside [1-ε, 1+ε]
  • Multiple Epochs: Can run 4+ epochs on collected data without collapse, dramatically improving sample efficiency
  • Simple Implementation: First-order method with only minor changes to vanilla policy gradient

🏗️ PPO Algorithm

PPO alternates between collecting data by running the policy in the environment, and optimizing the surrogate objective using stochastic gradient ascent. The clipped objective acts as a soft trust region, preventing the new policy from deviating too far from the old policy.

ppo.py - Core Algorithm PyTorch 
import torch
import torch.nn as nn
from torch.distributions import Categorical

class PPO(nn.Module):
    def __init__(self, state_dim, action_dim, lr=3e-4, gamma=0.99, 
                 eps_clip=0.2, K_epochs=4):
        super(PPO, self).__init__()
        
        # Actor network: outputs action probabilities
        self.actor = nn.Sequential(
            nn.Linear(state_dim, 64), nn.ReLU(),
            nn.Linear(64, 64), nn.ReLU(),
            nn.Linear(64, action_dim), nn.Softmax(dim=-1)
        )
        
        # Critic network: estimates state value
        self.critic = nn.Sequential(
            nn.Linear(state_dim, 64), nn.ReLU(),
            nn.Linear(64, 64), nn.ReLU(),
            nn.Linear(64, 1)
        )
        
        self.optimizer = torch.optim.Adam(self.parameters(), lr=lr)
        self.gamma = gamma
        self.eps_clip = eps_clip
        self.K_epochs = K_epochs
    
    def select_action(self, state):
        with torch.no_grad():
            probs = self.actor(state)
            dist = Categorical(probs)
            action = dist.sample()
            log_prob = dist.log_prob(action)
        return action.item(), log_prob.item()
    
    def compute_advantages(self, rewards, values, dones):
        # GAE (Generalized Advantage Estimation)
        advantages = []
        gae = 0
        
        for t in reversed(range(len(rewards))):
            if t == len(rewards) - 1:
                next_value = 0
            else:
                next_value = values[t + 1]
            
            delta = rewards[t] + self.gamma * next_value * (~dones[t]) - values[t]
            gae = delta + self.gamma * 0.95 * gae * (~dones[t])
            advantages.insert(0, gae)
        
        return torch.tensor(advantages)
    
    def update(self, states, actions, old_log_probs, 
               advantages, returns):
        # Convert to tensors
        states = torch.stack(states)
        actions = torch.tensor(actions)
        old_log_probs = torch.tensor(old_log_probs)
        advantages = (advantages - advantages.mean()) / (advantages.std() + 1e-8)
        
        # PPO update for K epochs
        for _ in range(self.K_epochs):
            # Evaluate current policy
            probs = self.actor(states)
            dist = Categorical(probs)
            log_probs = dist.log_prob(actions)
            
            # Compute probability ratio
            ratios = torch.exp(log_probs - old_log_probs)
            
            # Clipped surrogate objective
            surr1 = ratios * advantages
            surr2 = torch.clamp(ratios, 1 - self.eps_clip, 
                               1 + self.eps_clip) * advantages
            actor_loss = -torch.min(surr1, surr2).mean()
            
            # Value loss
            values = self.critic(states).squeeze()
            critic_loss = nn.MSELoss()(values, returns)
            
            # Combined loss
            loss = actor_loss + 0.5 * critic_loss - 0.01 * dist.entropy().mean()
            
            # Optimize
            self.optimizer.zero_grad()
            loss.backward()
            nn.utils.clip_grad_norm_(self.parameters(), 0.5)
            self.optimizer.step()
        
        return loss.item()
Training Loop PyTorch 
def train_ppo(env, agent, max_episodes=1000, 
              steps_per_update=2048, batch_size=64):
    
    for episode in range(max_episodes):
        states, actions, rewards = [], [], []
        log_probs, values, dones = [], [], []
        
        state = env.reset()
        episode_reward = 0
        
        # Collect experience
        for step in range(steps_per_update):
            # Get action and value estimate
            state_tensor = torch.FloatTensor(state)
            action, log_prob = agent.select_action(state_tensor)
            value = agent.critic(state_tensor).item()
            
            # Interact with environment
            next_state, reward, done, _ = env.step(action)
            
            # Store transition
            states.append(state_tensor)
            actions.append(action)
            rewards.append(reward)
            log_probs.append(log_prob)
            values.append(value)
            dones.append(done)
            
            episode_reward += reward
            state = next_state
            
            if done:
                state = env.reset()
        
        # Compute returns and advantages
        advantages = agent.compute_advantages(rewards, values, dones)
        returns = advantages + torch.tensor(values)
        
        # Update policy with collected data
        loss = agent.update(states, actions, log_probs, 
                          advantages, returns)
        
        if episode % 10 == 0:
            print(f"Episode {episode}, Reward: {episode_reward:.2f}")
