PPO

Proximal Policy Optimization Algorithms

OpenAI • arXiv:1707.06347 • August 2017

📄 arXiv 🌐 OpenAI 💻 Code

🎯 Problem Statement

Before PPO, policy gradient methods faced a fundamental dilemma:

  • Vanilla Policy Gradient (VPG) — Simple but sample inefficient (one gradient update per data sample)
  • Trust Region Policy Optimization (TRPO) — Theoretically sound but computationally expensive with complex conjugate gradient methods
  • Constraint-based methods — Hard to implement, don't work well with architectures involving parameter sharing

The key question: How can we achieve the stability and reliability of trust region methods while keeping the algorithm simple enough for general use?

💡 Key Innovation

Clipped Surrogate Objective

PPO introduces a clipped surrogate objective that penalizes large policy changes without the computational burden of second-order methods. Instead of enforcing hard constraints like TRPO, PPO uses a soft clipping mechanism that discourages but doesn't prohibit large ratio deviations.

The PPO-Clip objective: Take the minimum of unclipped and clipped objectives, ensuring pessimistic bound

🏗️ Architecture & Algorithm

🔬 The Clipping Mechanism

Probability Ratio: rt(θ) = πθ(at|st) / πθold(at|st)

When the ratio exceeds (1+ε), the objective is "clipped" — preventing the policy from changing too dramatically in a single update. The min() operator takes the pessimistic bound, ensuring we don't over-optimize when the advantage estimate might be noisy.

💻 PyTorch Implementation

# PPO Implementation in PyTorch
import torch
import torch.nn as nn
import torch.optim as optim
from torch.distributions import Categorical

class ActorCritic(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super(ActorCritic, self).__init__()
        # Shared feature extractor
        self.shared = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU()
        )
        # Actor head (policy)
        self.actor = nn.Linear(hidden_dim, action_dim)
        # Critic head (value)
        self.critic = nn.Linear(hidden_dim, 1)
    
    def forward(self, state):
        features = self.shared(state)
        action_probs = torch.softmax(self.actor(features), dim=-1)
        state_value = self.critic(features)
        return action_probs, state_value
    
    def get_action(self, state):
        probs, value = self.forward(state)
        dist = Categorical(probs)
        action = dist.sample()
        return action, dist.log_prob(action), value

class PPO:
    def __init__(self, state_dim, action_dim, lr=3e-4, 
                 gamma=0.99, epsilon=0.2, 
                 epochs=4, batch_size=64):
        self.policy = ActorCritic(state_dim, action_dim)
        self.optimizer = optim.Adam(self.policy.parameters(), lr=lr)
        self.gamma = gamma
        self.epsilon = epsilon  # Clipping parameter
        self.epochs = epochs
        self.batch_size = batch_size
    
    def compute_advantages(self, rewards, values, dones):
        """GAE (Generalized Advantage Estimation)"""
        advantages = []
        gae = 0
        for t in reversed(range(len(rewards))):
            if t == len(rewards) - 1:
                next_value = 0
            else:
                next_value = values[t + 1]
            delta = rewards[t] + self.gamma * next_value * (1 - dones[t]) - values[t]
            gae = delta + self.gamma * 0.95 * (1 - dones[t]) * gae
            advantages.insert(0, gae)
        return torch.tensor(advantages)
    
    def update(self, states, actions, old_log_probs, 
                returns, advantages):
        states = torch.stack(states)
        actions = torch.stack(actions)
        old_log_probs = torch.stack(old_log_probs)
        
        # Multiple epochs on same data (key PPO innovation)
        for _ in range(self.epochs):
            # Get current policy distribution
            probs, state_values = self.policy(states)
            dist = Categorical(probs)
            
            # Calculate new log probabilities
            log_probs = dist.log_prob(actions)
            
            # Probability ratio r(θ)
            ratios = torch.exp(log_probs - old_log_probs)
            
            # Clipped surrogate objective (THE KEY PPO EQUATION)
            surr1 = ratios * advantages
            surr2 = torch.clamp(ratios, 1 - self.epsilon, 
                               1 + self.epsilon) * advantages
            actor_loss = -torch.min(surr1, surr2).mean()
            
            # Critic loss (MSE)
            critic_loss = nn.MSELoss()(state_values.squeeze(), returns)
            
            # Entropy bonus for exploration
            entropy = dist.entropy().mean()
            
            # Combined loss
            loss = actor_loss + 0.5 * critic_loss - 0.01 * entropy
            
            self.optimizer.zero_grad()
            loss.backward()
            self.optimizer.step()
        
        return loss.item()

📚 Key Insights

Why PPO Became the Default

1. Simplicity Over Complexity: PPO achieves 90% of TRPO's benefits with 10% of the code. No conjugate gradients, no Fisher information matrices.

2. Multiple Epochs: Unlike vanilla PG, PPO can perform multiple gradient updates on the same collected data, dramatically improving sample efficiency.

3. The Min Trick: Taking the minimum of clipped and unclipped objectives is a "pessimistic bound" — it prevents over-optimization when the advantage estimate is noisy.

4. Architecture Agnostic: Works with any differentiable policy, including RNNs, Transformers, and architectures with parameter sharing (unlike TRPO).

5. The ε Hyperparameter: Typically set to 0.1 or 0.2. This single hyperparameter controls the "step size" — much simpler than adaptive KL penalties.
