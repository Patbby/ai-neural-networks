QLoRA: Efficient Finetuning of Quantized LLMs

Tim Dettmers · Artidoro Pagnoni · Ari Holtzman · Luke Zettlemoyer

University of Washington

📄 arXiv 💻 Code 🤗 Hugging Face
← Back to Home
We present QLoRA, an efficient finetuning approach that reduces memory usage enough to finetune a 65B parameter model on a single 48GB GPU while preserving full 16-bit finetuning task performance. QLoRA backpropagates gradients through a frozen, 4-bit quantized pretrained language model into Low Rank Adapters (LoRA). Our best model family, which we name Guanaco, outperforms all previous openly released models on the Vicuna benchmark, reaching 99.3% of the performance level of ChatGPT while only requiring 24 hours of finetuning on a single GPU.

❓ Problem Statement

Before QLoRA, finetuning large language models (LLMs) with billions of parameters required substantial computational resources:

  • A 65B parameter model requires ~780GB of GPU memory in FP16 mode
  • LoRA alone doesn't solve the memory bottleneck of storing activations
  • Existing quantization methods lose too much precision for finetuning
  • Commercial APIs are the only option for most researchers without access to multi-GPU setups

This created a fundamental barrier: only well-resourced organizations could experiment with and improve the largest models.

💡 Key Innovation

QLoRA combines three complementary techniques to achieve unprecedented memory efficiency while maintaining training stability and performance:

  • 1
    4-bit NormalFloat (NF4)

    A new data type that is information theoretically optimal for normally distributed weights. Unlike standard quantization, NF4 adapts to the weight distribution, achieving better precision with the same bit budget.

  • 2
    Double Quantization

    Quantizes the quantization constants themselves, reducing the average memory footprint by an additional ~0.4 bits per parameter. This nested quantization is key to fitting large models in limited memory.

  • 3
    Paged Optimizers

    Uses NVIDIA unified memory to automatically transfer data between CPU and GPU when the GPU runs out of memory. This prevents OOM errors during the memory spikes of gradient checkpointing.

🏅 Competitive Results

The QLoRA-finetuned Guanaco-65B model outperformed all previous openly released models on the Vicuna benchmark at the time of publication. Remarkably, even Guanaco-13B (trained on a single consumer GPU in 12 hours) achieved competitive performance with much larger proprietary models.

💻 PyTorch Implementation

QLoRA can be easily implemented using Hugging Face's PEFT and bitsandbytes libraries:

# QLoRA Implementation with Hugging Face from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training from trl import SFTTrainer # 1. Configure 4-bit quantization with NF4 bnb_config = BitsAndBytesConfig( load_in_4bit=True, bnb_4bit_quant_type="nf4", # NormalFloat 4-bit bnb_4bit_compute_dtype=torch.bfloat16, # BF16 computation bnb_4bit_use_double_quant=True, # Double quantization ) # 2. Load model in 4-bit model = AutoModelForCausalLM.from_pretrained( "meta-llama/Llama-2-7b-hf", quantization_config=bnb_config, device_map="auto", # Auto device placement trust_remote_code=True ) # 3. Prepare model for training model = prepare_model_for_kbit_training(model) # 4. Configure LoRA adapters lora_config = LoraConfig( r=64, # LoRA rank lora_alpha=16, # Scaling factor target_modules=["q_proj", "v_proj"], # Target attention layers lora_dropout=0.1, bias="none", task_type="CAUSAL_LM" ) # 5. Apply LoRA to quantized model model = get_peft_model(model, lora_config) # 6. Train with gradient checkpointing (activates paged optimizers) trainer = SFTTrainer( model=model, train_dataset=dataset, max_seq_length=2048, args=TrainingArguments( per_device_train_batch_size=1, gradient_accumulation_steps=4, num_train_epochs=3, learning_rate=2e-4, optim="paged_adamw_8bit", # Paged optimizer! warmup_ratio=0.03, logging_steps=10, save_strategy="epoch", ) ) trainer.train()

🌍 Impact & Legacy

QLoRA democratized access to large language model finetuning, enabling:

  • Individual researchers to finetune 70B+ parameter models on consumer hardware
  • Open-source community to create high-quality instruction-following models
  • Enterprise adoption of LLM finetuning without massive infrastructure costs
  • New research directions in parameter-efficient transfer learning

It has been integrated into major frameworks including Hugging Face PEFT, Axolotl, and Unsloth, becoming the de facto standard for efficient LLM finetuning.
