🎯 R-CNN

Rich Feature Hierarchies for Accurate Object Detection and Semantic Segmentation

Ross Girshick Jeff Donahue Trevor Darrell Jitendra Malik
Conference
CVPR 2014
Institution
UC Berkeley
arXiv
1311.2524

📄 Abstract

Object detection performance, as measured on the canonical PASCAL VOC dataset, had plateaued in the years prior to this work. The best-performing methods were complex ensemble systems that typically combined multiple low-level image features with high-level context. In this paper, we propose a simple and scalable detection algorithm that improves mean average precision (mAP) by more than 30% relative to the previous best result on VOC 2012—achieving a mAP of 53.3%. Our approach combines two key insights: (1) one can apply high-capacity convolutional neural networks (CNNs) to bottom-up region proposals in order to localize and segment objects and (2) when labeled training data is scarce, supervised pre-training for an auxiliary task, followed by domain-specific fine-tuning, yields a significant performance boost. Since we combine Region proposals with CNNs, we call our method R-CNN: Regions with CNN features.

💡 Key Innovations

1
CNN + Region Proposals

The revolutionary combination of selective search region proposals with deep CNN features, replacing traditional hand-crafted features (HOG, SIFT) with learned hierarchical representations.

2
Supervised Pre-training

Demonstrated that pre-training on ImageNet (large-scale classification) followed by fine-tuning on detection tasks dramatically improves performance when labeled detection data is scarce.

3
Transfer Learning Paradigm

Established the "pre-train then fine-tune" paradigm that became standard practice in computer vision. Showed CNN features are transferable across tasks.

4
Rich Feature Hierarchies

Demonstrated that CNNs learn rich, hierarchical visual features where deeper layers capture increasingly complex and semantic information.

🏗️ Architecture Overview

Three-Stage Pipeline

1

Region Proposals

Generate ~2,000 category-independent region proposals using Selective Search. These proposals capture potential object locations at multiple scales and aspect ratios. Each proposal is dilated by p=16 pixels to include context.

2

Feature Extraction

Extract a 4096-dimensional feature vector from each region proposal using a CNN (AlexNet or VGG-16). Input regions are warped to a fixed 227×227 pixel size. This is the computationally expensive step—each region goes through the full CNN forward pass.

3

Classification & Localization

Classify each region using class-specific linear SVMs. Apply bounding box regression to refine the proposal location. Finally, use non-maximum suppression to eliminate duplicate detections.

Key Findings from Ablation Studies

  • Pre-training matters: Using a CNN pre-trained on ImageNet boosted mAP by 8.0 percentage points
  • Fine-tuning is crucial: Fine-tuning the full network (not just the classifier) improved mAP by 5.2 points
  • SVM vs Softmax: Using SVM classifiers instead of softmax improved mAP by 3.3 points (54.2% vs 50.9%)
  • More proposals help: Performance improved with more region proposals up to ~2,000
  • Bounding box regression: Added ~3-4 percentage points to mAP

⚙️ Technical Implementation

Training Process

Step 1: Pre-train CNN on ImageNet (1000-class classification)
  └─> Use AlexNet or VGG architecture
  └─> Train on 1.2M images

Step 2: Fine-tune on detection data
  └─> Replace 1000-way classifier with (N+1)-way (N classes + background)
  └─> Positive samples: IoU ≥ 0.5 with ground-truth
  └─> Background: IoU < 0.5
  └─> Learning rate: 0.001, Batch: 128 (32 positive, 96 negative)

Step 3: Train class-specific SVMs
  └─> One linear SVM per object class
  └─> Positive: Ground-truth boxes for that class
  └─> Negative: IoU < 0.3 with all ground-truth boxes
  └─> Hard negative mining applied

Step 4: Train bounding box regressors
  └─> One regressor per class
  └─> Targets: Parametrized coordinate transforms
      tx = (Gx - Px) / Pw
      ty = (Gy - Py) / Ph
      tw = log(Gw / Pw)
      th = log(Gh / Ph)

Region Warping Methods

R-CNN tested three methods to convert arbitrary region proposals to the fixed 227×227 CNN input:

  • Tightest square with context: Enclose proposal in tightest square, isotropically scale, include context
  • Tightest square without context: Same but exclude context
  • Wrap: Anisotropically scale the region directly to 227×227 (used in final system)

Adding context (padding the proposal by 16 pixels) improved mAP by 3-5 percentage points.

🔬 Feature Visualization

R-CNN demonstrated that CNNs learn hierarchical features: early layers detect edges and textures, while deeper layers capture object parts and semantic concepts—enabling transfer learning across visual tasks.

🌍 Impact & Legacy

R-CNN was a watershed moment in computer vision, demonstrating that deep learning could revolutionize object detection. It established the template for modern detection systems and sparked the entire R-CNN family of methods.

🏆 Competition Results

Won the ILSVRC2013 detection competition, outperforming the second-place entry by a large margin using a single model (without ensemble).

📈 Detection Revolution

Triggered the shift from hand-crafted features (HOG, SIFT) to learned CNN features across all computer vision tasks.

🔄 Transfer Learning

Popularized the "pre-train on ImageNet, fine-tune on target task" paradigm that became standard practice.

⏱️ Real Progress

While slow (~47 seconds per image on GPU), it proved the concept and inspired Fast R-CNN, Faster R-CNN, and beyond.

📚 Semantic Segmentation

The same architecture achieved competitive results on semantic segmentation by computing features for each region proposal.

🧬 Architecture Evolution

Led to Fast R-CNN (shared convolutions), Faster R-CNN (learned proposals), Mask R-CNN (instance segmentation).
