🎯 The Problem

The attention mechanism is the computational bottleneck in Transformers. While theoretically efficient with O(n²) FLOPs, the standard attention implementation is memory-bound on modern GPUs:

Memory Wall Crisis

• GPU Memory Hierarchy Gap: HBM (High Bandwidth Memory) is ~20-50× slower than SRAM

• Excessive Memory I/O: Standard attention reads/writes Q, K, V matrices from HBM O(n) times

• Memory Scaling: O(n²) memory for attention matrix prevents long sequences (e.g., 16K tokens)

• Low Arithmetic Intensity: Most time spent waiting for memory, not computing

Standard Attention Memory = O(n²) for S = QKT and P = softmax(S)

💡 Core Innovation: IO-Aware Algorithm

FlashAttention reformulates attention as a tiled computation that never materializes the full attention matrix. The key insight: avoid loading inputs from slow HBM to fast SRAM more than necessary.

Three Key Techniques

1. Tiling: Split Q, K, V into blocks that fit in SRAM, compute attention incrementally

2. Recomputation: Recompute attention weights during backward pass instead of storing them

3. Softmax Rescaling: Online softmax algorithm to handle tile boundaries correctly

🏗️ Tiling Architecture

FlashAttention Tiling Mechanism

Online Softmax with Rescaling

The challenge with tiling: softmax requires the maximum value across all keys to compute correct attention weights. FlashAttention uses an online softmax algorithm that updates statistics incrementally:

For each tile j:
m̃j = max(mj-1, max(QKjT))
ℓ̃j = emj-1-m̃jℓj-1 + sum(eQKjT-m̃j)
Oj = diag(ℓ̃j)-1(diag(ℓj-1)emj-1-m̃jOj-1 + eQKjT-m̃jVj)

# FlashAttention-style Tiled Attention (simplified)
import torch
import torch.nn.functional as F

def flash_attention_forward(Q, K, V, block_size=64):
    """
    Simplified FlashAttention forward pass.
    Q, K, V: (batch, seq_len, d_head)
    """
    batch, N, d = Q.shape
    
    # Initialize output and statistics
    O = torch.zeros_like(Q)
    M = torch.full((batch, N), float('-inf'), device=Q.device)
    L = torch.zeros(batch, N, device=Q.device)
    
    # Tile size for Q (B_r) and KV (B_c)
    B_r, B_c = block_size, block_size
    
    # Loop over query blocks
    for i in range(0, N, B_r):
        Q_i = Q[:, i:i+B_r, :]  # Load Q block to SRAM
        m_i = M[:, i:i+B_r]     # Running max
        l_i = L[:, i:i+B_r]     # Running sum exp
        o_i = O[:, i:i+B_r, :]  # Running output
        
        # Loop over key/value blocks
        for j in range(0, N, B_c):
            K_j = K[:, j:j+B_c, :]  # Load K block
            V_j = V[:, j:j+B_c, :]  # Load V block
            
            # Compute S_ij = Q_i @ K_j^T
            S_ij = torch.matmul(Q_i, K_j.transpose(-2, -1))
            
            # Online softmax update
            m_ij = torch.maximum(m_i, S_ij.max(dim=-1).values)
            P_ij = torch.exp(S_ij - m_ij.unsqueeze(-1))
            l_ij = torch.exp(m_i - m_ij) * l_i + P_ij.sum(dim=-1)
            
            # Update output accumulator
            o_i = (l_i.unsqueeze(-1) * torch.exp(m_i - m_ij).unsqueeze(-1) * o_i \
                   + torch.matmul(P_ij, V_j)) / l_ij.unsqueeze(-1)
            
            m_i = m_ij
            l_i = l_ij
        
        O[:, i:i+B_r, :] = o_i
        M[:, i:i+B_r] = m_i
        L[:, i:i+B_r] = l_i
    
    return O

import torch.nn as nn

# Native PyTorch 2.0+ implementation
class FlashAttention(nn.Module):
    def __init__(self, dim, num_heads=8):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = dim // num_heads
        self.qkv = nn.Linear(dim, 3 * dim)
        self.proj = nn.Linear(dim, dim)
    
    def forward(self, x):
        B, N, C = x.shape
        qkv = self.qkv(x).reshape(B, N, 3, self.num_heads, self.head_dim)
        qkv = qkv.permute(2, 0, 3, 1, 2)  # (3, B, H, N, D)
        q, k, v = qkv[0], qkv[1], qkv[2]
        
        # Use PyTorch 2.0 scaled_dot_product_attention with Flash backend
        out = F.scaled_dot_product_attention(q, k, v, is_causal=False)
        
        out = out.transpose(1, 2).reshape(B, N, C)
        return self.proj(out)
