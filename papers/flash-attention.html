FlashAttention: Fast and Memory-Efficient Exact Attention

IO-aware exact attention algorithm using tiling to reduce HBM-SRAM memory traffic

📅 NeurIPS 2022 👤 Tri Dao, Dan Fu, Stefano Ermon, Atri Rudra, Christopher Ré 🏢 Stanford University
📄 arXiv:2205.14135 📑 PDF 💻 Flash Attention Code

🎯 The Problem: Memory-Bound Attention

Standard attention implementations are memory-bound, not compute-bound. The dominant cost is not the arithmetic operations (FLOPs), but reading and writing large N×N attention matrices to GPU High Bandwidth Memory (HBM).

The Memory Bottleneck: Standard attention materializes the full N×N attention matrix in HBM. For sequence length N=16K, this is a 256M-element matrix requiring ~1GB of memory just for intermediate storage!

The standard attention algorithm performs:

  • Compute S = QK^T: O(N²) reads from HBM
  • Compute P = softmax(S): O(N²) writes + O(N²) reads
  • Compute O = PV: O(N²) reads from HBM

Each operation requires reading/writing the full N×N matrix from slow HBM, even though modern GPUs have much faster on-chip SRAM.

Figure 1: GPU Memory Hierarchy - The Flash Attention Insight

💡 The Innovation: IO-Aware Tiling with Online Softmax

FlashAttention's key insight is that we don't need to materialize the full attention matrix. Instead, it uses a tiling algorithm that:

1. Tiling: Split Q, K, V into blocks that fit in fast SRAM
2. Online Softmax: Compute softmax incrementally without storing the full matrix
3. Recomputation: Recompute attention on-the-fly during backward pass instead of storing it

The standard softmax requires the full matrix to compute the normalization denominator. FlashAttention's breakthrough was an online softmax algorithm that maintains running statistics:

mnew = max(mold, xj)
ℓnew = ℓold · emold - mnew + exj - mnew

This allows computing softmax block by block, accumulating only the running maximum (m) and normalization factor (ℓ).

Figure 2: FlashAttention Tiling Algorithm

🔧 The Algorithm: Two-Pass Online Softmax

FlashAttention's algorithm requires two passes over the key/value blocks for each query block:

Pass 1 (Forward): Compute running max (m) and normalization factor (ℓ) for each query block by iterating over all key/value blocks.
Pass 2 (Backward): Recompute attention weights on-the-fly instead of storing them. This trades compute for memory—a favorable tradeoff since attention is memory-bound.

💻 PyTorch Implementation

Using Flash Attention in PyTorch (via the official flash-attn package):

import torch from flash_attn import flash_attn_func # Input tensors: (batch, seqlen, nheads, headdim) batch_size = 32 seq_len = 4096 # Can now be 16K, 32K, even 64K! num_heads = 12 head_dim = 64 # Create query, key, value tensors q = torch.randn(batch_size, seq_len, num_heads, head_dim).cuda() k = torch.randn(batch_size, seq_len, num_heads, head_dim).cuda() v = torch.randn(batch_size, seq_len, num_heads, head_dim).cuda() # Flash Attention - memory-efficient and fast! # Uses optimized CUDA kernels with tiling output = flash_attn_func(q, k, v, dropout_p=0.0, causal=True, # For autoregressive models softmax_scale=None) # output shape: (batch, seqlen, nheads, headdim) # Memory usage: O(N) instead of O(N²)!

🚀 FlashAttention-2: Further Optimizations

The follow-up paper (FlashAttention-2, 2023) introduced additional optimizations:

1. Better Parallelism: The original partitioned work across batch and head dimensions. FA-2 also partitions over the sequence dimension, achieving better occupancy on GPUs.

2. Reduced Non-Matmul FLOPs: FA-2 optimizes the online softmax computation, reducing the overhead from non-matrix-multiplication operations.

3. Better Warp Scheduling: Improved instruction-level parallelism within warps, reducing shared memory reads/writes.

Combined, FlashAttention-2 achieves up to 2× speedup over the original FlashAttention and up to 9× speedup over standard attention implementations.

📝 Summary

FlashAttention revolutionized attention computation by recognizing that the bottleneck was memory bandwidth, not arithmetic. By using a tiled algorithm with online softmax, it:

  • Reduces memory complexity from O(N²) to O(N) without approximation
  • Achieves 2-4× speedup on long sequences (2K-4K tokens)
  • Enables 64K+ sequence lengths on modern GPUs
  • Maintains exact attention—no approximation or sparsity
  • Enables new capabilities like long-document understanding and high-resolution image generation
Bottom Line: FlashAttention is now the standard attention implementation in modern LLMs (GPT-4, Claude, LLaMA, etc.). It demonstrates the power of algorithm-hardware co-design—understanding GPU memory hierarchies to design algorithms that run efficiently on real hardware.
