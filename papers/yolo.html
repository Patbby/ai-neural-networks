YOLO

You Only Look Once: Unified Real-Time Object Detection

📅 CVPR 2016 👥 Joseph Redmon, Santosh Divvala, Ross Girshick, Ali Farhadi 🏛️ University of Washington, Allen Institute for AI, Facebook AI Research
📄 arXiv 📑 PDF 💻 Code

The Problem

Before YOLO: Slow Multi-Stage Detection

Prior object detection systems like R-CNN and DPM used complex multi-stage pipelines:

  • Generate region proposals (selective search or RPN)
  • Extract features from each region
  • Run classifier on each region
  • Apply post-processing (NMS, bbox refinement)

These pipelines were slow (7-10 FPS), hard to optimize, and each component required separate training.

Object detection at the time was treated as a classification problem applied to multiple regions. This approach was computationally expensive and prevented real-time applications like autonomous driving or video analysis.

Key Innovation

Reframing Detection as Regression

YOLO treats object detection as a single regression problem: directly from image pixels to bounding box coordinates and class probabilities. A single neural network predicts both bounding boxes AND class probabilities in one forward pass.

🎯 Unified Architecture

One network, one pass. The entire detection pipeline is a single convolutional network optimized end-to-end.

⚡ Real-Time Speed

Base YOLO: 45 FPS. Fast YOLO: 155 FPS — while achieving double the mAP of other real-time detectors.

🌍 Generalization

YOLO learns general object representations, outperforming other methods when generalizing to new domains like artwork.

🧠 Global Context

By seeing the entire image during training and testing, YOLO encodes contextual information about classes and their appearance.

Architecture & Method

YOLO divides the input image into an S × S grid. If an object's center falls into a grid cell, that cell is responsible for detecting that object.

How It Works

1️⃣
Divide image
into S×S grid
→
2️⃣
Each cell predicts
B bounding boxes
→
3️⃣
Each box has:
x, y, w, h, confidence
→
4️⃣
Class probabilities
conditional on object

Each grid cell predicts B bounding boxes and confidence scores for those boxes. These confidence scores reflect how confident the model is that the box contains an object and how accurate it thinks the box is. Each bounding box consists of 5 predictions: x, y, w, h, and confidence.

If a grid cell contains an object center, it also predicts C conditional class probabilities (P(Class|Object)). At test time, we multiply the conditional class probabilities and the individual box confidence predictions:

P(Class|Object) × P(Object) × IOU = P(Class) × IOU

This gives class-specific confidence scores for each box, encoding both the probability of that class appearing in the box and how well the predicted box fits the object.

Key Achievements

  • First real-time object detector with competitive accuracy — enabled applications like autonomous driving, robotics, and live video analysis
  • End-to-end trainable — the entire system can be trained jointly, optimizing detection performance directly
  • Generalizes better — when tested on artwork datasets (Picasso, People-Art), YOLO significantly outperformed R-CNN and DPM
  • Global reasoning — by seeing the full image, YOLO encodes contextual information, reducing false positives on background
  • Foundation for YOLO family — spawned YOLOv2, YOLOv3, YOLOv4, YOLOv5, YOLOv8 and countless derivatives

Trade-offs

YOLO made some intentional trade-offs:

✓ Strengths

Extremely fast, global context, generalizes well, simple unified architecture

✗ Limitations

Struggles with small objects, makes more localization errors than region-based methods, coarse grid limits precision

Code Implementation

Here's a simplified PyTorch implementation showing the core YOLO architecture:

import torch
import torch.nn as nn

class YOLO(nn.Module):
    def __init__(self, num_classes=20, S=7, B=2):
        super(YOLO, self).__init__()
        self.S = S  # Grid size (S×S)
        self.B = B  # Bounding boxes per cell
        self.num_classes = num_classes
        
        # 24 convolutional layers + 2 FC layers
        self.conv_layers = nn.Sequential(
            # First conv block
            nn.Conv2d(3, 64, kernel_size=7, stride=2, padding=3),
            nn.LeakyReLU(0.1),
            nn.MaxPool2d(2, stride=2),
            
            # ... more conv layers ...
            nn.Conv2d(512, 1024, kernel_size=3, padding=1),
            nn.LeakyReLU(0.1),
            nn.MaxPool2d(2, stride=2),
        )
        
        # Fully connected layers
        self.fc = nn.Sequential(
            nn.Flatten(),
            nn.Linear(1024 * S * S, 4096),
            nn.LeakyReLU(0.1),
            nn.Dropout(0.5),
            # Output: S×S×(B*5 + C) = 7×7×30 for VOC
            nn.Linear(4096, S * S * (B * 5 + num_classes))
        )
    
    def forward(self, x):
        x = self.conv_layers(x)
        x = self.fc(x)
        # Reshape to (batch, S, S, B*5 + C)
        return x.view(-1, self.S, self.S, self.B * 5 + self.num_classes)

Loss Function

YOLO uses a multi-part loss function combining:

Loss = λ_coord × (localization loss) 
     + λ_noobj × (confidence loss for no-object)
     + (confidence loss for object)
     + (classification loss)

where:
- λ_coord = 5 (emphasize coordinate predictions)
- λ_noobj = 0.5 (down-weight no-object loss)
