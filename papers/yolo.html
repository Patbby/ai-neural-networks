You Only Look Once

Joseph Redmon, Santosh Divvala, Ross Girshick, Ali Farhadi

University of Washington, Allen Institute for AI, FAIR · CVPR 2016

⚡ Real-time Detection 🎯 Unified Architecture

🎯 Problem Statement

Before YOLO, object detection systems faced a fundamental speed-accuracy trade-off:

  • Two-stage detectors (R-CNN, Fast R-CNN, Faster R-CNN): Generate region proposals first, then classify each region. Accurate but slow (~7 FPS).
  • Complex pipelines: Multiple separate components (region proposal, feature extraction, classification, bounding box regression).
  • Non-realtime performance: Could not achieve real-time processing required for autonomous vehicles, robotics, and video analysis.
  • Background errors: Traditional methods struggle with distinguishing background regions from objects.

Key Question: Can we create a unified, single-shot object detector that processes images in real-time while maintaining competitive accuracy?

💡 Core Innovation

YOLO introduced a paradigm shift by treating object detection as a single regression problem:

Unified Architecture: A single neural network predicts bounding boxes AND class probabilities directly from full images in one evaluation. No separate region proposal stage needed!

🔄 Single Forward Pass

Unlike two-stage detectors, YOLO looks at the image once and directly predicts what objects are present and where they are.

⚡ Real-time Speed

Base YOLO achieves 45 FPS on Titan X GPU. Fast YOLO reaches 155 FPS with slight accuracy trade-off.

🌍 Global Context

The network reasons globally about the full image and all objects simultaneously, reducing background errors.

🎓 Generalizable

Learns generalizable representations of objects. When trained on natural images, performs well on artwork.

⚖️ Advantages & Limitations

✓ Advantages

  • Extremely fast (45-155 FPS)
  • Single network evaluation
  • Global context reasoning
  • Learns generalizable features
  • Real-time capable
  • Simple unified architecture
  • Open source implementation

✗ Limitations

  • Lower accuracy than two-stage methods
  • Struggles with small objects
  • Localization errors on unusual aspect ratios
  • Each cell predicts only 2 boxes
  • Spatial constraints on nearby objects
  • Coarse feature maps hurt precision
  • Fixed grid size limits detail

🔄 YOLO Evolution

Since the original YOLO paper in 2016, the architecture has evolved significantly:

Version Year Key Innovation mAP
YOLOv1 2016 Single-shot unified detector 63.4
YOLOv2 (YOLO9000) 2017 Batch norm, anchor boxes, multi-scale 78.6
YOLOv3 2018 Darknet-53, feature pyramid, 3 scales ~81
YOLOv4 2020 CSPDarknet53, SPP, PANet, mosaic ~88
YOLOv5 2020 PyTorch, auto-anchor, focus layer ~90
YOLOv8 2023 Anchor-free, decoupled head, C2f ~93
