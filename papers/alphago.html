AlphaGo

Mastering the game of Go with deep neural networks and tree search

📅 Nature 529, 484-489 (2016) 👥 Silver et al., Google DeepMind 🏆 First AI to defeat human Go champion
Nature Paper DeepMind Blog Community Code

🎯 The Challenge

For decades, Go was considered the "grand challenge" for artificial intelligence—a game so complex that it was thought to be at least a decade away from being mastered by computers.

Enormous Search Space

The game of Go has approximately 10170 possible board positions—more than the number of atoms in the observable universe. This dwarfs chess (1047), making exhaustive search impossible.

Complex Position Evaluation

Unlike chess where pieces have clear values, evaluating Go board positions requires deep pattern recognition and strategic understanding that was difficult to formalize programmatically.

Deep Neural Networks + MCTS

AlphaGo combined deep neural networks for pattern recognition with Monte Carlo Tree Search, trained through both supervised learning from expert games and reinforcement learning from self-play.

💡 Key Innovation

AlphaGo introduced a novel approach that combined two types of deep neural networks with an advanced tree search algorithm:

Dual Network Architecture + APV-MCTS

  • Policy Networks: Trained to predict expert moves, providing a probability distribution over legal moves
  • Value Networks: Trained to evaluate board positions, estimating the probability of winning from a given state
  • APV-MCTS: Asynchronous Policy and Value Monte Carlo Tree Search combining both networks for move selection
  • Two-stage Training: Supervised learning from 30 million expert games + reinforcement learning from self-play

💻 Implementation

While DeepMind's original AlphaGo used extensive computational resources, the core concepts can be implemented in PyTorch:

# AlphaGo-style Policy Network Architecture
import torch
import torch.nn as nn

class PolicyNetwork(nn.Module):
    def __init__(self, board_size=19, num_channels=192):
        super().__init__()
        
        # Input: 48 feature planes representing board state
        self.conv_initial = nn.Conv2d(48, num_channels, kernel_size=5, padding=2)
        self.bn_initial = nn.BatchNorm2d(num_channels)
        
        # 12 residual blocks (AlphaGo used 12-13)
        self.residual_blocks = nn.ModuleList([
            ResidualBlock(num_channels) for _ in range(12)
        ])
        
        # Policy head: outputs probability distribution over moves
        self.policy_conv = nn.Conv2d(num_channels, 2, kernel_size=1)
        self.policy_bn = nn.BatchNorm2d(2)
        self.policy_fc = nn.Linear(board_size * board_size * 2, 
                                   board_size * board_size + 1)  # +1 for pass
        
        # Value head: estimates win probability
        self.value_conv = nn.Conv2d(num_channels, 1, kernel_size=1)
        self.value_bn = nn.BatchNorm2d(1)
        self.value_fc1 = nn.Linear(board_size * board_size, 256)
        self.value_fc2 = nn.Linear(256, 1)
        
    def forward(self, x):
        # Feature extraction
        x = torch.relu(self.bn_initial(self.conv_initial(x)))
        
        # Residual tower
        for block in self.residual_blocks:
            x = block(x)
        
        # Policy output
        policy = torch.relu(self.policy_bn(self.policy_conv(x)))
        policy = policy.view(policy.size(0), -1)
        policy = self.policy_fc(policy)
        
        # Value output
        value = torch.relu(self.value_bn(self.value_conv(x)))
        value = value.view(value.size(0), -1)
        value = torch.relu(self.value_fc1(value))
        value = torch.tanh(self.value_fc2(value))  # [-1, 1] for win probability
        
        return policy, value

class ResidualBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, kernel_size=3, padding=1)
        self.bn1 = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, kernel_size=3, padding=1)
        self.bn2 = nn.BatchNorm2d(channels)
        
    def forward(self, x):
        residual = x
        out = torch.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out += residual  # Skip connection
        return torch.relu(out)

Training Pipeline

# Two-stage training as described in the paper
# Stage 1: Supervised Learning (SL) on expert games
def train_sl_policy(network, expert_games, epochs=100):
    optimizer = torch.optim.SGD(network.parameters(), lr=0.003, momentum=0.9)
    
    for epoch in range(epochs):
        for batch in expert_games:
            positions, expert_moves = batch
            policy_pred, _ = network(positions)
            
            # Cross-entropy loss on expert moves
            loss = nn.cross_entropy_loss(policy_pred, expert_moves)
            
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

🔬 Technical Details

Network Architecture

Policy Network: 13 convolutional layers, 192 filters, 5×5 kernels

Value Network: Similar architecture with single scalar output

Rollout Policy: Small linear softmax with pattern features

Training Data

SL: 30 million positions from KGS Go server

RL: Self-play games against previous iterations

Hardware: 50 GPUs for SL, distributed training for RL

Search Parameters

Simulations: 1,600 per move (tournament play)

Parallel: Asynchronous multi-threaded search

Exploration: UCT formula with policy priors
