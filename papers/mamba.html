🎯 Abstract

Foundation models are almost universally based on the Transformer architecture and its core attention module. Many subquadratic-time architectures such as linear attention, gated convolution, recurrent models, and structured state space models (SSMs) have been developed to address Transformers' computational inefficiency on long sequences, but they have not performed as well as attention on important modalities such as language.

This paper identifies a key weakness of such models: their inability to perform content-based reasoning. By making SSM parameters functions of the input, Mamba enables selective propagation or forgetting of information based on the current token. Combined with a hardware-aware parallel algorithm, Mamba achieves 5× higher throughput than Transformers with linear scaling in sequence length, while matching or exceeding Transformer performance on language modeling tasks.

🔑 Key Innovations

🎯 Selective State Spaces (S6)

Mamba introduces input-dependent SSM parameters (Δ, B, C), allowing the model to selectively focus on or ignore specific tokens. This "selection mechanism" enables content-based reasoning previously only possible with attention.

⚡ Hardware-Aware Parallel Scan

A custom CUDA kernel implements parallel scan algorithms that avoid materializing large intermediate states, achieving efficient computation on modern GPUs while maintaining linear complexity.

🏗️ Simplified Architecture

Mamba combines the best of SSM architectures with Transformer MLP blocks into a single, homogeneous design—no attention layers, no separate MLP blocks, just selective SSM blocks.

📈 Linear Complexity

Unlike Transformers' O(n²) attention, Mamba scales linearly with sequence length, enabling efficient processing of million-length sequences with constant inference memory.

🏗️ Architecture

State Space Model Foundation

State Space Models (SSMs) are inspired by control theory and describe how a system's internal state evolves over time. The continuous-time formulation maps an input x(t) to output y(t) through a latent state h(t):

# Continuous-time SSM equations
h'(t) = Ah(t) + Bx(t)    # State evolution
y(t) = Ch(t)               # Output projection

To use SSMs in deep learning, we discretize them using the Zero-Order Hold (ZOH) method with step size Δ:

# Discretized SSM (recurrent form)
ht = Āht-1 + B̄xt
yt = Cht

# Where:
Ā = exp(Δ · A)                     # Discretized A
B̄ = (ΔA)-1(exp(ΔA) - I) · ΔB  # Discretized B

Traditional SSM vs Selective SSM

Evolution: From Fixed to Selective State Spaces
t)
Traditional SSMs use fixed parameters (LTI systems), while Mamba's selective SSM makes parameters depend on the input, enabling content-based filtering and selective memory.

The Mamba Block Architecture

The Mamba block elegantly combines components from both SSM and Transformer architectures. Unlike Transformers which use separate attention and MLP blocks, Mamba integrates everything into a single block:

Mamba Block Structure
t = Āht-1 + B̄xt Linear x → (Δ, B, C) × SiLU + y Residual No Attention • No MLP Blocks • Pure Selective SSM
The Mamba block: Input is projected, one branch goes through Conv1D → SiLU → Selective SSM, the other through SiLU only. Outputs are multiplied and added with residual connection.

Complexity Comparison

🔴 Transformer

  • Training: O(n²) time, O(n) memory
  • Inference: O(n) per step, O(n) KV cache
  • Context: Limited by memory
  • Attention: Global pairwise comparisons

🟢 Mamba

  • Training: O(n) time, O(n) memory
  • Inference: O(1) per step, O(1) state
  • Context: Million+ tokens feasible
  • Selectivity: Content-dependent gating

Long Context Performance

One of Mamba's most impressive capabilities is handling extremely long sequences. Unlike Transformers where attention becomes prohibitive, Mamba scales linearly:

# Inference memory per step
Transformer: O(n) (growing KV cache)    # Memory increases with sequence length
Mamba: O(1) (fixed state size)          # Constant memory regardless of length

This property enables applications like processing entire books, genomic sequences, or hours of audio with a single forward pass.

Follow-up Research

The Mamba architecture has inspired significant follow-up work:

  • Mamba-2 (2024): Structured State Space Duality (SSD) framework for better theoretical understanding
  • Vision Mamba (Vim): Bidirectional SSM for vision transformers replacement
  • VMamba: Cross-scan module for 2D spatial modeling
  • Griffin/Gemma 2: Google's hybrid attention-SSM architecture
  • Jamba: Production-scale hybrid models combining Mamba with attention

💡 Impact & Significance

Mamba represents a paradigm shift in sequence modeling. For nearly a decade, Transformers dominated deep learning due to their expressive attention mechanism. Mamba proves that:

🔄 New Paradigm

Linear-complexity models can match or exceed Transformer performance on dense modalities like language, challenging the assumption that attention is necessary.

🚀 Efficiency at Scale

The 5× inference speedup and linear scaling make long-context applications practical and affordable at production scale.

🔬 Theory Meets Practice

Mamba bridges classical control theory (state spaces) with modern deep learning, opening new research directions.

🌐 Broad Applicability

From language to genomics to vision, Mamba demonstrates that selective state spaces are a general-purpose sequence modeling primitive.

The paper's release in December 2023 sparked intense research interest, with hundreds of follow-up papers extending Mamba to new domains and improving its theoretical foundations. As foundation models continue to scale, Mamba's efficiency advantages become increasingly critical—making it one of the most influential architecture papers of the 2020s.
