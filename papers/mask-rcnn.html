Mask R-CNN

Instance Segmentation & Object Detection with Pixel-Level Precision

📅 ICCV 2017 (Best Paper) 👥 Kaiming He, Georgia Gkioxari, Piotr Dollár, Ross Girshick 🏛️ Facebook AI Research (FAIR)
📄 PDF 📚 arXiv 💻 Code

🎯 Problem Statement

Before Mask R-CNN, the computer vision community faced a significant gap between object detection and instance segmentation:

Core Challenge: How can we simultaneously detect objects in an image AND generate high-quality segmentation masks for each individual instance, all within a unified framework?

Previous approaches like R-CNN, Fast R-CNN, and Faster R-CNN excelled at object detection but couldn't perform pixel-level segmentation. Meanwhile, semantic segmentation methods like FCN could classify pixels but couldn't distinguish between different instances of the same class.

The key technical hurdle was the misalignment problem: Faster R-CNN's RoIPool operation performed coarse spatial quantization, causing feature misalignment that severely degraded mask quality for pixel-level prediction tasks.

💡 Key Innovation

Core Idea: Extend Faster R-CNN by adding a parallel branch for predicting segmentation masks on each Region of Interest (RoI), combined with a novel RoIAlign layer that preserves exact spatial locations through bilinear interpolation.

🔄 RoIAlign Layer

Replaces RoIPool with a quantization-free alignment mechanism using bilinear interpolation. Despite seeming minor, this improves mask accuracy by 10-50% relative improvement.

🔀 Decoupled Prediction

Predicts a binary mask for each class independently, without competition among classes. Classification and segmentation are separate tasks, unlike FCN's multi-class per-pixel approach.

📐 FCN for Masks

Uses a small Fully Convolutional Network (FCN) to predict m×m masks from each RoI, maintaining explicit spatial layout without collapsing features into vectors.

⚡ Multi-Task Learning

Single framework handles object detection, instance segmentation, and human pose estimation simultaneously at 5 FPS with state-of-the-art results on all tasks.

🏗️ Architecture

The architecture follows a two-stage approach: (1) Region Proposal Network (RPN) generates candidate object proposals, and (2) For each RoI, the network predicts a class label, bounding box offset, AND a binary mask. The mask branch is a small FCN applied to each RoI independently.

🧮 Loss Function

Mask R-CNN is trained with a multi-task loss on each sampled RoI:

L = Lcls + Lbox + Lmask
  • Lcls: Classification loss (cross-entropy over K+1 classes)
  • Lbox: Bounding box regression loss (Smooth L1)
  • Lmask: Mask loss defined only on positive RoIs (IoU > 0.5 with GT)

For the mask branch, Mask R-CNN outputs K binary masks of resolution m×m (typically 28×28), one for each of the K classes. Only the mask corresponding to the ground-truth class contributes to the loss (average binary cross-entropy loss). This decouples mask prediction from classification.

# Mask loss computation (simplified)
# mask_pred: [N, K, H, W] - predicted masks for all classes
# mask_target: [N, H, W] - ground truth mask
# class_targets: [N] - ground truth class labels

# Select mask for the ground-truth class only
pred_mask_for_gt_class = mask_pred[range(N), class_targets]

# Binary cross-entropy loss
mask_loss = F.binary_cross_entropy_with_logits(
    pred_mask_for_gt_class, mask_target

💻 PyTorch Implementation

import torch
import torch.nn as nn
import torchvision
from torchvision.models.detection import maskrcnn_resnet50_fpn
from torchvision.models.detection.faster_rcnn import FastRCNNPredictor
from torchvision.models.detection.mask_rcnn import MaskRCNNPredictor

# Load pre-trained Mask R-CNN with ResNet-50-FPN backbone
model = maskrcnn_resnet50_fpn(pretrained=True)

# Modify for custom number of classes (e.g., 5 object classes + background)
num_classes = 6

# Replace box predictor
in_features = model.roi_heads.box_predictor.cls_score.in_features
model.roi_heads.box_predictor = FastRCNNPredictor(in_features, num_classes)

# Replace mask predictor
in_features_mask = model.roi_heads.mask_predictor.conv5_mask.in_channels
hidden_layer = 256
model.roi_heads.mask_predictor = MaskRCNNPredictor(
    in_features_mask, hidden_layer, num_classes
)

# Inference
model.eval()
with torch.no_grad():
    predictions = model(images)  # List of dicts with 'boxes', 'labels', 'scores', 'masks'
    
# Each prediction contains:
# - boxes: [N, 4] bounding box coordinates
# - labels: [N] class labels
# - scores: [N] confidence scores
# - masks: [N, 1, H,W] binary masks (after sigmoid)

Training Configuration:

  • Backbone: ResNet-50/101 + FPN
  • Image size: Short side 800px
  • RoIAlign output: 7×7 for box, 14×14 for mask
  • Mask resolution: 28×28
  • Batch size: 16 (2 images per GPU × 8 GPUs)
  • Learning rate: 0.02, decay at iterations 120k and 160k
  • Training iterations: 180k (COCO trainval35k)

📚 Related Work

Faster R-CNN

Ren et al., NeurIPS 2015
The foundation that Mask R-CNN extends, introducing the Region Proposal Network (RPN) for end-to-end object detection.

FCIS (Fully Convolutional Instance Segmentation)

Li et al., CVPR 2017
Contemporary work on instance segmentation that uses position-sensitive score maps but suffers from systematic errors on overlapping instances.

Feature Pyramid Networks (FPN)

Lin et al., CVPR 2017
The multi-scale feature extraction backbone used in Mask R-CNN, combining low-resolution and high-resolution features.

Mask Scoring R-CNN

Huang et al., CVPR 2019
Improvement that learns to score mask quality directly, addressing the misalignment between classification confidence and mask quality.

PANet

Liu et al., CVPR 2018
Path Aggregation Network that improves information flow in FPN-style architectures for instance segmentation.

HTC (Hybrid Task Cascade)

Chen et al., CVPR 2019
Multi-stage instance segmentation framework that interweaves detection and segmentation features for better performance.

🎓 Key Takeaways

  1. Pixel alignment matters: The seemingly simple change from RoIPool to RoIAlign had massive impact (10-50% improvement), showing the importance of careful architectural decisions for pixel-level tasks.
  2. Decouple when possible: Separating mask prediction from classification (binary mask per class vs. multi-class softmax) avoids competition between classes and improves results.
  3. Multi-task is powerful: A single framework can excel at detection, segmentation, and keypoint estimation simultaneously, suggesting shared representations are beneficial.
  4. Simplicity wins: Mask R-CNN's elegance and ease of implementation helped it become a standard baseline, despite more complex contemporaries.
  5. Generalization is key: The framework readily extends to new tasks (pose estimation), demonstrating the value of flexible, modular architectures.
