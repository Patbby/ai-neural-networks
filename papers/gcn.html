Semi-Supervised Classification with Graph Convolutional Networks

Thomas N. Kipf & Max Welling

🎯 Problem Statement

Before GCN, learning on graph-structured data faced significant challenges:

  • Traditional CNNs cannot directly operate on non-Euclidean graph data
  • Spectral methods were computationally expensive (O(n³) for eigendecomposition)
  • Graph neural networks of the time didn't scale to large graphs
  • Limited labeled data in many graph learning scenarios (e.g., citation networks)

The key question: How can we generalize convolutions to graphs efficiently, enabling scalable deep learning on graph-structured data?

💡 Key Innovation

Spectral Graph Convolution via First-Order Approximation

GCN introduces a scalable, layer-wise propagation rule based on a first-order approximation of spectral graph convolutions. This elegant simplification reduces complexity from O(n³) to O(|E|), making it linear in the number of edges.

H(l+1) = σ(D̃-1/2 Ã D̃-1/2 H(l) W(l))

The GCN propagation rule: Normalized adjacency matrix × Node features × Learnable weights

🏗️ Architecture

🔬 Method Details

1. Adjacency Matrix (A)

Represents graph structure. Aij = 1 if nodes i and j are connected, 0 otherwise. Self-connections added via I (identity).

2. Degree Matrix (D̃)

Diagonal matrix where D̃ii = Σj(Aij + Iij). Counts connections including self-loops.

3. Normalization (D̃-1/2ÃD̃-1/2)

Symmetric normalization prevents numerical instabilities and exploding/vanishing gradients.

4. Feature Transformation (W)

Learnable weight matrix transforms features at each layer. Enables complex feature learning.

Normalized Adjacency Matrix:

Ã = D̃-1/2 (A + IN) D̃-1/2

💻 PyTorch Implementation

# GCN Layer Implementation in PyTorch
import torch
import torch.nn as nn
import torch.nn.functional as F

class GCNLayer(nn.Module):
    def __init__(self, in_features, out_features):
        super(GCNLayer, self).__init__()
        self.linear = nn.Linear(in_features, out_features)
    
    def forward(self, X, adj):
        # X: Node features (N × in_features)
        # adj: Normalized adjacency matrix Ã (N × N)
        X = self.linear(X)           # Feature transformation
        X = torch.mm(adj, X)          # Graph convolution: ÃXW
        return X

class GCN(nn.Module):
    def __init__(self, nfeat, nhid, nclass, dropout=0.5):
        super(GCN, self).__init__()
        self.gc1 = GCNLayer(nfeat, nhid)
        self.gc2 = GCNLayer(nhid, nclass)
        self.dropout = dropout
    
    def forward(self, x, adj):
        x = F.relu(self.gc1(x, adj))
        x = F.dropout(x, self.dropout, training=self.training)
        x = self.gc2(x, adj)
        return F.log_softmax(x, dim=1)

# Compute normalized adjacency matrix Ã
def normalize_adj(adj):
    """Symmetric normalization: D̃^(-½)(A+I)D̃^(-½)"""
    adj = adj + torch.eye(adj.size(0))  # Add self-loops
    deg = adj.sum(dim=1)                 # Degree matrix
    deg_inv_sqrt = torch.pow(deg, -0.5)
    deg_inv_sqrt[torch.isinf(deg_inv_sqrt)] = 0
    deg_mat = torch.diag(deg_inv_sqrt)
    return deg_mat @ adj @ deg_mat       # Ã = D̃^(-½) Ã D̃^(-½)

📚 Key Insights

Why GCN Works So Well

1. Spectral Approximation: By taking only the first-order Chebyshev approximation (K=1), GCN achieves an optimal balance between computational efficiency and expressiveness.

2. Renormalization Trick: Adding self-loops (IN) and symmetric normalization prevents the numerical instabilities that plagued earlier spectral methods.

3. Semi-Supervised Structure: The graph Laplacian regularization naturally propagates labels from labeled to unlabeled nodes through graph connections.

4. Weight Sharing: Like CNNs, GCN uses shared weights across all nodes, enabling the model to generalize to unseen graph structures.
