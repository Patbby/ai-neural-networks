Adam

A Method for Stochastic Optimization

📅 ICLR 2015 👥 Diederik P. Kingma & Jimmy Lei Ba 🏛️ University of Amsterdam & University of Toronto 📊 100,000+ Citations
arXiv:1412.6980 PDF PyTorch Code

🎯 Problem Statement

Before Adam, training deep neural networks required careful tuning of learning rates and optimization hyperparameters. Stochastic Gradient Descent (SGD), while effective, needed manual learning rate scheduling and performed poorly with sparse gradients. AdaGrad adapted learning rates per parameter but had aggressive, monotonically decreasing rates. RMSProp improved upon this but lacked momentum.

Core Challenge

How can we design an optimization algorithm that combines the best of both worlds: AdaGrad's ability to handle sparse gradients and RMSProp's ability to handle non-stationary objectives, while being computationally efficient and requiring minimal memory?

💡 Key Innovation

Adam (Adaptive Moment Estimation) introduces a method that computes individual adaptive learning rates for different parameters from estimates of first-order moments (mean) and second-order moments (uncentered variance) of the gradients. The name "Adam" derives from adaptive moment estimation.

θt = θt-1 - α · m̂t / (√v̂t + ε)

Where m̂t is the bias-corrected first moment estimate (mean) and v̂t is the bias-corrected second raw moment estimate (uncentered variance). This update rule combines momentum with adaptive learning rates, making it invariant to diagonal rescaling of gradients.

⚙️ The Adam Algorithm

Require: α: Stepsize (default: 0.001)
Require: β₁, β₂ ∈ [0, 1): Exponential decay rates (defaults: 0.9, 0.999)
Require: f(θ): Stochastic objective function
Require: θ₀: Initial parameter vector
m₀ ← 0 (Initialize 1st moment vector)
v₀ ← 0 (Initialize 2nd moment vector)
t ← 0 (Initialize timestep)
while θt not converged do
    t ← t + 1
    gt ← ∇θ ft(θt-1) (Get gradients)
    mt ← β₁ · mt-1 + (1 - β₁) · gt (Update biased first moment)
    vt ← β₂ · vt-1 + (1 - β₂) · gt² (Update biased second moment)
    m̂t ← mt / (1 - β₁t) (Compute bias-corrected first moment)
    v̂t ← vt / (1 - β₂t) (Compute bias-corrected second moment)
    θt ← θt-1 - α · m̂t / (√v̂t + ε) (Update parameters)
end while
return θt

🔧 Initialization Bias Correction

Adam's moment estimates are initialized to zero, causing them to be biased toward zero, especially during the initial timesteps. The authors introduce bias correction terms to counteract this:

m̂t = mt / (1 - β₁t)    and    v̂t = vt / (1 - β₂t)

These correction terms ensure that early estimates are properly scaled, preventing the algorithm from taking overly conservative steps at the beginning of training when the moment estimates haven't accumulated enough information.

Impact on Deep Learning

Adam has become the de facto standard optimizer for training deep neural networks. It is the default choice in popular frameworks like PyTorch, TensorFlow, and JAX. The algorithm's robustness to hyperparameter choices and its ability to handle diverse problem types have made it indispensable in the deep learning toolkit.

Real-World Applications

Adam powers training for: Large Language Models (GPT, LLaMA, Claude), Computer Vision models (ResNet, ViT), Generative models (Stable Diffusion, GANs), Reinforcement Learning agents, and virtually every modern deep learning system.

From-Scratch Implementation

class Adam:
    def __init__(self, params, lr=0.001, 
                 betas=(0.9, 0.999), eps=1e-8):
        self.params = list(params)
        self.lr = lr
        self.beta1, self.beta2 = betas
        self.eps = eps
        self.t = 0
        
        # Initialize moment estimates
        self.m = [torch.zeros_like(p) for p in self.params]
        self.v = [torch.zeros_like(p) for p in self.params]
    
    def step(self):
        self.t += 1
        
        for i, param in enumerate(self.params):
            if param.grad is None:
                continue
            
            g = param.grad.data
            
            # Update biased first moment estimate
            self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * g
            
            # Update biased second raw moment estimate
            self.v[i] = self.beta2 * self.v[i] + (1 - self.beta2) * (g ** 2)
            
            # Compute bias-corrected estimates
            m_hat = self.m[i] / (1 - self.beta1 ** self.t)
            v_hat = self.v[i] / (1 - self.beta2 ** self.t)
            
            # Update parameters
            param.data -= self.lr * m_hat / (torch.sqrt(v_hat) + self.eps)

🔄 AdaMax Variant

The paper also introduces AdaMax, a variant based on the infinity norm (L∞). Instead of using the L2 norm for the second moment, AdaMax uses the maximum of past gradients:

ut = max(β₂ · ut-1, |gt|)    and    θt = θt-1 - (α / (1 - β₁t)) · mt / ut

AdaMax is more stable than Adam when gradients have large variance, as the max operation is less sensitive to outliers than the squared gradient average.

📚 Related Work

  • 2011
    AdaGrad - Adaptive Subgradient Methods
    Duchi et al. - Adapts learning rates per parameter based on historical gradients. Inspiration for Adam's adaptive rates.
  • 2012
    RMSProp
    Tieleman & Hinton - Uses exponential moving average of squared gradients for adaptive learning. Inspiration for Adam's second moment.
  • 2014
    Adam (this paper)
    Kingma & Ba - Combines momentum with adaptive learning rates and adds bias correction.
  • 2017
    AdamW - Decoupled Weight Decay
    Loshchilov & Hutter - Fixes weight decay implementation in Adam, now widely used in practice.
  • 2018
    LAMB/LARS - Large Batch Optimization
    You et al. - Layer-wise adaptive optimizers for training with large batches, used in BERT training.
  • 2023
    Sophia - Second-order Optimizer
    Liu et al. - Uses diagonal Hessian estimation for faster LLM training, potentially challenging Adam's dominance.

🎓 Key Takeaways

Why Adam Works

  • Combines best of both worlds: Momentum from SGD + adaptive rates from RMSProp
  • Bias correction: Ensures stable early training behavior
  • Invariant to scaling: Update magnitudes don't change with gradient rescaling
  • Memory efficient: Only stores two values per parameter
  • Default hyperparameters work well: α=0.001, β₁=0.9, β₂=0.999, ε=1e-8

Adam represents a pivotal moment in optimization for deep learning. Its elegant combination of momentum and adaptive learning rates, along with the crucial insight of bias correction, made it the go-to optimizer for researchers and practitioners alike. While newer optimizers continue to emerge, Adam's influence on the field remains unparalleled.
