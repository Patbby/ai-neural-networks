Adam Optimizer

A Method for Stochastic Optimization

📅 ICLR 2015 👥 Diederik P. Kingma & Jimmy Lei Ba 🏆 100,000+ Citations
📄 PDF 📚 arXiv 💻 PyTorch

🎯 Problem Statement

Before Adam, training deep neural networks required careful tuning of learning rates and was particularly challenging for:

  • Sparse gradients (NLP, vision with embeddings)
  • Non-stationary objectives (online learning, RL)
  • Noisy gradients (mini-batch training)

SGD with momentum helped but still required manual learning rate scheduling. RMSProp adapted per-parameter learning rates but lacked momentum benefits.

Core Question: Can we combine the best of both worlds—adaptive per-parameter learning rates with momentum—while being computationally efficient and memory-friendly?

💡 Key Innovation

Adam (Adaptive Moment Estimation) computes adaptive learning rates for each parameter using estimates of first and second moments of the gradients:

Adam Update Rule:

mt = β₁ · mt-1 + (1 - β₁) · gt     (biased first moment estimate)

vt = β₂ · vt-1 + (1 - β₂) · gt²     (biased second raw moment estimate)

m̂t = mt / (1 - β₁ᵗ)     (bias-corrected first moment)

v̂t = vt / (1 - β₂ᵗ)     (bias-corrected second moment)

θt = θt-1 - α · m̂t / (√v̂t + ε)

The algorithm maintains running averages of both the gradient (momentum) and the squared gradient (adaptive scaling), then corrects for initialization bias.

⚙️ Algorithm Architecture

t First Moment mt = β₁·mt-1 + (1-β₁)·gt (Momentum: β₁ = 0.9) Second Moment vt = β₂·vt-1 + (1-β₂)·gt² (Adaptive: β₂ = 0.999) Bias Correction m̂t = mt/(1-β₁ᵗ) Bias Correction v̂t = vt/(1-β₂ᵗ) θt = θt-1 - α·m̂t/(√v̂t+ε) Default Hyperparameters α (learning rate): 0.001 β₁ (momentum): 0.9 β₂ (RMS decay): 0.999 ε (epsilon): 10⁻⁸ These values work well across most problems! ■ First Moment ■ Second Moment ■ Bias Correction

💻 Code Implementation

PyTorch implementation of Adam optimizer:

import torch

# Adam optimizer - default choice for most deep learning
optimizer = torch.optim.Adam(
    model.parameters(),
    lr=0.001,      # Learning rate (α)
    betas=(0.9, 0.999), # (β₁, β₂)
    eps=1e-8        # Numerical stability
)

# Training loop
for epoch in range(num_epochs):
    optimizer.zero_grad()
    loss = model(inputs)
    loss.backward()
    optimizer.step()  # Applies Adam update

Key hyperparameters and their effects:

  • lr (α): Step size. Larger = faster but unstable. Default 0.001 works well.
  • beta1 (β₁): Momentum decay. 0.9 means 90% previous, 10% new gradient.
  • beta2 (β₂): Second moment decay. 0.999 gives long-term gradient magnitude memory.
  • eps (ε): Prevents division by zero. Usually 1e-8, sometimes 1e-6 for stability.

🔗 Related Work

Adam builds upon and improves several earlier optimization methods:

SGD + Momentum
Polyak (1964)
Accelerates by accumulating velocity, but same LR for all parameters.
AdaGrad
Duchi et al. (2011)
Adapts LR per parameter, but accumulates all squared gradients → LR decays too fast.
RMSProp
Hinton (2012)
Moving average of squared gradients, but no momentum term.
AdamW
Loshchilov & Hutter (2017)
Decouples weight decay from gradient update → better regularization, now preferred for Transformers.
Legacy: Adam's combination of momentum + adaptive learning rates became the template for modern optimizers. AdamW is now the preferred variant for training large models, but Adam remains the default for most practitioners.
