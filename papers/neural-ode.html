🔄 Neural ODE

Neural Ordinary Differential Equations: Continuous-Depth Neural Networks

👤 Ricky T. Q. Chen, Yulia Rubanova, Jesse Bettencourt, David Duvenaud 📅 NeurIPS 2018 (Best Paper Award) 🏢 University of Toronto, Vector Institute
📄 arXiv 📑 PDF 💻 Code 🏆 Best Paper

🎯 Problem Statement

Traditional deep neural networks are built as discrete sequences of layers. Each layer applies a transformation, and information flows through a fixed number of computational steps. This discrete architecture creates several fundamental limitations:

Key Challenges:

• Fixed Depth: The number of layers is a hyperparameter that must be chosen before training

• Memory Cost: Backpropagation requires storing all intermediate activations — O(L) memory for L layers

• Rigid Computation: Every input, regardless of complexity, goes through the same number of layers

• ResNet Approximation: Residual networks are discrete approximations of continuous dynamics

The question arises: What if we could define neural networks as continuous functions of depth, where the hidden state evolves according to a differential equation?

💡 Key Innovation

Neural ODEs parameterize the derivative of the hidden state using a neural network, rather than specifying discrete layers. The hidden state evolves continuously through "infinite depth" using an ODE solver:

dh(t)/dt = f(h(t), t, θ)

Where f is a neural network that defines the instantaneous rate of change of the hidden state. The output is computed by integrating from input time t₀ to output time t₁:

h(t₁) = h(t₀) + ∫t₀t₁ f(h(t), t, θ) dt
Core Breakthroughs:

• Adjoint Sensitivity Method: O(1) constant memory cost for backpropagation through the ODE solver

• Adaptive Computation: The ODE solver automatically adjusts "depth" based on input complexity

• Continuous Normalizing Flows: Change of variables formula simplifies to trace (O(D)) instead of log-determinant (O(D³))

Key Findings:

• Neural ODEs achieve comparable accuracy to ResNets while using constant memory regardless of depth

• The number of function evaluations (NFE) adapts during training — the model automatically increases complexity as it learns

• Backward pass NFE is roughly half the forward pass, making adjoint method more computationally efficient than direct backprop

• Continuous Normalizing Flows enable generative modeling with maximum likelihood training

import torch import torch.nn as nn from torchdiffeq import odeint_adjoint as odeint # Define the ODE function (the neural network for dh/dt) class ODEFunction(nn.Module): def __init__(self, dim): super(ODEFunction, self).__init__() self.net = nn.Sequential( nn.Linear(dim, 50), nn.Tanh(), nn.Linear(50, dim) ) def forward(self, t, y): # Returns f(h(t), t, θ) = dh/dt return self.net(y) # Neural ODE Layer class NeuralODE(nn.Module): def __init__(self, ode_func, solver='dopri5'): super(NeuralODE, self).__init__() self.ode_func = ode_func self.solver = solver def forward(self, z0, t): # Solve ODE: z(t) = z0 + ∫ f(z,t)dt from t[0] to t[-1] # odeint_adjoint uses O(1) memory via adjoint method! return odeint(self.ode_func, z0, t, method=self.solver) # Usage Example dim = 2 ode_func = ODEFunction(dim) model = NeuralODE(ode_func) # Initial state and time points z0 = torch.randn(1, dim) # Initial condition h(t₀) t = torch.linspace(0., 1., 100) # Time points to evaluate # Forward pass - ODE solver computes the trajectory trajectory = model(z0, t) # Shape: [100, 1, 2]
# Continuous Normalizing Flow (CNF) - Generative Modeling class CNF(nn.Module): def __init__(self, dim): super(CNF, self).__init__() self.net = nn.Sequential( nn.Linear(dim + 1, 64), # +1 for time nn.Softplus(), nn.Linear(64, dim) ) def forward(self, t, states): z, logpz = states # z: sample, logpz: log probability with torch.enable_grad(): z.requires_grad_(True) dz_dt = self.net(torch.cat([z, t.unsqueeze(0)], -1)) # Trace of Jacobian for change of variables dlogpz_dt = -trace_df_dz(dz_dt, z) return (dz_dt, dlogpz_dt) # Log probability via change of variables formula # log p(z₁) = log p(z₀) - ∫ tr(∂f/∂z)dt # Much cheaper than O(D³) log-determinant!

🎓 Key Takeaways

  1. Continuous Depth: Neural ODEs treat network depth as a continuous variable, using ODE solvers to compute forward/backward passes
  2. Constant Memory: The adjoint sensitivity method enables O(1) memory backpropagation, regardless of "network depth"
  3. Adaptive Computation: ODE solvers automatically adjust the number of function evaluations based on input complexity
  4. Generative Modeling: Continuous Normalizing Flows enable tractable maximum likelihood training without architectural constraints
  5. Physics Connection: Neural ODEs bridge deep learning with dynamical systems, enabling physics-informed machine learning
