Sequence to Sequence Learning with Neural Networks

The foundational architecture that revolutionized machine translation and sequence modeling

📅 NeurIPS 2014 👤 Ilya Sutskever, Oriol Vinyals, Quoc V. Le 🏢 Google Brain
📄 arXiv:1409.3215 📑 PDF 💻 TensorFlow Code

🎯 The Problem: Sequence-to-Sequence Mapping

Deep Neural Networks (DNNs) excel at classification tasks with fixed input and output dimensions. However, real-world problems often involve sequences of varying lengths:

The Challenge: How do we map an input sequence of arbitrary length (e.g., "Hello world") to an output sequence of potentially different length (e.g., "Bonjour le monde")? Traditional DNNs cannot handle this due to fixed-size vector constraints.

Before Seq2Seq, machine translation relied on complex statistical models with separate components for alignment, translation, and language modeling. These systems were difficult to train and couldn't capture long-range dependencies effectively.

Figure 1: The Sequence-to-Sequence Problem

💡 The Innovation: Encoder-Decoder LSTM

The Seq2Seq model introduces an elegant solution using two LSTM networks: an encoder that compresses the input sequence into a fixed-size vector, and a decoder that generates the output sequence from that vector.

Key Idea: Use one LSTM to read the input sequence and obtain a fixed-dimensional vector representation, then use another LSTM to decode the target sequence from that vector. The LSTM's ability to capture long-range dependencies makes this approach particularly effective.
The Architecture:
• Encoder: Reads input tokens one by one, updating its hidden state. The final hidden state becomes the "thought vector" encoding the entire input.
• Decoder: Uses the thought vector as initial state, generating output tokens sequentially. Each prediction feeds back as the next input (autoregressive generation).
Figure 2: Seq2Seq Architecture with LSTM

🔄 The Trick: Reversing Input Sequence

One of the most surprising findings in the paper was that reversing the order of words in the input sequence dramatically improved performance. This simple trick made a significant difference in translation quality.

Why Reversing Works: Reversing introduces many short-term dependencies between the source and target sentences. For example, the first few words of the reversed source will be close to the first few words of the target, making optimization easier for the LSTM.
Figure 3: Reversing Input Sequence for Better Learning

This simple but effective technique improved BLEU scores significantly and is now standard practice in many sequence-to-sequence models.

💻 PyTorch Implementation

Here's a clean PyTorch implementation of the Seq2Seq architecture:

import torch import torch.nn as nn import random class Encoder(nn.Module): """LSTM Encoder for Seq2Seq""" def __init__(self, input_dim, emb_dim, hid_dim, n_layers, dropout): super().__init__() self.hid_dim = hid_dim self.n_layers = n_layers self.embedding = nn.Embedding(input_dim, emb_dim) self.rnn = nn.LSTM(emb_dim, hid_dim, n_layers, dropout=dropout, batch_first=True) self.dropout = nn.Dropout(dropout) def forward(self, src): # src = [batch_size, src_len] embedded = self.dropout(self.embedding(src)) # embedded = [batch_size, src_len, emb_dim] outputs, (hidden, cell) = self.rnn(embedded) # outputs = [batch_size, src_len, hid_dim] # hidden = [n_layers, batch_size, hid_dim] return hidden, cell class Decoder(nn.Module): """LSTM Decoder for Seq2Seq""" def __init__(self, output_dim, emb_dim, hid_dim, n_layers, dropout): super().__init__() self.output_dim = output_dim self.hid_dim = hid_dim self.n_layers = n_layers self.embedding = nn.Embedding(output_dim, emb_dim) self.rnn = nn.LSTM(emb_dim, hid_dim, n_layers, dropout=dropout, batch_first=True) self.fc_out = nn.Linear(hid_dim, output_dim) self.dropout = nn.Dropout(dropout) def forward(self, input, hidden, cell): # input = [batch_size] (single token) input = input.unsqueeze(1) # [batch_size, 1] embedded = self.dropout(self.embedding(input)) # embedded = [batch_size, 1, emb_dim] output, (hidden, cell) = self.rnn(embedded, (hidden, cell)) # output = [batch_size, 1, hid_dim] prediction = self.fc_out(output.squeeze(1)) # prediction = [batch_size, output_dim] return prediction, hidden, cell class Seq2Seq(nn.Module): """Complete Seq2Seq Model""" def __init__(self, encoder, decoder, device): super().__init__() self.encoder = encoder self.decoder = decoder self.device = device def forward(self, src, trg, teacher_forcing_ratio=0.5): # src = [batch_size, src_len] # trg = [batch_size, trg_len] batch_size = trg.shape[0] trg_len = trg.shape[1] trg_vocab_size = self.decoder.output_dim # Tensor to store decoder outputs outputs = torch.zeros(batch_size, trg_len, trg_vocab_size).to(self.device) # Encoder produces thought vector hidden, cell = self.encoder(src) # First decoder input is <SOS> token input = trg[:, 0] # [batch_size] for t in range(1, trg_len): # Decoder step output, hidden, cell = self.decoder(input, hidden, cell) outputs[:, t] = output # Teacher forcing: use ground truth or prediction teacher_force = random.random() < teacher_forcing_ratio top1 = output.argmax(1) input = trg[:, t] if teacher_force else top1 return outputs

Training configuration from the paper:

# Hyperparameters from the paper INPUT_DIM = 38000 # Source vocabulary size OUTPUT_DIM = 38000 # Target vocabulary size ENC_EMB_DIM = 1000 # Embedding dimension DEC_EMB_DIM = 1000 HID_DIM = 1000 # LSTM hidden dimension N_LAYERS = 4 # 4-layer deep LSTM ENC_DROPOUT = 0.2 DEC_DROPOUT = 0.2 # Initialize model enc = Encoder(INPUT_DIM, ENC_EMB_DIM, HID_DIM, N_LAYERS, ENC_DROPOUT) dec = Decoder(OUTPUT_DIM, DEC_EMB_DIM, HID_DIM, N_LAYERS, DEC_DROPOUT) model = Seq2Seq(enc, dec, device).to(device) # Optimizer: SGD with gradient clipping optimizer = torch.optim.SGD(model.parameters(), lr=0.7) def clip_gradients(model, max_norm=5): torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm) # Loss function criterion = nn.CrossEntropyLoss(ignore_index=PAD_IDX)

🔑 Key Insights

1. End-to-end learning is powerful: Unlike previous approaches that required complex pipelines, Seq2Seq showed that a single neural network can learn to translate directly from raw text pairs.

2. Deep LSTMs work better: The paper used 4-layer deep LSTMs (1000 cells per layer) and found that increasing depth improved performance significantly compared to shallow models.

3. Reversing helps optimization: The simple trick of reversing the input sequence introduces short-term dependencies that make gradient flow easier during training.

4. LSTMs capture meaning: The model learned to represent sentences as fixed-size vectors that preserved semantic information, showing sensitivity to word order and invariance to voice changes.

Concurrent work on neural machine translation using GRU-based encoder-decoder with attention.

Predecessor

Long Short-Term Memory (Hochreiter & Schmidhuber, 1997)

The foundational recurrent architecture that made Seq2Seq possible by solving vanishing gradients.

Successor

Neural Machine Translation with Attention (Bahdanau et al., 2015)

Introduced attention mechanism, allowing the decoder to focus on different encoder states.

Successor

Replaced recurrence entirely with self-attention, achieving parallelization and better performance.

Successor

Google Neural Machine Translation (Wu et al., 2016)

Production-scale Seq2Seq system deployed at Google, translating billions of sentences daily.

Application

Neural Conversational Model (Vinyals & Le, 2015)

Applied Seq2Seq to dialogue generation, showing versatility beyond translation.

📝 Summary

Seq2Seq demonstrated that a single deep LSTM architecture could achieve state-of-the-art results in machine translation, outperforming complex statistical systems. The encoder-decoder framework, combined with the input reversal trick, established a new paradigm for sequence modeling.

The impact of Seq2Seq extends throughout modern deep learning:

  • Machine Translation (Google Translate, DeepL)
  • Text Summarization (news summarization, abstract generation)
  • Conversational AI (chatbots, virtual assistants)
  • Speech Recognition (sequence-to-sequence ASR)
  • Code Generation (GitHub Copilot, CodeT5)
  • Image Captioning (vision + language models)
Bottom Line: Seq2Seq proved that neural networks can directly learn complex sequence transformations end-to-end, paving the way for the attention revolution and modern transformer-based language models.
