Sequence to Sequence Learning with Neural Networks

Ilya Sutskever, Oriol Vinyals, Quoc V. Le
Google Research • NIPS 2014 • arXiv:1409.3215
📄 Paper 📥 PDF 💻 Code

🎯 Core Innovation

Seq2Seq introduced the encoder-decoder architecture for end-to-end sequence learning using LSTMs. The encoder compresses a variable-length input sequence into a fixed-size vector representation (thought vector), and the decoder generates a variable-length output sequence from this vector. This revolutionary approach eliminated the need for complex feature engineering in machine translation and established the foundation for modern neural NLP.

📖 Problem Statement

Before Seq2Seq, mapping sequences to sequences required complex pipelines with separate components for alignment, translation models, and language models. Deep neural networks excelled at fixed-size input/output problems (like image classification), but struggled with variable-length sequences common in language.

The key challenges were:

  • Variable input and output lengths (sentences have different lengths)
  • Long-term dependencies (words at the start affect words at the end)
  • Alignment between source and target sequences
  • End-to-end learning without handcrafted features

⚡ Key Innovations

1. Encoder-Decoder Architecture

The paper proposed using two separate LSTM networks: one to encode the input sequence into a dense vector representation, and another to decode this vector into the output sequence.

2. Reversing the Input Sequence

One of the most surprising findings was that reversing the order of words in the input sequence (but not the output) significantly improved performance. This simple trick introduces many short-term dependencies between source and target, making the optimization problem easier for the LSTM.

3. Deep LSTM Architecture

The paper used 4-layer deep LSTMs with 1000 cells per layer. The deep architecture proved crucial for performance - shallower networks performed significantly worse. This was one of the early demonstrations that depth matters for sequence modeling.

p(y₁, ..., yₜ' | x₁, ..., xₜ) = ∏t=1T' p(yₜ | v, y₁, ..., yₜ₋₁)

The decoder generates each output token conditioned on the context vector v and previous outputs

📊 Results & Impact

🏆 BLEU Score: 34.8

Achieved 34.8 BLEU on WMT'14 English-to-French, surpassing phrase-based SMT systems (33.3) without complex feature engineering.

🔄 End-to-End Learning

Eliminated need for separate alignment, translation, and language models. One neural network learned everything.

📈 Long Sentences

Unlike previous neural approaches, the LSTM had no difficulty with long sentences due to the thought vector compression.

🌐 General Framework

The same architecture works for any sequence-to-sequence task: summarization, dialogue, code generation, etc.

💻 PyTorch Implementation

import torch
import torch.nn as nn

class Seq2Seq(nn.Module):
    def __init__(self, input_dim, output_dim, hidden_dim, n_layers=4):
        super().__init__()
        
        # Encoder: 4-layer deep LSTM
        self.encoder = nn.LSTM(
            input_dim, hidden_dim, 
            n_layers, 
            batch_first=True
        )
        
        # Decoder: 4-layer deep LSTM
        self.decoder = nn.LSTM(
            output_dim, hidden_dim,
            n_layers,
            batch_first=True
        )
        
        # Output projection
        self.fc_out = nn.Linear(hidden_dim, output_dim)
    
    def forward(self, src, trg):
        # src: (batch, src_len, input_dim)
        # Encoder: compress to thought vector
        _, (hidden, cell) = self.encoder(src)
        
        # Decoder: generate output sequence
        outputs, _ = self.decoder(trg, (hidden, cell))
        
        # Project to vocabulary
        predictions = self.fc_out(outputs)
        return predictions

# Key trick: Reverse input sequence
def reverse_sequence(seq):
    return torch.flip(seq, dims=[1])  # Reverse along sequence dimension

🔗 Related Work

  • → Transformer (2017): Replaced LSTMs with attention mechanisms, building on the Seq2Seq architecture
  • → BERT (2018): Bidirectional encoder using the Transformer architecture
  • → LSTM (1997): The underlying architecture that enabled Seq2Seq's success
  • → Neural Machine Translation with Attention (2015): Added attention to Seq2Seq

🎯 Why It Matters

The Seq2Seq paper was a watershed moment in NLP history. It demonstrated that neural networks could directly learn to map sequences to sequences without hand-crafted features or complex pipelines. This work:

  • Established the encoder-decoder paradigm that dominates NLP today
  • Showed that deep LSTMs (4 layers) significantly outperform shallow ones
  • Introduced the "thought vector" concept - a fixed-size representation of meaning
  • Proved neural MT could beat traditional statistical MT
  • Led directly to attention mechanisms and the Transformer

Every modern NLP system—from Google Translate to ChatGPT—traces its lineage back to this paper's core insight: compress, then generate.
