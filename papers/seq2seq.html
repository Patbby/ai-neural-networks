Sequence to Sequence Learning with Neural Networks

NeurIPS 2014 · December 2014
Ilya Sutskever, Oriol Vinyals, Quoc V. Le
Google Brain, Mountain View, CA
📄 arXiv 📥 PDF 💻 Code 🔥 25,000+ Citations

🎯 The Breakthrough

Before 2014, neural networks excelled at fixed-input problems like image classification, but struggled with variable-length sequence-to-sequence mapping. Statistical machine translation (SMT) dominated, using complex feature engineering and n-gram models. Seq2Seq proved that a simple end-to-end neural approach could beat these sophisticated systems.

The Seq2Seq Proposal

Use one deep LSTM to read the input sequence into a fixed-dimensional vector, then another deep LSTM to decode the target sequence from that vector. This elegant approach became the foundation for neural machine translation and, ultimately, modern large language models.

❌ The Problem Before Seq2Seq

Deep Neural Networks Were Limited

• DNNs require fixed-size input and output vectors — unsuitable for sequences

• Traditional RNNs could theoretically handle sequences, but struggled with long-range dependencies

• Statistical Machine Translation required extensive feature engineering and separate components

• No end-to-end learning approach existed for sequence transduction tasks

💡 Core Innovation

Three Key Ideas

1. Encoder-Decoder LSTM Architecture: Two separate LSTMs — one compresses input to a thought vector, the other expands it to output

2. Deep 4-Layer LSTMs: Using depth (4 layers) to increase model capacity without exploding computational cost

3. Reverse Input Sequences: Reversing the order of words in source sentences introduced short-term dependencies that dramatically improved optimization

🏗️ Seq2Seq Architecture

Encoder-Decoder LSTM Architecture

The Reversal Trick

The authors discovered that reversing the order of words in the input sequence dramatically improved performance. Why? Because the first few words in the output now depend on the first few words in the (reversed) input, creating short-term dependencies that LSTMs handle much better than long-term ones.

Original: "The cat sat on the mat" → "Le chat était assis sur le tapis"
Reversed: "mat the on sat cat The" → "Le chat était assis sur le tapis"
Result: +5 BLEU points improvement!

Mathematical Formulation

The model computes the conditional probability of the output sequence given the input:

p(y₁, ..., yₜ′ | x₁, ..., xₜ) = ∏t=1T′ p(yₜ | v, y₁, ..., yₜ₋₁)

Where v is the final hidden state of the encoder LSTM — the "thought vector" containing the compressed meaning of the entire input sentence.

💻 PyTorch Implementation

# Seq2Seq with LSTM Encoder-Decoder
import torch
import torch.nn as nn

class Seq2Seq(nn.Module):
    def __init__(self, input_dim, output_dim, emb_dim=256, 
                 hid_dim=512, n_layers=4, dropout=0.5):
        super().__init__()
        
        self.hid_dim = hid_dim
        self.n_layers = n_layers
        
        # Encoder
        self.encoder_emb = nn.Embedding(input_dim, emb_dim)
        self.encoder = nn.LSTM(emb_dim, hid_dim, n_layers, 
                               dropout=dropout, batch_first=True)
        
        # Decoder
        self.decoder_emb = nn.Embedding(output_dim, emb_dim)
        self.decoder = nn.LSTM(emb_dim, hid_dim, n_layers,
                               dropout=dropout, batch_first=True)
        self.fc_out = nn.Linear(hid_dim, output_dim)
        self.dropout = nn.Dropout(dropout)
    
    def forward(self, src, trg, teacher_forcing_ratio=0.5):
        # src: [batch_size, src_len]
        # trg: [batch_size, trg_len]
        
        batch_size = src.shape[0]
        trg_len = trg.shape[1]
        trg_vocab_size = self.fc_out.out_features
        
        # Tensor to store decoder outputs
        outputs = torch.zeros(batch_size, trg_len, trg_vocab_size).to(src.device)
        
        # Encode source sequence (REVERSED input!)
        src_embedded = self.dropout(self.encoder_emb(src))
        _, (hidden, cell) = self.encoder(src_embedded)
        # hidden, cell: [n_layers, batch_size, hid_dim]
        
        # First decoder input is <sos> token
        input_token = trg[:, 0]  # [batch_size]
        
        for t in range(1, trg_len):
            # Decode one step
            input_embedded = self.dropout(self.decoder_emb(input_token.unsqueeze(1)))
            # input_embedded: [batch_size, 1, emb_dim]
            
            output, (hidden, cell) = self.decoder(input_embedded, (hidden, cell))
            # output: [batch_size, 1, hid_dim]
            
            prediction = self.fc_out(output.squeeze(1))
            # prediction: [batch_size, output_dim]
            
            outputs[:, t, :] = prediction
            
            # Teacher forcing: decide next input
            teacher_force = torch.rand(1).item() < teacher_forcing_ratio
            top1 = prediction.argmax(1)
            input_token = trg[:, t] if teacher_force else top1
        
        return outputs

# Hyperparameters matching the paper
INPUT_DIM = 160000  # Source vocabulary size
OUTPUT_DIM = 80000   # Target vocabulary size
EMB_DIM = 256         # Embedding dimension
HID_DIM = 1000        # Hidden dimension (1000 cells)
N_LAYERS = 4          # Number of LSTM layers

model = Seq2Seq(INPUT_DIM, OUTPUT_DIM, EMB_DIM, HID_DIM, N_LAYERS)

🌟 Impact & Legacy

Seq2Seq pioneered the end-to-end neural approach to sequence modeling and became the foundation for modern NLP:

  • Neural Machine Translation (2014-2016): Replaced statistical MT systems at Google
  • Attention Mechanism (2015): Bahdanau et al. added attention to Seq2Seq, solving the bottleneck problem
  • Transformer (2017): "Attention Is All You Need" removed recurrence entirely
  • Google Neural MT (2016): Deployed Seq2Seq-based GNMT for production translation
  • Conversational AI: Neural Responding Machine (2015), Google Neural Conversational Model
  • Speech Recognition: Listen, Attend and Spell (LAS) for end-to-end ASR

The Seq2Seq Legacy

This paper established the encoder-decoder paradigm that persists today. The idea of compressing input into a "thought vector" and then expanding it — now implemented with transformers instead of LSTMs — remains the core architecture behind ChatGPT, Claude, and virtually all modern language models.

📚 Related Work & Follow-ups

RNN Encoder-Decoder (Cho et al., 2014)

Introduced the encoder-decoder architecture with GRU units, concurrent with Seq2Seq but using different units

Neural Machine Translation with Attention (Bahdanau et al., 2015)

Added the attention mechanism to Seq2Seq, allowing the decoder to focus on different parts of the input

Google's Neural Machine Translation (Wu et al., 2016)

Production-scale Seq2Seq system with 8 encoder + 8 decoder layers, achieving near-human translation quality

Attention Is All You Need (Vaswani et al., 2017)

The Transformer — removed recurrence entirely, using only attention mechanisms

🔍 Key Insights

Why Seq2Seq Mattered

1. Simplicity: No complex feature engineering, no alignment models — just LSTMs

2. End-to-End: Single neural network trained directly to optimize translation quality

3. Generalization: Same architecture works for machine translation, summarization, dialogue, and more

4. Scalability: Deep LSTMs (4 layers) with large hidden states (1000 cells) proved depth matters

5. The Bottleneck: The "thought vector" limitation directly motivated the attention mechanism
