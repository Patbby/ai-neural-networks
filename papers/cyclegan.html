CycleGAN

Unpaired Image-to-Image Translation using Cycle-Consistent Adversarial Networks

Jun-Yan Zhu, Taesung Park, Phillip Isola, Alexei A. Efros

ICCV 2017 • Berkeley AI Research (BAIR)
📄 arXiv 🏠 Project Page 💻 Code 📥 PDF

🎯 Problem Statement

Before CycleGAN, image-to-image translation methods like Pix2Pix required paired training data — thousands of aligned image pairs (input and output). However, obtaining paired data is often:

Expensive

Collecting paired datasets requires manual annotation or specialized equipment, making it cost-prohibitive for many applications.

Time-Consuming

Creating paired examples for tasks like "horse → zebra" or "summer → winter" requires capturing the same scene in both conditions.

Impossible

Some translations simply cannot be paired: artistic styles, medical imaging modalities, or photo enhancement from old photos.

The Key Question

"Can we learn to translate images between domains without paired examples, using only unpaired data from two distinct domains?"

💡 Key Innovation: Cycle Consistency

CycleGAN introduces the concept of cycle consistency — if we translate an image from domain X to Y and back to X, we should arrive at the original image. This simple but powerful idea enables learning without paired supervision.

Cycle Consistency Loss
Lcyc(G, F) = Ex[||F(G(x)) − x||1] + Ey[||G(F(y)) − y||1]

Where:

  • G: Generator mapping X → Y (e.g., horse → zebra)
  • F: Generator mapping Y → X (e.g., zebra → horse)
  • DX, DY: Discriminators for domains X and Y
  • ||·||1: L1 loss ensuring reconstruction fidelity

🏗️ Architecture

CycleGAN consists of two generators and two discriminators arranged in a cyclic configuration. This architecture enables bidirectional translation while maintaining content consistency.

Architecture Components
  • Generator Network: Uses ResNet-based architecture with 6-9 residual blocks. Contains encoder (convolutional), transformer (residual blocks), and decoder (fractionally-strided convolutions).
  • Discriminator Network: 70×70 PatchGAN architecture that classifies whether overlapping image patches are real or fake, enabling translation of arbitrarily-sized images.
  • Skip Connections: U-Net style skip connections between encoder and decoder for better gradient flow and content preservation.
  • Instance Normalization: Used instead of batch normalization for better results in style transfer tasks.

📊 Loss Functions

CycleGAN optimizes a combined objective with three loss components:

Total Loss
L(G, F, DX, DY) = LGAN(G, DY) + LGAN(F, DX) + λLcyc(G, F)

Adversarial Loss

Standard GAN loss ensuring generated images match the target domain distribution.

LGAN(G, DY) = Ey[log DY(y)] + Ex[log(1 − DY(G(x)))]

Cycle Consistency Loss

Ensures forward-backward translation reconstructs the original image.

Lcyc = ||F(G(x)) − x||1 + ||G(F(y)) − y||1

Identity Loss (Optional)

Encourages generators to preserve input when it's already in the target domain.

Lidentity = ||G(y) − y||1 + ||F(x) − x||1

Key Achievements
  • No Paired Data: First general-purpose method for unpaired image-to-image translation, eliminating the need for expensive paired datasets.
  • Qualitative Results: Successfully demonstrated on diverse tasks: horses↔zebras, summer↔winter, Monet↔photos, semantic labels↔photos.
  • Quantitative Evaluation: Used perceptual studies (AMT) to show generated images are often indistinguishable from real target domain images.
  • FCN Score: Achieved competitive or better semantic segmentation scores compared to paired methods on photo→label tasks.
  • Training Stability: Cycle consistency provides a strong training signal that stabilizes GAN training without mode collapse.
Why It Matters

CycleGAN democratized image-to-image translation by removing the paired data bottleneck. It enabled researchers and practitioners to tackle translation tasks that were previously impossible due to data constraints — from medical imaging to artistic style transfer. The cycle consistency principle has since been adopted in countless follow-up works across vision, NLP, and reinforcement learning.

import torch import torch.nn as nn # Residual Block for Generator class ResidualBlock(nn.Module): def __init__(self, channels): super(ResidualBlock, self).__init__() self.conv = nn.Sequential( nn.ReflectionPad2d(1), nn.Conv2d(channels, channels, 3), nn.InstanceNorm2d(channels), nn.ReLU(inplace=True), nn.ReflectionPad2d(1), nn.Conv2d(channels, channels, 3), nn.InstanceNorm2d(channels) ) def forward(self, x): return x + self.conv(x) # Skip connection (residual) # Generator with ResNet architecture class Generator(nn.Module): def __init__(self, in_channels=3, out_channels=3, n_residual=9): super(Generator, self).__init__() # Initial convolution model = [nn.ReflectionPad2d(3), nn.Conv2d(in_channels, 64, 7), nn.InstanceNorm2d(64), nn.ReLU(inplace=True)] # Downsampling in_features = 64 out_features = in_features * 2 for _ in range(2): model += [nn.Conv2d(in_features, out_features, 3, stride=2, padding=1), nn.InstanceNorm2d(out_features), nn.ReLU(inplace=True)] in_features = out_features out_features = in_features * 2 # Residual blocks for _ in range(n_residual): model += [ResidualBlock(in_features)] # Upsampling out_features = in_features // 2 for _ in range(2): model += [nn.ConvTranspose2d(in_features, out_features, 3, stride=2, padding=1, output_padding=1), nn.InstanceNorm2d(out_features), nn.ReLU(inplace=True)] in_features = out_features out_features = in_features // 2 # Output layer model += [nn.ReflectionPad2d(3), nn.Conv2d(64, out_channels, 7), nn.Tanh()] self.model = nn.Sequential(*model) def forward(self, x): return self.model(x) # Cycle Consistency Loss def cycle_consistency_loss(real, reconstructed): return torch.mean(torch.abs(real - reconstructed)) # L1 loss # Training Step def train_step(G_XY, G_YX, D_X, D_Y, real_X, real_Y, opt_G, opt_D, criterion_GAN, lambda_cyc=10): # Adversarial loss fake_Y = G_XY(real_X) loss_GAN_XY = criterion_GAN(D_Y(fake_Y), torch.ones_like(fake_Y)) fake_X = G_YX(real_Y) loss_GAN_YX = criterion_GAN(D_X(fake_X), torch.ones_like(fake_X)) # Cycle consistency loss rec_X = G_YX(fake_Y) loss_cyc_X = cycle_consistency_loss(real_X, rec_X) rec_Y = G_XY(fake_X) loss_cyc_Y = cycle_consistency_loss(real_Y, rec_Y) # Total generator loss loss_G = loss_GAN_XY + loss_GAN_YX + lambda_cyc * (loss_cyc_X + loss_cyc_Y) opt_G.zero_grad() loss_G.backward() opt_G.step() return loss_G.item()

🎨 Applications

Style Transfer

Convert photos to artistic styles (Monet, Van Gogh, Cézanne, Ukiyo-e) without paired paintings.

Season Transfer

Transform summer photos to winter landscapes, enabling synthetic data generation for autonomous driving.

Object Transfiguration

Change object categories in images: horses ↔ zebras, apples ↔ oranges, cats ↔ dogs.

Medical Imaging

Translate between MRI and CT scans, or between different contrast agents, without paired patient scans.

Photo Enhancement

Improve photo quality: smartphone → DSLR, low-res → high-res, old photo restoration.

Game Graphics

Transfer between real photos and game renderings (GTA5 ↔ Cityscapes) for domain adaptation.

🎯 Key Takeaways

  • Cycle Consistency Principle: The insight that round-trip translation should reconstruct the original provides a powerful training signal without paired data.
  • Dual Generator-Discriminator: The symmetric architecture with two generators and two discriminators enables bidirectional learning and mutual regularization.
  • Practical Impact: CycleGAN made image-to-image translation accessible for countless applications where paired data collection was impossible.
  • Architectural Choices: ResNet generators with skip connections and PatchGAN discriminators proved crucial for high-quality results.
  • Broad Influence: The cycle consistency concept has been applied far beyond vision — to NLP, speech, and reinforcement learning domains.
