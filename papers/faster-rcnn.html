Faster R-CNN

Towards Real-Time Object Detection with Region Proposal Networks

Shaoqing Ren, Kaiming He, Ross Girshick, Jian Sun | NeurIPS 2015 | Microsoft Research

📄 Paper (arXiv) 💻 Code (Python) 💻 Code (MATLAB) 🏆 COCO/ILSVRC 2015 Winner

Problem Statement

State-of-the-art object detection networks depended on region proposal algorithms (like Selective Search) to hypothesize object locations. While advances like SPPnet and Fast R-CNN had reduced detection network running time, the region proposal computation became the bottleneck — taking 1-2 seconds per image on CPU while the detection network ran efficiently on GPU.

The Core Question: Can we create a unified, end-to-end trainable object detection system where region proposal generation is nearly cost-free by sharing convolutional features with the detection network?

Existing methods like Selective Search (2s per image) and EdgeBoxes (0.2s per image) were implemented on CPU, creating a fundamental speed mismatch with GPU-accelerated detection networks.

Key Innovation

  • Region Proposal Network (RPN)

    A fully convolutional network that simultaneously predicts object bounds and objectness scores at each position, sharing full-image convolutional features with the detection network for nearly cost-free proposals.

  • Anchor Boxes

    Novel translation-invariant reference boxes serving as regression references at multiple scales (128², 256², 512²) and aspect ratios (1:1, 1:2, 2:1), eliminating the need for image pyramids or filter pyramids.

  • Unified Network with "Attention"

    RPN and Fast R-CNN merged into a single network where RPN tells the unified network "where to look" — implementing a neural attention mechanism for object detection.

Architecture

Faster R-CNN consists of two modules: (1) a deep fully convolutional Region Proposal Network (RPN) that proposes regions, and (2) the Fast R-CNN detector that uses the proposed regions.

Anchor Mechanism: At each sliding-window position, the network predicts 9 anchors (3 scales × 3 aspect ratios). For a feature map of size W×H, there are W×H×k anchors total. Translation-invariant anchors enable efficient multi-scale detection without image pyramids.

Code Implementation

PyTorch-style Faster R-CNN Forward Pass
# Shared convolutional feature extraction
def forward(self, image):
    # Extract features using VGG/ResNet backbone
    features = self.backbone(image)  # [B, C, H, W]
    
    # Region Proposal Network
    rpn_cls, rpn_reg = self.rpn(features)
    # rpn_cls: [B, 2k, H, W] - objectness scores
    # rpn_reg: [B, 4k, H, W] - box regression
    
    # Generate proposals from anchors
    proposals = self.generate_proposals(
        rpn_cls, rpn_reg, anchors, 
        pre_nms_top_n=12000,
        post_nms_top_n=2000,
        nms_thresh=0.7
    )
    
    # RoI Pooling
    pooled_features = self.roi_pool(features, proposals)
    # [num_proposals, C, 7, 7]
    
    # Fast R-CNN head
    fc_features = self.fc_layers(pooled_features)
    cls_scores = self.cls_layer(fc_features)   # Classification
    bbox_preds = self.bbox_layer(fc_features)  # Box refinement
    
    return cls_scores, bbox_preds, proposals

# Anchor generation (k=9: 3 scales × 3 ratios)
scales = [128, 256, 512]      # Box areas
aspect_ratios = [0.5, 1.0, 2.0]  # 1:2, 1:1, 2:1
Multi-task Loss Function
# RPN Loss (classification + regression)
L_rpn = L_cls(p_i, p_i_star) + λ * p_i_star * L_reg(t_i, t_i_star)

# Where:
# p_i: predicted probability of anchor i being object
# p_i_star: ground-truth label (1=positive, 0=negative)
# t_i: predicted box coordinates (4-d)
# t_i_star: ground-truth box coordinates
# λ = 10: balancing parameter

# Positive/negative assignment:
# Positive: IoU > 0.7 with any GT box OR highest IoU with GT
# Negative: IoU < 0.3 with all GT boxes
# Ignored: 0.3 ≤ IoU ≤ 0.7

Related Work

R-CNN: Rich Feature Hierarchies for Accurate Object Detection

The foundational work that introduced region-based CNNs for object detection. Used Selective Search for proposals and SVMs for classification.

Girshick et al., 2014

Fast R-CNN

Introduced single-stage training, shared convolutional features, and RoI pooling. Formed the detection component of Faster R-CNN.

Girshick, ICCV 2015

Mask R-CNN

Extended Faster R-CNN for instance segmentation by adding a mask prediction branch alongside the existing box and class outputs.

He et al., ICCV 2017

YOLO: You Only Look Once

Single-stage detector that abandoned the two-stage approach entirely, achieving even faster speeds with unified prediction.

Redmon et al., CVPR 2016

FPN: Feature Pyramid Networks

Enhanced object detection by building multi-scale feature pyramids, further improving small object detection capabilities.

Lin et al., CVPR 2017
