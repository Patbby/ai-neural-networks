Faster R-CNN

Towards Real-Time Object Detection with Region Proposal Networks

📅 NeurIPS 2015 👤 Ren, He, Girshick, Sun 🏛️ Microsoft Research
📄 arXiv 📑 PDF 💻 Code

🎯 Problem Statement

Before Faster R-CNN, state-of-the-art object detection systems relied on region proposal algorithms like Selective Search to hypothesize object locations. While advances like SPPnet and Fast R-CNN had reduced detection network running time, region proposal computation emerged as a critical bottleneck.

The Bottleneck: Traditional region proposal methods (Selective Search, EdgeBoxes) took 2-3 seconds per image, making real-time object detection impossible even with fast detection networks.

Previous approaches suffered from:

  • Computational redundancy: Region proposal and detection networks processed the same image independently
  • Slow proposal generation: CPU-based algorithms couldn't leverage GPU acceleration
  • Disjoint training: Multi-stage pipelines were complex to train and optimize

💡 Key Innovation

Faster R-CNN introduced the Region Proposal Network (RPN) — a fully convolutional network that shares full-image convolutional features with the detection network, enabling nearly cost-free region proposals.

Core Insight: By sharing convolutional features between region proposal and object detection, both tasks can be trained end-to-end in a single unified network. The RPN acts as the "attention" mechanism that tells the network where to look.

Key Technical Contributions:

  • Anchor Boxes: Multi-scale, multi-aspect-ratio reference boxes for efficient proposal generation
  • Translation-Invariant Anchors: Same detection approach works at any image position
  • 4-Step Alternating Training: Shared convolutional features between RPN and Fast R-CNN
  • Unified Network: Single model for both region proposal and object detection

🏗️ Architecture

Faster R-CNN consists of two modules: (1) a deep fully convolutional Region Proposal Network (RPN) that proposes regions, and (2) the Fast R-CNN detector that uses the proposed regions.

Architecture Details:

  • Backbone: VGG-16 or ResNet-101 for feature extraction
  • RPN: 3×3 sliding window over feature maps → 1×1 conv for cls/reg
  • Anchors: 9 anchors per location (3 scales × 3 aspect ratios)
  • ROI Pooling: Extracts fixed-size features from variable proposals
  • Head: Fully connected layers for final classification and bounding box refinement
RPN Multi-Task Loss = Lcls(pi, pi*) + λ Lreg(ti, ti*)

Where pi is predicted objectness, ti is predicted box coordinates, and * denotes ground truth

  • Real-time capability: 5 fps including all steps on a single GPU (vs. 0.5 fps for Fast R-CNN + Selective Search)

  • Competition wins: Foundation of 1st-place entries in ILSVRC and COCO 2015 competitions
Speed Comparison: Faster R-CNN achieved 10× speedup over Fast R-CNN + Selective Search, reducing region proposal time from ~2s to just 10ms while maintaining or improving accuracy.

Industry Impact: The RPN architecture became the foundation for:

  • Mask R-CNN (instance segmentation)
  • Faster object detectors (FPN, RetinaNet)
  • Video analysis pipelines
  • Autonomous driving perception systems

💻 PyTorch Implementation

A simplified implementation of the RPN component in PyTorch:

import torch
import torch.nn as nn

class RegionProposalNetwork(nn.Module):
    """
    Region Proposal Network (RPN) for Faster R-CNN
    """
    def __init__(self, in_channels=512, num_anchors=9):
        super().__init__()
        
        # Intermediate layer
        self.conv = nn.Conv2d(in_channels, 512, kernel_size=3, padding=1)
        
        # Classification head: objectness score
        self.cls_layer = nn.Conv2d(512, num_anchors * 2, kernel_size=1)
        
        # Regression head: box coordinates
        self.reg_layer = nn.Conv2d(512, num_anchors * 4, kernel_size=1)
        
        self.relu = nn.ReLU(inplace=True)
    
    def forward(self, features):
        """
        Args:
            features: Feature maps from backbone (N, C, H, W)
        Returns:
            cls_scores: Objectness scores (N, num_anchors*2, H, W)
            bbox_preds: Box predictions (N, num_anchors*4, H, W)
        """
        x = self.relu(self.conv(features))
        
        # Predict objectness (object vs. background)
        cls_scores = self.cls_layer(x)
        
        # Predict box deltas (dx, dy, dw, dh)
        bbox_preds = self.reg_layer(x)
        
        return cls_scores, bbox_preds


# Anchor generation
def generate_anchors(base_size=16, ratios=[0.5, 1, 2], 
                     scales=[8, 16, 32]):
    """Generate anchor boxes for RPN"""
    anchors = []
    for ratio in ratios:
        h = base_size * torch.sqrt(ratio)
        w = base_size / torch.sqrt(ratio)
        for scale in scales:
            anchors.append([0, 0, w * scale, h * scale])
    return torch.tensor(anchors)  # 9 anchors


# Full Faster R-CNN model structure
class FasterRCNN(nn.Module):

            nn.Dropout(0.5),
            nn.Linear(4096, 4096),
            nn.ReLU(inplace=True),
            nn.Dropout(0.5)
        )
        
        # Classification and regression heads
        self.cls_score = nn.Linear(4096, num_classes)
        self.bbox_pred = nn.Linear(4096, num_classes * 4)
    
    def forward(self, images, targets=None):
        # Extract features
        features = self.backbone(images)
        
        # Generate proposals with RPN
        rpn_cls, rpn_reg = self.rpn(features)
        proposals = self.generate_proposals(rpn_cls, rpn_reg)
        
        # ROI pooling and detection
        roi_features = self.roi_pool(features, proposals)
        roi_features = roi_features.view(roi_features.size(0), -1)
        
        fc_out = self.fc(roi_features)
        
        # Final predictions
        class_logits = self.cls_score(fc_out)
        bbox_deltas = self.bbox_pred(fc_out)
        
        return class_logits, bbox_deltas, proposals
