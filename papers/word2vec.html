🎯 Abstract & Overview

Word2Vec revolutionized natural language processing by introducing a simple yet powerful way to represent words as dense, low-dimensional vectors. Before this work, words were typically represented as one-hot vectors (sparse, high-dimensional) or using count-based methods that failed to capture semantic relationships.

The Core Insight

Words that appear in similar contexts tend to have similar meanings. By training a neural network to predict either context words from a target word (Skip-gram) or target words from context (CBOW), Word2Vec learns vector representations where semantic relationships are encoded as geometric relationships.

🏗️ Architecture: Two Flavors

Word2Vec introduces two model architectures that learn word embeddings through different prediction tasks:

CBOW: Continuous Bag of Words

Predict the target word from its surrounding context

Skip-gram

Predict context words from the target word (works well with large datasets)

✨ Key Innovations

  • Dense Word Embeddings Replaced sparse one-hot vectors (|V| dimensions) with dense vectors (100-300 dimensions). This compression captures semantic meaning while enabling efficient computation.
  • Semantic Relationships in Vector Space The geometric arrangement of word vectors encodes semantic relationships. Similar words cluster together, and relationships manifest as consistent vector offsets.
  • Efficient Training Techniques Hierarchical Softmax reduces complexity from O(|V|) to O(log|V|) using a binary tree. Negative Sampling further accelerates by updating only a small sample of negative examples.
  • The Famous Vector Arithmetic The most celebrated result: vector operations capture linguistic relationships. king - man + woman ≈ queen demonstrates that semantic relationships are linear translations in vector space.

📐 Mathematical Formulation

Objective Functions

CBOW: (1/T) Σt=1T log P(wt | wt-c, ..., wt-1, wt+1, ..., wt+c)
Skip-gram: (1/T) Σt=1T Σ-c≤j≤c,j≠0 log P(wt+j | wt)

Where c is the training context window size, T is the number of words in the corpus, and the conditional probability is defined using the softmax function:

P(wO | wI) = exp(v'wOT · vwI) / Σw=1W exp(v'wT · vwI)

Hierarchical Softmax

To avoid the expensive normalization over the entire vocabulary, Word2Vec uses a binary Huffman tree:

p(w | wI) = Πj=1L(w) σ([n(w,j+1) = ch(n(w,j))] · v'n(w,j)T · vwI)

This reduces training complexity from O(|V|) to O(log|V|), making it feasible to train on billion-word corpora.

Negative Sampling (NEG)

An even simpler alternative that replaces the log P(wO|wI) term with:

log σ(v'wOT · vwI) + Σi=1k Ewi~Pn(w)[log σ(-v'wiT · vwI)]

Instead of updating all output vectors, only the target word and k randomly sampled negative examples are updated.

from gensim.models import Word2Vec

# Prepare corpus (list of tokenized sentences)

# PyTorch implementation sketch
import torch.nn as nn

class SkipGram(nn.Module):
    def __init__(self, vocab_size, embedding_dim):
        super().__init__()
        self.embeddings = nn.Embedding(vocab_size, embedding_dim)
        self.output = nn.Embedding(vocab_size, embedding_dim)
    
    def forward(self, target, context):
        # target: (batch_size,)
        # context: (batch_size,)
        target_emb = self.embeddings(target)      # (batch, embed_dim)
        context_emb = self.output(context)        # (batch, embed_dim)
        
        # Dot product for similarity score
        score = torch.sum(target_emb * context_emb, dim=1)
        return torch.sigmoid(score)

🌟 Impact & Legacy

Word2Vec fundamentally changed how we represent text in machine learning:

  • GloVe (2014): Global Vectors combined count-based and prediction-based methods
  • FastText (2016): Facebook's extension using subword information for rare words
  • ELMo (2018): Deep contextualized word representations with bidirectional LSTMs
  • BERT (2018): Contextual embeddings using Transformers — the next evolution
  • Modern LLMs: Word2Vec's insight about dense representations lives on in all modern language models

The Enduring Insight

Word2Vec demonstrated that neural networks can learn meaningful representations from raw text without any supervision. The learned vectors capture semantic properties that align with human intuition — a breakthrough that paved the way for all subsequent NLP advances. The famous "king - man + woman ≈ queen" analogy became a symbol of deep learning's power to extract human-interpretable knowledge from data.

📚 Key Design Decisions

Context Window Size

Larger windows (c=10) capture more semantic relationships; smaller windows (c=2) capture syntactic patterns. Default c=5 balances both.

Subsampling Frequent Words

Words like "the", "is", "of" appear too frequently and provide little information. Discarding them with probability proportional to frequency improves both speed and quality.

Hierarchical vs. Negative Sampling

Hierarchical softmax works better for infrequent words; negative sampling is faster and works better for frequent words with large datasets.

Vector Dimensionality

Typically 100-1000 dimensions. Quality improves up to ~600 dimensions, then plateaus. 300 is the most common choice balancing quality and memory.
