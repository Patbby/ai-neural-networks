GRU

Junyoung Chung, Caglar Gulcehre, Kyunghyun Cho, Yoshua Bengio

NIPS 2014 Workshop on Deep Learning
arXiv Code

Problem Statement

Before GRU, training recurrent neural networks (RNNs) with long-term dependencies was notoriously difficult due to the vanishing gradient problem. While LSTM (1997) provided a solution with its complex gating mechanism, it came with significant computational overhead and a large number of parameters.

The Challenge: Could a simpler gating mechanism achieve comparable performance to LSTM while being computationally more efficient? The research sought to identify the minimal set of gating mechanisms necessary for capturing long-term dependencies.

Traditional RNNs use simple activation functions that struggle to maintain gradient flow over long sequences. The question was whether reducing the number of gates (from LSTM's three gates to GRU's two) would significantly impact performance while improving efficiency.

Key Innovation

The Gated Recurrent Unit (GRU) introduced a streamlined architecture that merges the cell state and hidden state into a single vector, reducing the number of gates from three (in LSTM) to two:

  • Reset Gate (r): Determines how much of the previous hidden state to forget when computing the new candidate activation
  • Update Gate (z): Controls how much of the previous hidden state to carry forward versus how much of the new candidate to accept
zt = σ(Wz · [ht-1, xt])

rt = σ(Wr · [ht-1, xt])

h̃t = tanh(W · [rt ⊙ ht-1, xt])

ht = (1 - zt) ⊙ ht-1 + zt ⊙ h̃t

Core Insight: GRU demonstrated that the forget and input gates in LSTM could be coupled into a single update gate without significant performance degradation, simplifying the model while maintaining its ability to capture long-term dependencies.

Architecture

t ht-1 ⊕ Reset Gate σ(Wr·[h,x]) Update Gate σ(Wz·[h,x]) × Candidate h̃t = tanh (W·[r⊙h,x]) Update ht = z⊙h̃t + (1-z)⊙ht-1 ht ● Input/State ● Gating Mechanism ● Element-wise Ops ● Final Update

Key architectural differences from LSTM:

  • No separate cell state — everything flows through the hidden state
  • Only two gates (reset and update) vs. LSTM's three (input, forget, output)
  • Reset gate modulates how much past information to forget
  • Update gate controls the interpolation between old and new information

Results & Impact

The empirical evaluation compared GRU against LSTM and traditional tanh RNNs on two challenging sequence modeling tasks: polyphonic music modeling and speech signal modeling.

Key Findings:

  • GRU consistently outperformed traditional tanh RNNs by a significant margin
  • GRU achieved comparable performance to LSTM on both evaluated tasks
  • GRU converged faster and required fewer parameters than LSTM
  • Gated units (both LSTM and GRU) are essential for capturing long-term dependencies

The paper demonstrated that the forget gate (or its equivalent in the update gate) is the most crucial component for learning long-term dependencies, while other gates provide additional flexibility but are not strictly necessary.

Code Implementation

import torch import torch.nn as nn class GRUCell(nn.Module): """ Manual GRU Cell implementation showing the internal mechanics. In practice, use nn.GRU for efficiency. """ def __init__(self, input_size, hidden_size): super().__init__() self.hidden_size = hidden_size # Update gate parameters self.W_z = nn.Linear(input_size + hidden_size, hidden_size) # Reset gate parameters self.W_r = nn.Linear(input_size + hidden_size, hidden_size) # Candidate activation parameters self.W_h = nn.Linear(input_size + hidden_size, hidden_size) def forward(self, x, h_prev): # x: (batch, input_size), h_prev: (batch, hidden_size) # Concatenate input and previous hidden state combined = torch.cat([x, h_prev], dim=1) # Update gate: how much of new info to accept z = torch.sigmoid(self.W_z(combined)) # Reset gate: how much of past to forget r = torch.sigmoid(self.W_r(combined)) # Candidate hidden state (with reset applied) combined_reset = torch.cat([x, r * h_prev], dim=1) h_tilde = torch.tanh(self.W_h(combined_reset)) # Final hidden state: blend of old and new h = (1 - z) * h_prev + z * h_tilde return h # Practical usage with PyTorch's optimized implementation gru = nn.GRU( input_size=128, # Input feature dimension hidden_size=256, # Hidden state dimension num_layers=2, # Stacked GRU layers batch_first=True, # Input format: (batch, seq, features) dropout=0.3, # Dropout between layers bidirectional=True # Bidirectional GRU ) # Forward pass x = torch.randn(32, 50, 128) # (batch, seq_len, input_size) output, hidden = gru(x) # output: (32, 50, 512) - bidirectional concatenates 256*2 # hidden: (4, 32, 256) - 2 layers * 2 directions

Key Takeaways

  • Simplicity Wins: GRU proved that simpler architectures can match complex ones when designed thoughtfully. By reducing from 3 gates to 2 and eliminating the separate cell state, GRU achieved ~25% fewer parameters than LSTM.
  • Task-Dependent Choice: While GRU and LSTM perform similarly on most tasks, GRU is often preferred for smaller datasets or resource-constrained environments, while LSTM may have slight advantages on very large-scale tasks.
  • Gating is Essential: The key insight of both LSTM and GRU is that gating mechanisms are crucial for learning long-term dependencies, regardless of the specific gating architecture chosen.
  • Legacy and Impact: GRU remains widely used today in applications like speech recognition, machine translation, and time series forecasting where efficiency and performance must be balanced.
