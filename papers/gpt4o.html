OpenAI Technical Report · May 2024

GPT-4o

GPT-4o ("o" for "omni") is OpenAI's first natively multimodal model — processing and generating text, audio, image, and video in a single end-to-end neural network without separate modality-specific modules.

Organization: OpenAI
Released: May 2024
Type: Omni LMM
Context: 128K tokens

Architecture Overview

Unlike GPT-4V which used a separate vision encoder bolted onto the language model, GPT-4o processes all modalities natively within a single Transformer. Audio, image, and video tokens are projected into the same embedding space as text tokens, enabling truly joint cross-modal attention.

Key Innovations

🎵

Native Audio Understanding

Processes audio waveforms directly without ASR transcription, preserving prosody, emotion, and speaker characteristics lost in text conversion.

⚡

232ms Response Latency

Achieves human-level conversational response speed. Previous GPT-4V pipeline (ASR → LLM → TTS) averaged 2.8 seconds of inherent delay.

👁️

Vision-Language Integration

Unified attention over interleaved text and image tokens. Handles complex document understanding, charts, and real-time video streams.

🌐

50+ Language Support

Strong multilingual audio performance with improved non-English speech recognition, surpassing Whisper large-v3 on many benchmarks.

Technical Details

🔑 Omni Architecture Key Insight

By training a single Transformer on all modalities jointly, GPT-4o can leverage cross-modal information during both encoding and decoding — e.g., voice tone can influence text generation, and visual context can shape audio output prosody.

# GPT-4o Token Processing (simplified) audio_tokens = AudioEncoder(waveform) # spectral patches → embeddings image_tokens = VisionEncoder(pixels) # 14×14 patches → embeddings text_tokens = TextTokenizer(text) # BPE tokens → embeddings all_tokens = concat([audio_tokens, image_tokens, text_tokens]) output = TransformerLM(all_tokens) # Unified attention over everything # Output modality selected via special tokens: # <|text_output|>, <|audio_output|>, <|image_output|>

The model is trained with a next-token prediction objective across all modalities simultaneously. Special boundary tokens delineate modality transitions, and the model learns to predict the appropriate output type based on context.

GPT-4o Capabilities

Real-time Voice Conversation

Interruption-aware dialogue with natural turn-taking. Detects laughter, sarcasm, stress in speaker voice and responds appropriately.

Screen & Vision Analysis

Live screen sharing, document OCR, mathematical diagrams, complex charts, and multi-image reasoning in a single turn.

Multilingual Audio

Seamless code-switching across 50+ languages in both audio input and output, outperforming dedicated ASR systems on non-English languages.

Reasoning Under Multimodal Context

Complex tasks combining visual, textual, and audio context — e.g., "Listen to this audio clip and identify the chart in this image that best represents it."

Impact & Significance

GPT-4o represents a fundamental shift from pipeline-based multimodal AI (separate specialized models chained together) to native multimodal AI (a single model that understands all modalities jointly). This architectural shift enables capabilities that pipeline models structurally cannot achieve, such as emotional awareness through voice tone or cross-modal grounding.

The 232ms latency target — matching human conversational response time — opened the door to genuinely natural human-AI voice interaction, a milestone previously blocked by pipeline latency. GPT-4o's release triggered rapid industry adoption of unified multimodal architectures across Google (Gemini 1.5), Anthropic (Claude 3.5), and Meta (LLaMA 3.2).
