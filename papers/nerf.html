NeRF

Neural Radiance Fields for View Synthesis
Ben Mildenhall, Pratul P. Srinivasan, Matthew Tancik, Jonathan T. Barron, Ravi Ramamoorthi, Ren Ng · UC Berkeley · ECCV 2020
🏆 ECCV 2020 Oral 🎯 3D Vision 🖼️ Novel View Synthesis
📄 arXiv 📥 PDF 🌐 Project Page 💻 Code

📋 Abstract

We present a method that achieves state-of-the-art results for synthesizing novel views of complex scenes by optimizing an underlying continuous volumetric scene function using a sparse set of input views.

Our algorithm represents a scene using a fully-connected (non-convolutional) deep network, whose input is a single continuous 5D coordinate (spatial location (x,y,z) and viewing direction (θ, φ)) and whose output is the volume density and view-dependent emitted radiance at that spatial location.

We synthesize views by querying 5D coordinates along camera rays and use classic volume rendering techniques to project the output colors and densities into an image. Because volume rendering is naturally differentiable, the only input required to optimize our representation is a set of images with known camera poses.

🎯 Problem Statement

Before NeRF: The View Synthesis Challenge

Synthesizing novel views of complex scenes from sparse input images had been a long-standing challenge in computer vision. Traditional approaches faced fundamental limitations:

  • Mesh-based methods: Required explicit 3D reconstruction, struggling with complex geometry, thin structures, and translucent materials
  • Light field methods: Needed dense sampling of views, failing with sparse inputs
  • Voxel-based approaches: Suffered from discretization artifacts and memory inefficiency at high resolutions
  • Deep learning methods: Prior neural rendering used CNNs on 2D projections, not exploiting 3D scene structure

Why It Mattered

Enabling photorealistic novel view synthesis from just a few images would revolutionize virtual reality, augmented reality, cinematography, and 3D content creation. The key challenge was representing continuous 3D scenes in a way that could be optimized end-to-end from 2D images.

💡 Key Innovation

Core Idea: Represent Scenes as Neural Fields

Instead of explicit 3D representations, represent the scene as a continuous volumetric function parameterized by a neural network. Given any 5D coordinate (3D position + 2D viewing direction), the network predicts density and color — enabling differentiable rendering and end-to-end optimization.

The Neural Radiance Field Representation

  • 5D Input: Spatial coordinates (x, y, z) + viewing direction (θ, φ)
  • 4D Output: Volume density σ (scalar) + RGB radiance (3D vector)
  • MLP Architecture: 8 layers with 256 channels, ReLU activations
  • Positional Encoding: High-frequency encoding of inputs to capture fine details
  • View-Dependent Color: Viewing direction processed later for specular effects
F_Θ : (x, y, z, θ, φ) → (r, g, b, σ)

Positional Encoding

A critical innovation: mapping inputs to higher dimensions using sinusoidal functions enables the MLP to represent high-frequency details:

γ(p) = [sin(2⁰πp), cos(2⁰πp), ..., sin(2^(L-1)πp), cos(2^(L-1)πp)]

🏗️ Architecture

NeRF Pipeline Overview:

  1. 5D Coordinate Input: For each point along camera rays, input (x,y,z,θ,φ)
  2. Positional Encoding: High-frequency encoding enables fine detail representation
  3. MLP Processing: 8-layer network with skip connections
  4. Density Prediction: Output volume density σ (geometry)
  5. View-Dependent Color: Concatenate direction, predict RGB
  6. Volume Rendering: Accumulate colors and densities along rays
  7. Photometric Loss: Compare rendered vs. ground truth pixels

Hierarchical Sampling

NeRF uses a two-stage sampling strategy: first, a coarse network samples uniformly along rays. Then, the coarse densities guide importance sampling for the fine network, focusing samples near surfaces.

💻 Code Implementation

NeRF Core Model (PyTorch-style)

import torch
import torch.nn as nn
import torch.nn.functional as F

class NeRF(nn.Module):
    def __init__(self, D=8, W=256, input_ch=63, 
                 input_ch_views=27, output_ch=4, 
                 skips=[4], use_viewdirs=True):
        super(NeRF, self).__init__()
        self.D = D
        self.W = W
        self.input_ch = input_ch
        self.input_ch_views = input_ch_views
        self.skips = skips
        self.use_viewdirs = use_viewdirs
        
        # Position encoding network (8 layers)
        self.pts_linears = nn.ModuleList(
            [nn.Linear(input_ch, W)] + 
            [nn.Linear(W, W) if i not in skips 
             else nn.Linear(W + input_ch, W) 
             for i in range(D-1)]
        )
        
        # View direction network (for view-dependent color)
        self.views_linears = nn.ModuleList([nn.Linear(input_ch_views + W, W//2)])
        
        # Output layers
        self.feature_linear = nn.Linear(W, W)
        self.alpha_linear = nn.Linear(W, 1)  # Density
        self.rgb_linear = nn.Linear(W//2, 3)  # RGB
    
    def forward(self, x):
        input_pts, input_views = torch.split(x, [self.input_ch, self.input_ch_views], dim=-1)
        h = input_pts
        for i, l in enumerate(self.pts_linears):
            h = self.pts_linears[i](h)
            h = F.relu(h)
            if i in self.skips:
                h = torch.cat([input_pts, h], -1)
        
        alpha = self.alpha_linear(h)  # Density
        feature = self.feature_linear(h)
        
        if self.use_viewdirs:
            h = torch.cat([feature, input_views], -1)
            for i, l in enumerate(self.views_linears):
                h = self.views_linears[i](h)
                h = F.relu(h)
            rgb = self.rgb_linear(h)
        
        return torch.cat([rgb, alpha], -1)

# Positional encoding function
def get_embedder(multires, input_dims=3):
    def embed_fn(x):
        freq_bands = 2.**torch.arange(0, multires, dtype=torch.float32) * torch.pi
        freq_bands = freq_bands.to(x.device)
        out_list = [x]
        for freq in freq_bands:
            out_list += [torch.sin(x * freq), torch.cos(x * freq)]
        return torch.cat(out_list, -1)
    return embed_fn, input_dims * multires * 2 + input_dims

Volume Rendering

def render_rays(ray_batch, network_fn, N_samples, N_importance=0):
    # Extract ray information
    rays_o, rays_d = ray_batch[:,0:3], ray_batch[:,3:6]  # Origins, directions
    viewdirs = ray_batch[:,-3:] if ray_batch.shape[-1] > 8 else None
    bounds = ray_batch[..., 6:8]  # Near, far bounds
    near, far = bounds[...,0], bounds[...,1]
    
    # Sample points along rays (coarse sampling)
    t_vals = torch.linspace(0., 1., N_samples).to(rays_o.device)
    z_vals = near[..., None] * (1. - t_vals) + far[..., None] * t_vals
    z_vals = z_vals.expand([N_rays, N_samples])
    
    # Get 3D points along rays
    pts = rays_o[..., None, :] + rays_d[..., None, :] * z_vals[..., :, None]
    
    # Run network
    raw = network_fn(pts)
    rgb, density = raw[..., :3], raw[..., 3:]
    
    # Volume rendering (discretized)
    dists = z_vals[..., 1:] - z_vals[..., :-1]
    dists = torch.cat([dists, torch.ones_like(dists[...,:1])*1e10], -1)
    
    alpha = 1. - torch.exp(-F.relu(density) * dists)
    transmittance = torch.cumprod(
        torch.cat([torch.ones_like(alpha[...,:1]), 1.-alpha+1e-10], -1), -1)[..., :-1]
    weights = alpha * transmittance
    
    # Composite colors
    rgb_map = torch.sum(weights[..., None] * rgb, -2)
    depth_map = torch.sum(weights * z_vals, -1)
    
    return rgb_map, depth_map, weights

✨ Unique Features

1. Continuous Implicit Representation

Unlike voxel grids or mesh representations, NeRF represents scenes as a continuous function. This eliminates discretization artifacts and enables arbitrarily high-resolution rendering without memory explosion.

2. View-Dependent Appearance

By feeding viewing direction to the later layers of the network, NeRF can model complex optical phenomena like specular reflections, transparency, and changing appearance with viewpoint — something impossible with view-independent methods.

3. Positional Encoding

The key insight that enables high-frequency detail: mapping low-dimensional coordinates to higher dimensions using sinusoidal functions allows the MLP to represent fine geometric details that would otherwise be smoothed out.

4. Hierarchical Sampling

Two-stage sampling with importance sampling guided by coarse densities focuses network capacity on visible surfaces, dramatically improving efficiency compared to uniform sampling.

5. Differentiable Rendering

By making classical volume rendering differentiable, NeRF enables end-to-end optimization from 2D images alone — no 3D supervision, depth maps, or multi-view stereo required.
