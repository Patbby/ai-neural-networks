RLHF: Reinforcement Learning from Human Feedback

Training language models to follow instructions with human feedback (InstructGPT methodology)

Long Ouyang Jeffrey Wu Xu Jiang Diogo Almeida Carroll Wainwright Pamela Mishkin + OpenAI Team
arXiv Project Page Code

The Problem

Large language models like GPT-3 demonstrated remarkable capabilities in text generation, but they had a critical flaw: misalignment with human intent. These models were trained to predict the next token on internet text, not to follow instructions or be helpful assistants.

Instruction Ignoring

GPT-3 often failed to follow user instructions, generating outputs that didn't address the actual request or misunderstood the task format.

Toxic & Biased Outputs

Training on unfiltered internet text led to models that could generate toxic, biased, or harmful content when prompted.

Hallucinations

Models would confidently generate plausible-sounding but factually incorrect information, making them unreliable.

Truthfulness Issues

Without explicit training on truthfulness, models prioritized fluency over accuracy, leading to fabricated information.

Key Innovation

Three-stage RLHF Pipeline: A methodology that combines supervised fine-tuning with reinforcement learning guided by a reward model trained on human preferences, enabling language models to learn what humans actually want rather than just predicting text.

The revolutionary insight was that comparing outputs is easier than writing perfect ones. By having humans rank multiple model outputs rather than writing ideal responses, RLHF scaled alignment to 175B parameter models. A 1.3B parameter aligned model could outperform a 175B unaligned model on instruction-following tasks.

The Three-Stage Architecture

1

Supervised Fine-Tuning (SFT)

Start with a pretrained language model (GPT-3) and fine-tune it on ~13,000 high-quality human demonstrations. Labelers write ideal responses to prompts, teaching the model basic instruction-following behavior.

Goal: Initialize the model with a strong starting policy

2

Reward Model Training

Take the SFT model, generate 4-9 outputs for ~33,000 prompts, and have humans rank them. Train a reward model to predict human preferences using the Bradley-Terry model and cross-entropy loss.

Goal: Create an automated preference judge

3

Reinforcement Learning (PPO)

Optimize the SFT model against the reward model using Proximal Policy Optimization (PPO) with KL-divergence penalty. Mix in pretraining gradients (PPO-ptx) to prevent forgetting general capabilities.

Goal: Maximize human preference scores

Mathematical Formulation

Reward Model Objective (Bradley-Terry)

LRM = -E(x,yw,yl)~D[log σ(rθ(x, yw) - rθ(x, yl))]

Where yw is the preferred (winning) response and yl is the less preferred (losing) response. The reward model learns to assign higher scores to outputs humans prefer.

PPO-ptx Objective

LPPO-ptx = E(x,y)~πθ[min(rtÂt, clip(rt, 1-ε, 1+ε)Ât)] - βKL(πθ || πref) + γEx~Dpretrain[log πθ(x)]

The objective has three components: (1) Clipped PPO surrogate loss for stability, (2) KL penalty to prevent reward hacking, (3) Pretraining mix to maintain general capabilities.

# RLHF Three-Stage Training Pipeline (Simplified)
import torch
import torch.nn as nn
from transformers import GPT2LMHeadModel, GPT2Tokenizer

# ========== Stage 1: Supervised Fine-Tuning ==========
def sft_training(pretrained_model, sft_dataset, epochs=16):
    """
    Fine-tune pretrained model on human demonstrations.
    sft_dataset: List of (prompt, response) pairs (~13K examples)
    """
    model = pretrained_model
    optimizer = torch.optim.AdamW(model.parameters(), lr=5e-6)
    
    for epoch in range(epochs):
        for prompt, response in sft_dataset:
            # Concatenate prompt + response
            inputs = tokenizer(prompt + response, return_tensors="pt")
            labels = inputs.input_ids.clone()
            labels[:, :len(prompt)] = -100  # Mask prompt tokens
            
            outputs = model(**inputs, labels=labels)
            loss = outputs.loss
            loss.backward()
            optimizer.step()
            optimizer.zero_grad()
    
    return model  # SFT model

# ========== Stage 2: Reward Model Training ==========
class RewardModel(nn.Module):
    def __init__(self, sft_model):
        super().__init__()
        self.transformer = sft_model.transformer
        # Replace LM head with reward head (outputs scalar)
        self.reward_head = nn.Linear(sft_model.config.n_embd, 1, bias=False)
    
    def forward(self, input_ids, attention_mask):
        hidden = self.transformer(input_ids, attention_mask=attention_mask).last_hidden_state
        # Use last token representation
        rewards = self.reward_head(hidden[:, -1, :])
        return rewards.squeeze(-1)

def reward_model_loss(reward_model, x, y_winner, y_loser):
    """
    Bradley-Terry pairwise preference loss.
    x: prompt, y_winner: preferred response, y_loser: less preferred response
    """
    # Get reward for both outputs
    r_w = reward_model(x + y_winner)
    r_l = reward_model(x + y_loser)
    
    # Bradley-Terry model: maximize P(y_w > y_l) = sigmoid(r_w - r_l)
    loss = -torch.log(torch.sigmoid(r_w - r_l)).mean()
    return loss

# ========== Stage 3: PPO Training ==========
def ppo_step(policy, reference_policy, reward_model, prompts, beta=0.02, epsilon=0.2):
    """
    Single PPO update with KL penalty.
    """
    # Generate responses
    responses, log_probs = policy.generate(prompts)
    
    # Compute rewards from reward model
    rewards = reward_model(prompts, responses)
    
    # KL penalty: prevent policy from drifting too far
    ref_log_probs = reference_policy.get_logprobs(prompts, responses)
    kl_penalty = beta * (log_probs - ref_log_probs)

Key Takeaways

  1. Alignment beats scale: A 1.3B aligned model can outperform a 175B unaligned model on instruction-following tasks.
  2. Comparisons scale better: Having humans rank outputs is more efficient than asking them to write perfect responses.
  3. KL penalty is critical: Without it, the model learns to "hack" the reward model rather than genuinely improve.
  4. Pretraining mix prevents forgetting: PPO-ptx maintains general capabilities while learning alignment.
  5. RLHF is now standard: Every major LLM (GPT-4, Claude, Llama, etc.) uses RLHF or a variant.
