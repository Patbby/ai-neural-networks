Neural Turing Machine

Coupling Neural Networks to External Memory Resources

Alex Graves, Greg Wayne, Ivo Danihelka | DeepMind | arXiv:1410.5401 | 2014

📄 arXiv 📑 PDF 💻 Code

The Problem

Traditional neural networks are fundamentally limited by their fixed-size internal state. While architectures like LSTM can maintain information over time, they suffer from:

  • Vanishing gradients when learning long-term dependencies
  • Fixed memory capacity determined by network size
  • Inability to perform algorithmic reasoning requiring external storage
  • Limited compositional generalization for sequence processing
Core Insight: Computer programs use three fundamental mechanisms: elementary operations, logical flow control, and external memory. Modern machine learning had largely neglected external memory—until NTM.

Key Innovation

Neural Turing Machines extend neural networks by coupling them to an external memory matrix that can be read from and written to via attentional processes. The architecture combines:

🔧 Neural Controller

An LSTM or feedforward network that processes inputs and emits read/write parameters

💾 External Memory Matrix

A large addressable memory bank (N × W matrix) for storing vectors

👁️ Read/Write Heads

Multiple attention mechanisms for content-based and location-based addressing

📍 Differentiable Attention

"Blurry" operations that interact with all memory locations via weighted addressing

Architecture

Read Operation

rt = Σi wt(i) · Mt(i)

Where wt is the weighting over memory locations (attention distribution) and Mt(i) is the content at location i.

Write Operation (Erase + Add)

Mt(i) = Mt-1(i) ⊙ [1 - wt(i)et] + wt(i)at

First, content is erased using erase vector et, then new content at is added. The ⊙ denotes element-wise multiplication.

Addressing Mechanisms

NTM uses two complementary addressing mechanisms:

📌 Content-Based Addressing

Focus on locations based on similarity to a key vector k. Uses cosine similarity:

K[u,v] = (u·v) / (||u|| · ||v||)

Allows retrieval by content similarity—like associative memory.

🎯 Location-Based Addressing

Iterative shifting of attention. The controller can:

  • Rotate attention (circular convolution)
  • Move focus sequentially
  • Sharpen selection

Enables sequential access patterns—critical for algorithms.

PyTorch Implementation

ntm.py 
# Neural Turing Machine - Core Components
import torch
import torch.nn as nn
import torch.nn.functional as F

class NTM(nn.Module):
    def __init__(self, input_size, output_size, controller_size, 
                 memory_size, memory_dim, num_read_heads, num_write_heads):
        super(NTM, self).__init__()
        
        # Controller: LSTM processes input
        self.controller = nn.LSTMCell(input_size + num_read_heads * memory_dim, 
                                       controller_size)
        
        # External Memory: N × W matrix
        self.memory_size = memory_size
        self.memory_dim = memory_dim
        self.num_read_heads = num_read_heads
        self.num_write_heads = num_write_heads
        
        # Read/Write head parameters
        self.read_params = nn.Linear(controller_size, num_read_heads * memory_dim)
        self.write_params = nn.Linear(controller_size, num_write_heads * (memory_dim * 2 + 3))
        
        # Output layer
        self.output = nn.Linear(controller_size + num_read_heads * memory_dim, output_size)
    
    def address(self, key, memory, strength, gate, shift, sharpen):
        # Content-based addressing
        key = key.view(-1, 1, self.memory_dim)
        sim = F.cosine_similarity(key, memory, dim=-1)
        wc = F.softmax(strength * sim, dim=-1)
        
        # Interpolation with previous weights
        w = gate * wc + (1 - gate) * self.prev_weights
        
        # Location-based shifting (circular convolution)
        w = self.circular_convolution(w, shift)
        
        # Sharpening
        w = w ** sharpen
        w = w / (w.sum(dim=-1, keepdim=True) + 1e-8)
        
        return w
    
    def read(self, weights, memory):
        # Weighted sum of memory locations
        return torch.bmm(weights.unsqueeze(1), memory).squeeze(1)
    
    def write(self, weights, erase, add, memory):
        # Erase-then-add operation
        erase = torch.outer(weights, erase)
        add = torch.outer(weights, add)
        memory = memory * (1 - erase) + add
        return memory
