Fast R-CNN

Fast Region-based Convolutional Networks for Object Detection

📅 ICCV 2015 👤 Ross Girshick 🏢 Microsoft Research 📊 ~8,000 Citations
📄 Paper PDF 📚 arXiv 💻 Code

🎯 Problem Statement

Before Fast R-CNN, object detection faced significant computational bottlenecks:

R-CNN Limitations: The original R-CNN required running a CNN forward pass for each of the ~2,000 region proposals per image. This resulted in massive redundant computation, slow training (84 hours), and even slower testing (47 seconds per image). Additionally, R-CNN required multi-stage training and substantial disk storage for cached features.

R-CNN's architecture processed each region proposal independently through the CNN, even when proposals overlapped significantly. This meant computing convolutional features thousands of times for the same image, making real-time detection impossible.

SPPnet offered partial improvement by sharing convolutional computations, but still used multi-stage training and couldn't update all network layers during fine-tuning.

💡 Key Innovation

Fast R-CNN introduced a single-stage training algorithm that combines classification and bounding-box regression into one unified framework:

RoI Pooling Layer: A specialized pooling layer that extracts fixed-length feature vectors from variable-sized region proposals. It maps each proposal onto the shared convolutional feature map, then uses max pooling to convert features into a fixed spatial extent (e.g., 7×7), enabling efficient batch processing.

The core insight was to process the entire image through convolutional layers once, producing a shared feature map. All region proposals then extract features from this single computation, dramatically reducing redundant calculations.

Multi-task Loss: Fast R-CNN jointly optimizes two objectives simultaneously: classification loss (softmax over K+1 classes) and bounding-box regression loss (smooth L1 for 4K coordinates). This end-to-end training allows all network layers to be updated, unlike R-CNN's frozen CNN features.

🏗️ Architecture

The architecture consists of five main components:

  • Convolutional Backbone: Processes the entire image through conv and pooling layers to produce a feature map
  • RoI Pooling Layer: Extracts fixed-size (7×7) features from variable-sized region proposals
  • Fully Connected Layers: Processes each RoI feature vector through FC layers
  • Classification Head: Outputs softmax probabilities over K object classes + background
  • Regression Head: Outputs refined bounding box coordinates for each class

📐 Multi-Task Loss Function

Fast R-CNN introduces a unified loss that jointly optimizes classification and bounding-box regression:

L(p, u, tu, v) = Lcls(p, u) + λ[u ≥ 1]Lloc(tu, v)
cls(p, u) = -log(pu) p = softmax probabilities, u = ground-truth class Localization Loss Smooth L1 Loss tu = predicted bbox, v = ground-truth bbox Smooth L1 Loss (Huber Loss) L1(x) = 0.5x² if |x| < 1 L1(x) = |x| - 0.5 otherwise

The smooth L1 loss is less sensitive to outliers than L2 loss, preventing gradient explosions during training. The indicator function [u ≥ 1] ensures localization loss is only computed for positive regions (not background).

📊 Results & Impact

Method Training Time (VGG16) Test Time (per image) mAP (VOC 2007)
R-CNN 84 hours 47.0 seconds 66.0%
SPPnet 25 hours 2.3 seconds 63.1%
Fast R-CNN 9.5 hours (8.9× faster) 0.22 seconds (213× faster) 70.0%

Key Achievements: Fast R-CNN achieved state-of-the-art mAP on PASCAL VOC 2007 (70.0%), while dramatically reducing training time by 8.9× and testing time by 213× compared to R-CNN. It also enabled end-to-end training of all network layers.

On VOC 2012, Fast R-CNN achieved 68.8% mAP, outperforming all competing methods at the time. The introduction of multi-task training improved mAP by 0.8-1.1 points over separate training.

💻 PyTorch Implementation

import torch
import torch.nn as nn
import torchvision.models as models

class FastRCNN(nn.Module):
    def __init__(self, num_classes):
        super(FastRCNN, self).__init__()
        # Use VGG16 as backbone
        vgg = models.vgg16(pretrained=True)
        self.backbone = nn.Sequential(*list(vgg.features._modules.values())[:-1])
        
        # RoI Pooling layer (7x7 output)
        self.roi_pool = nn.AdaptiveMaxPool2d((7, 7))
        
        # Fully connected layers
        self.fc1 = nn.Linear(512 * 7 * 7, 4096)
        self.fc2 = nn.Linear(4096, 4096)
        
        # Multi-task heads
        self.cls_score = nn.Linear(4096, num_classes + 1)  # +1 for background
        self.bbox_pred = nn.Linear(4096, 4 * num_classes)    # 4 coords per class
        
        self.relu = nn.ReLU(inplace=True)
        self.dropout = nn.Dropout(0.5)
    
    def forward(self, image, rois):
        # Extract shared conv features
        features = self.backbone(image)  # [1, 512, H/16, W/16]
        
        # RoI Pooling for each region proposal
        pooled_features = []
        for roi in rois:
            # roi = [x1, y1, x2, y2] in image coordinates
            # Map to feature map coordinates (1/16 scale for VGG)
            x1, y1, x2, y2 = roi / 16.0
            roi_feature = features[:, :, int(y1):int(y2), int(x1):int(x2)]
            pooled = self.roi_pool(roi_feature)
            pooled_features.append(pooled.view(-1))
        
        # Stack all RoI features
        x = torch.stack(pooled_features)
        
        # FC layers
        x = self.relu(self.fc1(x))
        x = self.dropout(x)
        x = self.relu(self.fc2(x))
        x = self.dropout(x)
        
        # Multi-task outputs
        cls_scores = self.cls_score(x)
        bbox_preds = self.bbox_pred(x)
        
        return cls_scores, bbox_preds

# Multi-task loss
def fast_rcnn_loss(cls_scores, bbox_preds, labels, bbox_targets, bbox_inside_weights):
    # Classification loss
    cls_loss = nn.CrossEntropyLoss()(cls_scores, labels)
    
    # Bounding box regression loss (Smooth L1)
    bbox_loss = nn.SmoothL1Loss(reduction='sum')(bbox_preds * bbox_inside_weights, 
                                                    bbox_targets * bbox_inside_weights)
    bbox_loss = bbox_loss / labels.numel()
    
    return cls_loss + bbox_loss

R-CNN (2014)

The predecessor that pioneered CNN-based object detection using region proposals and SVM classification.

Faster R-CNN (2015)

Introduced Region Proposal Network (RPN), making the entire detection pipeline end-to-end trainable.

SPPnet (2014)

Introduced spatial pyramid pooling, enabling arbitrary input sizes. Fast R-CNN's RoI pooling is a simplified version.

VGGNet (2014)

The backbone architecture commonly used with Fast R-CNN, featuring 3×3 convolutions and deep networks.
