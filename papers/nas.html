NAS

Neural Architecture Search with Reinforcement Learning

👤 Barret Zoph & Quoc V. Le 🏢 Google Brain 📅 2017 📊 ICLR
📄 Paper PDF 🏆 ICLR 2017

🎯 Paper Summary

Neural Architecture Search (NAS) revolutionized how we design neural networks by introducing the first automated architecture discovery system using reinforcement learning. The paper proposes using a controller RNN to generate architectural descriptions of neural networks, treating architecture search as a sequence generation problem. The controller is trained with REINFORCE policy gradient to maximize the expected validation accuracy of generated architectures. On CIFAR-10, NAS discovered architectures achieving 3.65% test error (0.09% better than human-designed), and on Penn Treebank, it designed recurrent cells outperforming LSTMs with 62.4 perplexity (3.6 better than prior state-of-the-art). This work launched the entire AutoML/NAS field, leading to follow-ups like NASNet, ENAS, and EfficientNet.

🏗️ Architecture Overview

NAS System Architecture

The controller RNN generates architecture strings that define child networks. The child is trained and its validation accuracy becomes the reward signal for updating the controller via REINFORCE.

REINFORCE Policy Gradient Training

💡 Key Innovations

🎮 RNN as Architecture Generator

The core insight: neural network architectures can be represented as variable-length strings. A controller RNN (2-layer LSTM with 35 hidden units) generates these strings token by token. Each token represents a hyperparameter: filter height, filter width, number of filters, stride, etc. The autoregressive nature allows generating architectures of arbitrary depth and complexity.

📊 REINFORCE for Architecture Optimization

Since architecture generation involves discrete decisions (non-differentiable), standard backpropagation doesn't apply. NAS uses the REINFORCE policy gradient algorithm to optimize the controller:

∇θJ(θ) = EP(a;θ)[(R - b) ∇θ log P(a;θ)]
θ = controller parameters | R = validation accuracy | b = baseline (exponential moving average)

🔗 Skip Connections via Anchor Points

To enable modern architectural patterns like ResNet's skip connections, NAS introduces anchor points. At each layer, the controller decides which previous layers to connect to using set attention. This allows building complex multi-branch architectures with residual connections and concatenations.

🧠 RNN Cell Search

Beyond CNNs, NAS searches for novel RNN cell structures using a tree-based search space. The controller combines inputs and previous hidden states using binary trees with learnable combination functions (addition, element-wise multiplication) and activation functions. This discovered the NASCell which outperformed LSTM on Penn Treebank.

🔧 Search Space (CNN)

  • Filter height: {1, 3, 5, 7}
  • Filter width: {1, 3, 5, 7}
  • Number of filters: {24, 36, 48, 64}
  • Stride: {1, 2, 3}
  • Skip connections via anchor points
  • Batch normalization + ReLU

🔄 Training Details

  • Controller: 2-layer LSTM, 35 hidden units
  • Optimizer: Adam (lr=0.0006)
  • Child training: 50 epochs (CIFAR-10)
  • Reward: max validation acc³ (last 5 epochs)
  • 800 GPUs, 3-4 weeks training
  • 12,800 architectures evaluated

📈 Key Results (CIFAR-10)

  • NAS test error: 3.65%
  • Previous SOTA: 3.74%
  • Improvement: 0.09% (1.05× faster)
  • 15-layer discovered architecture
  • Skip connections essential
  • Transferable to ImageNet

📝 Key Results (PTB)

  • NASCell perplexity: 62.4
  • LSTM baseline: 66.0
  • Improvement: 3.6 perplexity
  • Character PTB: 1.214 perplexity
  • New state-of-the-art
  • NASCell published in TensorFlow

🌍 Impact & Legacy

🚀 Launched the NAS Field

This paper sparked a revolution in automated machine learning. Follow-up works include: NASNet (transferable architectures), ENAS (parameter sharing for 1000× speedup), DARTS (differentiable NAS), EfficientNet (compound scaling), and countless industry applications at Google, Microsoft, and Meta.

⚡ Computational Cost Debate

The original NAS required 22,400 GPU-days (800 GPUs for 28 days), sparking research into efficient NAS methods. Modern approaches like weight sharing, supernets, and zero-cost proxies reduce this to hours on a single GPU while maintaining or improving results.

🎯 Key Insight: Meta-Learning

NAS demonstrated that we can learn to learn — using a neural network to design other neural networks. This meta-learning paradigm influenced architecture search, hyperparameter optimization, and even learned optimizers. The idea of "learning the learning process" has become central to modern AutoML.
