Neural Turing Machine

Neural Architecture Augmented with External Memory

📅 2014 👤 Alex Graves, Greg Wayne, Ivo Danihelka 🏢 Google DeepMind 📄 arXiv:1410.5401
📄 arXiv 📑 PDF 💻 PyTorch Code

🎯 Problem Statement

Traditional recurrent neural networks (RNNs) and LSTMs face a fundamental limitation: they must compress all long-range information into a fixed-size hidden state vector. As sequences grow longer, this compression becomes increasingly lossy, and the model struggles to retain and access specific details from the distant past.

The challenge is particularly acute for tasks requiring:

  • Long-term storage of intermediate results during computation
  • Precise retrieval of specific information at arbitrary time steps
  • Algorithmic reasoning that requires working memory (like sorting, copying, or associative recall)
  • Variable-length dependencies that standard attention mechanisms cannot efficiently handle

Core Question: How can we augment neural networks with an external memory system that allows them to store, retrieve, and manipulate information more like a conventional computer—while remaining fully differentiable and trainable via gradient descent?

💡 Key Innovation

The Differentiable Memory Interface

The Neural Turing Machine introduces a differentiable external memory matrix that the neural network controller can read from and write to using soft attention mechanisms. Unlike a standard Turing machine with discrete operations, every memory access is differentiable, enabling end-to-end training with backpropagation.

Key Breakthrough: The NTM separates computation (controller) from storage (memory matrix), allowing the network to learn where to store information and when to retrieve it—effectively learning algorithms rather than just patterns.

Architecture Components

Memory Operations

The NTM performs two fundamental operations on its external memory:

Read Operation:
rₜ = Σᵢ wₜ(i) · Mₜ(i)

The read vector is a weighted sum of all memory locations, where wₜ(i) is the attention weight for location i.
Write Operation:
Mₜ(i) = Mₜ₋₁(i) ⊙ (1 - wₜ(i) · eₜ) + wₜ(i) · aₜ

The write operation first erases (element-wise with erase vector eₜ) then adds (add vector aₜ) to selected memory locations.

🔍 Addressing Mechanisms

A crucial innovation in NTM is its differentiable addressing system that determines where to read and write. The model uses two complementary addressing modes:

Content-Based Addressing

The controller produces a key vector kₜ and compares it to each memory location using cosine similarity. A sharpening parameter βₜ focuses or blurs the attention:

wₜᶜ(i) = exp(βₜ · cos(kₜ, Mₜ(i))) / Σⱼ exp(βₜ · cos(kₜ, Mₜ(j)))

Location-Based Addressing

For sequential access patterns (like iterating through an array), the model can shift attention relative to the previous focus. This is controlled by a shift vector sₜ and sharpening γₜ:

wₜ(i) = Σⱼ wₜ₋₁(j) · sₜ(i - j)
followed by sharpening: wₜ(i) = wₜ(i)^γₜ / Σⱼ wₜ(j)^γₜ

Addressing Interpolation

A gating parameter gₜ ∈ [0,1] interpolates between content-based and location-based addressing, allowing the model to flexibly switch modes:

wₜ = gₜ · wₜᶜ + (1 - gₜ) · wₜ₋₁

📊 Results & Impact

🧮 Algorithm Learning

NTM learned to perform algorithmic tasks like copying, reversing, and sorting sequences—generalizing to longer sequences than seen during training.

🔄 Copy Task

Successfully generalized from training on sequences of length 20 to testing on sequences of length 100+ without degradation.

🧠 Associative Recall

Learned to store item pairs in memory and retrieve the second item when prompted with the first—similar to key-value lookup.

📈 Priority Sort

Learned to sort binary vectors by their priority scores, demonstrating the ability to learn complex data-dependent algorithms.

Why It Matters: NTM demonstrated that neural networks could learn to use external memory as a computational scratchpad, opening the door for "memory-augmented neural networks" (MANNs). This architecture inspired subsequent work including Memory Networks, Differentiable Neural Computers (DNC), and modern retrieval-augmented generation systems.

Key Advantages Over Standard RNNs

  • Long-term storage: Information can be preserved indefinitely in memory without vanishing gradients
  • Content-addressable: Can retrieve information by similarity, not just position
  • Algorithmic generalization: Learns procedures that apply to variable-length inputs
  • Interpretability: Memory access patterns reveal what the model is "thinking"

💻 PyTorch Implementation

import torch
import torch.nn as nn
import torch.nn.functional as F

class NeuralTuringMachine(nn.Module):
    def __init__(self, input_size, output_size, controller_size, 
                 memory_size, memory_vector_dim, num_read_heads=1):
        super().__init__()
        self.controller_size = controller_size
        self.memory_size = memory_size
        self.memory_vector_dim = memory_vector_dim
        self.num_read_heads = num_read_heads
        
        # Controller: LSTM processes input + previous read vectors
        self.controller = nn.LSTM(
            input_size + num_read_heads * memory_vector_dim,
            controller_size,
            batch_first=True
        )
        
        # Output layer combines controller output + read vectors
        self.output_layer = nn.Linear(
            controller_size + num_read_heads * memory_vector_dim,
            output_size
        )
        
        # Memory: N × M matrix initialized with small values
        self.register_buffer('initial_memory', 
                            torch.ones(1, memory_size, memory_vector_dim) * 1e-6)
        
        # Read/Write head parameters
        self.read_head = ReadHead(controller_size, memory_vector_dim)
        self.write_head = WriteHead(controller_size, memory_vector_dim)
    
    def forward(self, x, prev_state=None):
        batch_size = x.size(0)
        
        # Initialize memory and hidden state if needed
        if prev_state is None:
            memory = self.initial_memory.expand(batch_size, -1, -1)
            hidden = None
            prev_reads = torch.zeros(batch_size, self.num_read_heads, 
                                     self.memory_vector_dim)
        else:
            memory, hidden, prev_reads = prev_state
        
        # Controller input: concatenate input + previous read vectors
        controller_input = torch.cat([x, prev_reads.view(batch_size, -1)], dim=-1)
        controller_input = controller_input.unsqueeze(1)  # Add time dim
        
        # Run controller LSTM
        controller_out, hidden = self.controller(controller_input, hidden)
        controller_out = controller_out.squeeze(1)
        
        # Read from memory
        read_weights, read_vector = self.read_head(controller_out, memory)
        
        # Write to memory
        memory = self.write_head(controller_out, memory)
        
        # Generate output: combine controller output + read vector
        output_input = torch.cat([controller_out, read_vector.view(batch_size, -1)], dim=-1)
        output = self.output_layer(output_input)
        
        state = (memory, hidden, read_vector)
        return output, state

class ReadHead(nn.Module):
    def __init__(self, controller_size, memory_vector_dim):
        super().__init__()
        # Parameters for content-based addressing
        self.key_layer = nn.Linear(controller_size, memory_vector_dim)
        self.beta_layer = nn.Linear(controller_size, 1)
        
    def forward(self, controller_output, memory):
        # Content-based addressing
        key = torch.tanh(self.key_layer(controller_output))
        beta = F.softplus(self.beta_layer(controller_output))  # Sharpening
        
        # Cosine similarity between key and memory locations
        similarity = F.cosine_similarity(
            key.unsqueeze(1), memory, dim=-1
        )
        
        # Attention weights
        weights = F.softmax(beta * similarity, dim=1)
        
        # Read vector: weighted sum of memory
        read_vector = torch.bmm(weights.unsqueeze(1), memory).squeeze(1)
        
        return weights, read_vector

class WriteHead(nn.Module):
    def __init__(self, controller_size, memory_vector_dim):
        super().__init__()
        self.key_layer = nn.Linear(controller_size, memory_vector_dim)
        self.beta_layer = nn.Linear(controller_size, 1)
        self.erase_layer = nn.Linear(controller_size, memory_vector_dim)
        self.add_layer = nn.Linear(controller_size, memory_vector_dim)
        
    def forward(self, controller_output, memory):
        # Addressing
        key = torch.tanh(self.key_layer(controller_output))
        beta = F.softplus(self.beta_layer(controller_output))
        
        similarity = F.cosine_similarity(
            key.unsqueeze(1), memory, dim=-1
        )
        weights = F.softmax(beta * similarity, dim=1).unsqueeze(-1)
        
        # Erase and add vectors
        erase = torch.sigmoid(self.erase_layer(controller_output)).unsqueeze(1)
        add = torch.tanh(self.add_layer(controller_output)).unsqueeze(1)
        
        # Write operation: erase then add
        memory = memory * (1 - weights * erase) + weights * add
        
        return memory
