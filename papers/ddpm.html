The Problem

⚠️ Challenge

Prior generative models faced significant limitations: GANs suffered from training instability and mode collapse; autoregressive models were slow at generation; and VAEs produced blurry samples. There was a need for a stable, high-quality generative model that could produce diverse samples without adversarial training.

Deep generative models had made remarkable progress, but each approach had fundamental trade-offs. GANs could generate sharp images but were notoriously difficult to train. Flow-based models required specialized architectures. Score matching and energy-based models showed promise but had not yet achieved competitive sample quality on standard benchmarks like CIFAR-10 and ImageNet.

Key Innovation

💡 Core Idea

DDPM introduces a simplified training objective for diffusion probabilistic models, showing that optimizing a variational bound on log-likelihood is equivalent to denoising score matching. The model learns to reverse a fixed Markov chain that gradually adds Gaussian noise to data.

The key insight is a connection between diffusion models and denoising score matching with Langevin dynamics. By parameterizing the model to predict noise rather than the clean image, and using a simplified objective, DDPM achieves:

  • Stable training: No adversarial components, straightforward MSE loss
  • High-quality samples: State-of-the-art FID scores on image benchmarks
  • Flexibility: Progressive lossy decompression similar to autoregressive models
  • Theoretical grounding: Direct connection to score matching

Architecture & Method

DDPM consists of two processes: a forward (diffusion) process that gradually adds noise to data, and a reverse process that learns to denoise.

Forward: q(xt | xt-1) = N(xt; √(1-βt) xt-1, βt I)
Reverse: pθ(xt-1 | xt) = N(xt-1; μθ(xt, t), σt2 I)
T noise β₁=10⁻⁴ → βT=0.02 U-Net εθ(xt, t) Down Attn Mid Up Time embedding t → Reverse Process (Denoising) xT noise xT-1 ... x₀ clean q(xt|x₀) pθ(xt-1|xt) Training Objective L = 𝔼t,x₀,ε[‖ε - εθ(xt,t)‖²] Predict noise, not x₀

The architecture uses a U-Net backbone with:

  • Group normalization throughout all layers
  • Sinusoidal time embeddings injected at every residual block
  • Self-attention at 16×16 feature map resolution
  • Skip connections between encoder and decoder paths

On 256×256 LSUN bedrooms and churches, DDPM achieved sample quality comparable to ProgressiveGAN. The paper demonstrated that:

  • Diffusion models could match and exceed GAN quality without adversarial training
  • The simplified objective (predicting noise) was crucial for good performance
  • Models naturally supported progressive lossy decompression
  • Training was stable without mode collapse issues

Code Implementation

Key PyTorch implementation showing the forward diffusion and training process:

# DDPM Forward Diffusion and Training
import torch
import torch.nn as nn
import numpy as np

class DDPM(nn.Module):
    def __init__(self, model, timesteps=1000):
        super().__init__()
        self.model = model  # U-Net
        self.T = timesteps
        
        # Linear beta schedule
        self.beta = torch.linspace(1e-4, 0.02, timesteps)
        self.alpha = 1. - self.beta
        self.alpha_bar = torch.cumprod(self.alpha, dim=0)
    
    def forward_diffusion(self, x0, t, noise):
        # q(x_t | x_0) - closed form
        sqrt_alpha_bar = torch.sqrt(self.alpha_bar[t])
        sqrt_one_minus = torch.sqrt(1 - self.alpha_bar[t])
        return sqrt_alpha_bar * x0 + sqrt_one_minus * noise
    
    def compute_loss(self, x0):
        batch_size = x0.shape[0]
        t = torch.randint(0, self.T, (batch_size,))
        noise = torch.randn_like(x0)
        
        # Forward diffusion
        xt = self.forward_diffusion(x0, t, noise)
        
        # Predict noise
        noise_pred = self.model(xt, t)
        
        # MSE loss
        return nn.functional.mse_loss(noise_pred, noise)
