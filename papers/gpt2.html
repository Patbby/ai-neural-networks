2019

GPT-2

Language Models are Unsupervised Multitask Learners

语言模型是无监督的多任务学习器

📄 OpenAI Technical Report
👥 Alec Radford, Jeffrey Wu, Rewon Child, David Luan, Dario Amodei, Ilya Sutskever
📅 February 2019
📄 论文 PDF 💻 官方代码 🔗 OpenAI Blog

问题背景

🎯 传统方法的局限

传统 NLP 系统通常是针对特定任务训练的"狭窄专家"，需要大量标注数据和监督学习，难以适应新任务或领域变化。

📊 多任务学习的瓶颈

虽然多任务学习有潜力，但传统方法只能处理10-17个数据集-目标对，远不足以实现真正的通用智能。

🔄 预训练+微调的依赖

GPT-1 证明了预训练+微调的有效性，但仍需要任务特定的标注数据进行调整，无法实现真正的零样本学习。

核心创新

核心洞察：当语言模型在足够大且多样化的无标签文本语料上进行训练时，它们可以在没有任何明确监督的情况下，执行许多下游任务。这是零样本学习（Zero-Shot Learning）的开端。

GPT-2 的核心假设是：语言建模目标本身就等同于多任务学习——只是评估指标的范围不同。通过最大化条件概率 P(output|input)，模型实际上在学习执行各种任务。

🌐 WebText 数据集

创建了 WebText 数据集：从 Reddit 外链中抓取的 4500 万网页，约 40GB 高质量文本，包含 800 万篇文档。这是当时最大的开放文本数据集之一。

🚀 规模扩展

将模型规模从 GPT-1 的 1.17 亿参数扩展到最大 15 亿参数（GPT-2 XL），验证"更大模型=更好性能"的假设。

🔤 Byte-Level BPE

使用字节级字节对编码（Byte-Level BPE），词汇表扩展到 50,257，支持任意 Unicode 字符，无需预处理。

⚡ 架构改进

LayerNorm 移至子块输入（Pre-LN），改进残差初始化，上下文长度扩展到 1024 tokens。

模型架构

GPT-2 基于改进的 Transformer Decoder 架构，通过四种不同规模验证缩放法则：

GPT-2 架构流程

输入嵌入层

Token Embeddings + Position Embeddings

Vocab: 50,257 Context: 1024

Transformer Decoder × 48

Masked Multi-Head Self-Attention + FFN

Hidden: 1600 Heads: 25 FFN: 6400

输出层

LayerNorm → Linear → Softmax

输出: 词汇概率分布

实验结果

GPT-2 在 8 个语言建模数据集中的 7 个上实现了 SOTA，并在多项任务上展示了强大的零样本能力：

任务 数据集 指标 GPT-2 (XL) 表现
语言建模 WikiText-2 Perplexity 18.34 (SOTA)
阅读理解 CoQA F1 55.0 (接近监督方法)
机器翻译 WMT-14 (En-Fr) BLEU 11.5 (无监督)
摘要 CNN/DailyMail ROUGE-1 21.0 (接近监督方法)
问答 Natural Questions Accuracy 4.1% (L1距离: 3.5%)

关键发现：模型规模与性能呈现明显的幂律关系（Power Law）——更大的模型在所有任务上都表现更好，这为后续 GPT-3 的 175B 参数奠定了理论基础。

影响与意义

  • 零样本学习的开创

    首次证明大型语言模型可以在没有任何任务特定训练的情况下执行多种 NLP 任务，开启了提示工程（Prompt Engineering）时代。

  • 缩放法则的验证

    实验数据支持"规模就是一切"的观点，直接推动了 GPT-3、PaLM、LLaMA 等超大规模模型的研发。

  • AI 安全讨论

    OpenAI 因担心滥用风险，最初选择分阶段发布模型（先发布 117M 版本），引发了关于 AI 研究开放性的重要讨论。

  • 通用人工智能的路径

    展示了通过无监督预训练实现通用语言理解的潜力，为 AGI 研究提供了新的方向。

  • 开源生态

    尽管模型最初受限，但 GPT-2 的开源最终催生了大量复现、改进和应用，包括 Hugging Face Transformers 库的早期发展。

代码实现

🐍 PyTorch 简化版 GPT-2 推理 
import torch
import torch.nn as nn

class GPT2Block(nn.Module):
    def __init__(self, d_model=1600, n_heads=25, d_ff=6400):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = nn.MultiheadAttention(d_model, n_heads)
        self.ln2 = nn.LayerNorm(d_model)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model)
        )
    
    def forward(self, x):
        # Pre-LN: LayerNorm before attention
        attn_out, _ = self.attn(self.ln1(x), x, x)
        x = x + attn_out
        # Pre-LN: LayerNorm before FFN
        x = x + self.ffn(self.ln2(x))
        return x

class GPT2(nn.Module):
    def __init__(self, vocab_size=50257, d_model=1600, 
                 n_layers=48, n_heads=25, max_len=1024):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(max_len, d_model)
        self.blocks = nn.ModuleList([
            GPT2Block(d_model, n_heads) for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(d_model)
        self.head = nn.Linear(d_model, vocab_size, bias=False)
    
    def forward(self, x):
        b, t = x.size()
        pos = torch.arange(0, t, device=x.device).unsqueeze(0)
        x = self.token_emb(x) + self.pos_emb(pos)
        for block in self.blocks:
            x = block(x)
        x = self.ln_f(x)
        return self.head(x)

# 使用 transformers 库加载预训练模型
from transformers import GPT2LMHeadModel, GPT2Tokenizer

model = GPT2LMHeadModel.from_pretrained('gpt2-xl')
tokenizer = GPT2Tokenizer.from_pretrained('gpt2-xl')

input_text = "Neural networks are"
inputs = tokenizer(input_text, return_tensors='pt')
outputs = model.generate(**inputs, max_length=50)
print(tokenizer.decode(outputs[0]))
