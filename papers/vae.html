Problem Statement

Before VAE, training deep generative models with continuous latent variables faced two major challenges:

Intractable Posterior Inference: Computing the true posterior distribution p(z|x) was analytically intractable for most interesting models, making maximum likelihood training impossible.
Scalability Issues: Existing methods like MCMC (Markov Chain Monte Carlo) or variational inference were too slow for large datasets, making them impractical for real-world applications.

Traditional autoencoders could encode data into a compressed representation, but they couldn't generate new samples because the latent space lacked meaningful probabilistic structure.

Key Innovation

VAE introduced a revolutionary approach that combines deep learning with variational inference:

1. Reparameterization Trick: Instead of sampling z directly from q(z|x), the authors reparameterized z as z = μ + σ⊙ε where ε ~ N(0, I). This makes the sampling operation differentiable, enabling gradient-based optimization.
z = μφ(x) + σφ(x) ⊙ ε,    ε ~ 𝒩(0, I)
2. Recognition Model (Encoder): Instead of optimizing variational parameters for each data point separately, they use a neural network (the encoder) to output the parameters of the approximate posterior distribution q(z|x).
3. Evidence Lower Bound (ELBO): The training objective combines reconstruction quality with a KL divergence term that regularizes the latent space to match a prior distribution.
ELBO = 𝔼q(z|x)[log p(x|z)] − DKL(q(z|x) || p(z))

Architecture

The VAE consists of two neural networks working together:

φ(z|x) μ σ z = μ + σ⊙ε ε ~𝒩(0,I) z Decoder pθ(x|z) x̂ DKL(q(z|x)||p(z))
Encoder (Recognition Model): Maps input data x to parameters of the approximate posterior q(z|x), typically outputting mean μ and log-variance log σ² of a Gaussian distribution.
Latent Space: A compressed, probabilistic representation where nearby points decode to similar data. The KL term ensures this space follows a standard normal distribution.
Decoder (Generative Model): Reconstructs the input from the latent code z, learning to generate realistic data by maximizing the likelihood p(x|z).

Code Implementation

Here's a clean PyTorch implementation of the VAE:

import torch
import torch.nn as nn
import torch.nn.functional as F

class VAE(nn.Module):
    def __init__(self, input_dim=784, hidden_dim=400, latent_dim=20):
        super(VAE, self).__init__()
        
        # Encoder
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.fc_mu = nn.Linear(hidden_dim, latent_dim)
        self.fc_logvar = nn.Linear(hidden_dim, latent_dim)
        
        # Decoder
        self.fc_decode = nn.Linear(latent_dim, hidden_dim)
        self.fc_output = nn.Linear(hidden_dim, input_dim)
    
    def encode(self, x):
        h = F.relu(self.fc1(x))
        mu = self.fc_mu(h)
        logvar = self.fc_logvar(h)
        return mu, logvar
    
    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5 * logvar)
        eps = torch.randn_like(std)
        return mu + eps * std  # Reparameterization trick
    
    def decode(self, z):
        h = F.relu(self.fc_decode(z))
        return torch.sigmoid(self.fc_output(h))
    
    def forward(self, x):
        mu, logvar = self.encode(x.view(-1, 784))
        z = self.reparameterize(mu, logvar)
        return self.decode(z), mu, logvar

def loss_function(recon_x, x, mu, logvar):
    BCE = F.binary_cross_entropy(recon_x, x.view(-1, 784), reduction='sum')
    KLD = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return BCE + KLD

Key Results: VAE achieved competitive likelihood scores on MNIST and Frey Face datasets while being orders of magnitude faster than MCMC-based methods. The generated samples were sharp and diverse.
Disentanglement: When trained with a modified objective (β-VAE), the model learns to disentangle underlying factors of variation, such as pose, lighting, and identity in face images.
